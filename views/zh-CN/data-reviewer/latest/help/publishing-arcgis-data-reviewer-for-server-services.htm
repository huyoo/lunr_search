发布 ArcGIS Data Reviewer for Server 服务

在本主题中

  1. 系统要求
  2. 设置 Data Reviewer for Server
  3. 部署 Data Reviewer for Server 服务
  4. 使用 DRS 配置实用程序配置地图服务

设置 ArcGIS Data Reviewer for Server 涉及以下操作：

  1. 安装 ArcGIS Server。
  2. 获取要安装 Data Reviewer for Server 的计算机的授权文件。
  3. 验证站点是否满足系统要求。
  4. 安装 Data Reviewer for Server。
  5. 部署托管 Data Reviewer 功能服务。
  6. 使用 ArcGIS Server 配置服务器对象扩展。

本文档涵盖步骤 3 到 6。

系统要求

Data Reviewer for Server 系统要求与 ArcGIS Server 的系统要求一致。要查看 ArcGIS Server 的系统规范（包括硬件要求、支持的操作系统以及 Web 服务器），请参阅 ArcGIS Server 系统要求。

设置 Data Reviewer for Server

Data Reviewer for Server使用地图服务、地理处理服务和服务器对象扩展来为分布式客户端提供服务。安装该软件后，必须配置地图服务和扩展模块。

  1. 使用授权向导安装 ArcGIS Server 并为您的软件授权。在授权过程中，请验证授权的功能列表中是否包括 datareviewersvr 条目。
  2. 安装 Data Reviewer for Server。
    注：

    软件安装向导完成后，请重新启动 ArcGIS Server。这将确保 ArcGIS Server 帐户可以访问您新安装的功能。

部署 Data Reviewer for Server 服务

按照以下各部分中的步骤可部署用于托管 Data Reviewer 功能的服务。

部署 Data Reviewer 批处理验证服务

Data Reviewer for Server 包括其批处理验证地理处理服务的服务定义 (*.sd) 文件。您需要发布此 .sd 文件才能创建地理处理服务。

  1. 在 Web 浏览器中打开管理器 URL 以登录到 ArcGIS Server Manager。

    您的管理器的 URL 为 https://<server name>:6443/arcgis/manager。

  2. 单击顶端通栏上的服务。
  3. 单击发布服务。
  4. 在发布服务对话框中，单击选择文件。
  5. 浏览至 ExecuteBatchJob.sd 文件，然后单击打开。

    该文件位于 <ArcGIS Server 安装文件夹>\ArcGISDataReviewerServer\Server<版本>\Service Definitions 中。

  6. 在发布服务对话框上单击下一步。
  7. 或者单击文件夹下拉箭头，然后选择文件夹名称。
  8. 或者单击集群下拉箭头，然后选择集群名称。
  9. 单击下一步。
  10. 单击发布。
    注：

    ExecuteBatchJob 地理处理服务的服务安全设置必须设置为公共，面向所有人，才能确保计划的批处理验证能够按预期完成。

部署 Data Reviewer 仪表盘服务

Data Reviewer for Server 包括其 Reviewer 地图服务的服务定义 (*.sd) 文件。需要发布此 .sd 文件才能创建服务。

  1. 在 Web 浏览器中打开管理器 URL 以登录到 ArcGIS Server Manager。

    您的管理器的 URL 为 https://<server name>:6443/arcgis/manager。

  2. 单击顶端通栏上的服务。
  3. 单击发布服务。
  4. 在发布服务对话框中，单击选择文件。
  5. 浏览至 reviewer.sd 文件，然后单击打开。

    该文件位于 <ArcGIS Server 安装文件夹>\ArcGISDataReviewerServer\Server<版本>\Service Definitions 中。

  6. 在发布服务对话框上单击下一步。
  7. 或者单击文件夹下拉箭头，然后选择文件夹名称。
  8. 或者单击集群下拉箭头，然后选择集群名称。
  9. 单击下一步。
  10. 单击发布。

使用 DRS 配置实用程序配置地图服务

接下来，必须使用配置实用程序来配置 Reviewer 地图服务。此实用程序还会将 Data Reviewer for Server 扩展模块添加到 ArcGIS Server 站点。

  1. 单击开始 > 所有程序 > ArcGIS > ArcGIS Data Reviewer for Server > DRS 配置实用程序。
  2. 验证 URL 文本框中所列的 URL 是否指向 ArcGIS Server Administrator Directory。
    服务目录的格式为 https://localhost:6443/arcgis/admin。
    注：

    如果已经在集群中安装 Data Reviewer for Server，则只需针对集群中的一台计算机运行 DRS 配置实用程序。配置更改就会自动应用到集群中的所有计算机。

  3. 在用户名和密码文本框中，输入您的 ArcGIS Server 主站点管理员用户名和密码。这是您用于登录 ArcGIS Server Manager 的帐户。
  4. 单击连接。
  5. 在“添加 DRS 扩展”区域中，单击浏览。
  6. 浏览至包含 ESRI.ReviewerServer.soe 文件的位置，然后单击打开。

    该文件位于 <ArcGIS Server 安装文件夹>\ArcGISDataReviewerServer\Server<版本>\Bin 中。

  7. 单击添加。
  8. 单击选择地图服务旁边的下拉箭头，然后从列表中选择 reviewer.MapServer。
  9. 单击选择新的 Reviewer 工作空间文本框旁边的浏览。
  10. 导航至 Reviewer 工作空间的位置，然后单击确定。
    注：

    ArcGIS Server 帐户必须至少具有对指向 Data Reviewer 工作空间的 .sde 连接文件所在文件夹的读取权限。如果您仅使用 Dashboard 服务，请勿选择文件地理数据库。连接到已版本化的 Reviewer 工作空间时，将应用地理数据库连接文件的 (.sde) 版本属性。

    注：

    如果已在多机部署中安装了 Data Reviewer for Server for Server，则 Reviewer 工作空间路径必须为 UNC 路径，并可供部署中的所有计算机访问。

  11. 单击应用。
    注：

    如果 Reviewer 地图服务无法启动，则可以从 ArcGIS Server Manager 重新启动该服务。

  12. 单击关闭以关闭 DRS 配置实用程序对话框。
  13. 浏览至 Data Reviewer for Server 服务目录 URL 以测试您的配置：

    https://<server name>:6443/arcgis/rest/services/reviewer/MapServer/exts/DataReviewerServer

    要进行多机部署，请浏览至服务目录 URL（通过 ArcGIS Web Adaptor：

    https://<web adaptor url>/arcgis/rest/services/reviewer/MapServer/exts/DataReviewerServer

    Data Reviewer 服务目录页面将显示 Reviewer 工作空间的位置和空间参考以及四个子资源：批处理验证、Dashboard、Reviewer 结果和实用工具。

Data Reviewer for Server 已安装完成。下一步，您可以配置 Web 客户端以启用基于 Web 的数据质量工作流或将批处理验证计划程序数据库更改为一个企业级数据库。
