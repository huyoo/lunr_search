设置数据库服务器

在本主题中

  1. 启动安装向导
  2. 安装 SQL Server Express
  3. 启用地理数据库存储
  4. 接下来的步骤

在 Standard 或 Advanced 许可等级下可用。

数据库服务器是一个已启用的用来存储地理数据库的 SQL Server Express 实例。运行数据库服务器安装向导以安装 SQL Server Express 实例、启用地理数据库存储并向此数据库服务器授予 ArcGIS GIS Server 许可。

必须在同一台计算机上安装 ArcGIS Server 和数据库服务器（SQL Server Express 实例）。因此，您需要检查您使用的 ArcGIS Server 版本以及您安装的 SQL Server Express 版本的操作系统要求，原因是您的工作组级数据库服务器仅可在上述两者都支持的 Microsoft Windows 操作系统上受到支持。

启动安装向导

运行 Database_Server_Workgroup_<version>_<part_number>.exe 以启动安装向导。

  1. 以具有 Windows 管理员权限的用户身份登录到计算机。
  2. 关闭计算机上的所有应用程序。
  3. 访问安装可执行文件。
    • 如果您已从 My Esri 下载安装文件，请解压缩该文件并运行可执行文件。
    • 如果您拥有 ISO 文件，则请使用 ISO 提取程序进入安装文件并运行可执行文件。

    随即打开 Database Server 安装向导。

  4. 阅读第一个窗口中的信息，然后单击下一步继续进行数据库服务器的安装。

如果已安装希望用作数据库服务器的 SQL Server Express 实例，则取消选中安装选项对话框中的 Microsoft SQL Server Express Edition 选项并跳到启用地理数据库存储；否则继续执行 SQL Server Express 安装。

安装 SQL Server Express

按照以下步骤使用数据库服务器安装向导在计算机上安装 SQL Server Express。

  1. 保持安装选项对话框中的 Microsoft SQL Server Express Edition 选项为选中状态并单击下一步。

    随即打开 Microsoft SQL Server 安装向导。

  2. 阅读并接受 Microsoft 许可协议。单击下一步继续。

    注：
    如果不接受，安装将终止。

  3. 安装产品更新，然后单击下一步。
    4. 安装向导将进行检查以确保在计算机上安装了所需组件，如 .NET Framework 和 Windows PowerShell。如果未安装，向导将安装这些组件。
  4. 打开时，将在要素选择面板上提供以下信息：
    1. 请确保已选择数据库引擎服务。
    2. 接受默认目录位置或选择自己的目录位置。
  5. 单击下一步。
  6. 将在实例配置面板中提供以下信息：
    1. 确保已选择命名实例。
    2. 指定实例名称。默认名称为 SQLExpress。可接受此名称或输入其他名称。

      注：
      实例名称不能超过 16 个字符，必须以字母开头，不能包含空格或特殊字符，且不得与现有实例名称重复。

    3. 接受创建实例的默认位置或浏览至其他位置。
  7. 单击下一步。
  8. 将在服务器配置面板中提供以下信息：
    1. 指定运行“数据库引擎”和“浏览器”服务的帐户或使用默认帐户。有关运行服务所用帐户的详细信息，请参阅 Microsoft SQL Server 文档。
    2. 确保“数据库引擎”和“浏览器”服务设置为自动启动。
    3. SQL Server Express 实例使用所在计算机的默认排序规则。如果需要选择其他排序规则，请单击排序规则选项卡并更改“数据库引擎”语言，然后再继续操作。确保使用不区分大小写的排序规则，因为 ArcGIS 不支持区分大小写的排序规则。
  9. 单击下一步。
  10. 将在数据库引擎配置面板中提供以下信息：
    1. 请确保已选择 Windows 身份验证模式。这是在目录树中使用通过数据库服务器节点访问的地理数据库时唯一受支持的身份验证模式。
    2. 为 SQL Server Express 实例添加一个管理员用户。

      如果不添加管理员用户，则无法继续安装。

    3. 单击数据目录选项卡并指定 SQL Server Express 文件的位置，或接受默认位置并单击下一步。
  11. 在错误报告面板中，可以选择将信息发送到 Microsoft。单击下一步继续安装。
  12. 安装完成后，单击关闭退出 Microsoft SQL Server 安装向导。

继续启用 SQL Server 实例来存储地理数据库。

启用地理数据库存储

指定用作数据库服务器的 SQL Server Express 实例，将服务器管理员添加到数据库服务器，并向此数据库服务器授予 ArcGIS GIS Server Workgroup 许可。

为了运行此设置部分，您必须以具有 SQL Server Express 实例 sysadmin 权限的用户身份进行登录。该用户可以是上一部分 10b 步骤中添加到实例的管理员用户或其他 sysadmin 登录帐户。

提示：

如果希望使用 MS-DOS 命令启动地理数据库，则请参阅使用命令启用 SQL Server Express 以存储地理数据库。

指定实例和管理员

指定希望用作数据库服务器的 SQL Server Express 实例。同时，还可以将其他登录帐户添加为服务器管理员。

  1. 从 SQL Server 实例名称列表中选择要用来存储地理数据库的 SQL Server Express 实例的名称，例如 MyServer\SQLExpress。

    警告：

    请确保选择正确的实例。如果选择有误，例如，选择了 SQL Server Enterprise Edition 实例，将不会正确启用该实例并且用户将无法在 SQL Server Express 中创建任何地理数据库。

  2. 在要启用的 SQL Server Express 实例（数据库服务器）中输入要充当服务器管理员的用户的登录名。

    已在安装 SQL Server Express 时添加了管理员登录帐户。如果您希望具有其他管理员，则仅需要在此处添加其他登录帐户。

    登录名应为在域名或计算机名之后加反斜线 (\) 和用户的名称，例如 WORLD\David。该用户在操作系统级不具有任何附加权限，但可以在数据库服务器上执行所有必要的管理员功能。

  3. 单击下一步。
  4. 如果您正在设置数据库服务器以存储桌面地理数据库，则请单击完成退出向导。如果您正在设置数据库服务器以存储桌面地理数据库，则请继续进行下一步骤。如果您正在设置数据库服务器以存储工作组级地理数据库，则请继续授权。

对工作组级数据库服务器进行授权

您可以使用授权 GIS Server Workgroup 时创建的密钥代码文件或者使用 Esri 向您提供的 ArcGIS GIS Server Workgroup 许可对数据库服务器进行授权。

  1. 在授权选项 对话框中选择适合实际情况的选项：
    • 如果使用 Esri 向您提供的 ArcGIS Server workgroup 许可对数据库服务器进行授权，则请选择第一个注册选项，我已安装了软件，需要对它进行授权。单击下一步继续。
    • 如果您已授权 GIS Server 站点或者您已通过电子邮件向 Esri 请求许可文件，则请选择第二个注册选项，我已收到来自 Esri 的授权文件，并且现在已准备好完成授权过程。输入或浏览至 \\Program Files\ESRI\License<ArcGIS release#>\sysgen 文件夹中文件或密钥代码文件的路径和名称。单击下一步并继续执行步骤 8。
  2. 如果您选择的是我已安装了软件，需要对它进行授权，请在授权方法 对话框中选择一个选项：
    • 如果您在运行该向导的计算机中具备 Internet 访问权限，可以选择立即使用 Internet 通过 Esri 进行授权。选择该方法将自动通过 Esri 对软件进行授权。
    • 如果您在运行该向导的计算机中不具备 Internet 访问权限，可选择在 Esri 的网站或通过电子邮件接收授权文件进行授权。选择该选项后授权信息将被保存，然后，您便可在具备 Internet 访问权限的计算机中将该信息以电子邮件的形式发送至 Esri 或提交至 Esri 网站。您将收到授权文件，然后便需要重新运行向导以启用地理数据库，以便对软件进行授权。
  3. 必须提供帐户信息才能通过 Esri 授权软件。输入必要信息，然后单击下一步继续。
  4. 提供您的组织信息以及您在组织中的任职信息，然后单击下一步。
  5. 在软件授权码对话框中输入 GIS Server 授权码。

    该授权码已和有关如何获取软件的信息一起发送给贵组织。如果没有授权码，请联系贵组织负责软件采购和维护合同的相关人员。

  6. 如果选择的是选项立即使用 Internet 通过 Esri 进行授权，那么您提供的信息将自动发送至 Esri，随后您的软件便可得到授权。之后便会在计算机注册表的 SQL Server Express Services 条目中写入一个条目，从而使该 SQL Server Express 的示例具备存储地理数据库的能力。前进至步骤 8。
  7. 如果您选择的是选项在 Esri 的网站或通过电子邮件接收授权文件进行授权，则会立即弹出电子邮件授权对话框。单击保存创建文件，该文件可通过电子邮件提交至 Esri 或传送到另一台计算机中，然后通过 My Esri 提交。

    收到 Esri 的授权文件后，请重新运行该向导，在授权选项 对话框中选择我已收到来自 Esri 的授权文件，现在准备完成授权过程选项。

  8. 要完成授权，请单击完成。

接下来的步骤

现在，已安装 SQL Server Express 实例且已准备存储地理数据库，接下来请配置 ArcMap 以连接到数据库服务器。
