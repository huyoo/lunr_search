Mapping and Charting Solutions for Server sites

The setup package is designed to detect and upgrade an existing installation of Mapping and Charting Solutions for Server on the target system.

Upgrading from an earlier version of Maritime Chart Service

To upgrade from a Maritime Chart Service version earlier than 10.5, you will need to upgrade the software, Maritime Server SOE, and map services with Maritime Chart Service configured. If you are upgrading an existing 10.5 or a more recent version, follow the steps in the Upgrading the Maritime Chart Service section below.

注：

If the Maritime Chart Service has never been installed on this machine, see Installing Mapping and Charting Solutions for Server.

  1. Log in to the machine as a user with administrative privileges.
  2. Start the Mapping and Charting Solutions for Server setup program.
  3. Click Next on the Welcome to the Mapping and Charting Solutions for Server Setup Program pane of the installation wizard.

    The License Agreement dialog box appears.

  4. Review the license agreement and accept it, or exit if you do not agree with the terms.
  5. Click Next.
  6. The Maritime feature should already be selected. If not, click the drop-down arrow and choose This feature will be installed on local hard drive.
  7. Click Next.

    The Ready to Install the Program dialog box appears.

  8. Click Install to begin installation.
  9. Once the installation is complete, click Finish.
  10. Repeat steps 1–9 for each machine in a multiple-machine site.
  11. Log in as a Server Manager by launching https://gisserver.domain.com:6443/arcgis/manager.
  12. Click Services.
  13. Stop the map service with Maritime Chart Service capabilities configured.
  14. Move the controlfiles folder from <ArcGIS Server installation drive>\arcgisserver\directories\maritimeserver\maritimechartservice\controlfiles to a backup location.
  15. Copy the new controlfiles folder from <ArcGIS Install Directory>\MaritimeServer\Server<version>\maritimechartservice to <ArcGIS Server installation drive>\arcgisserver\directories\maritimeserver\maritimechartservice.
    注：

    Consolidate any modifications from the backup control files with the new control files. For example, coordinate systems and AutoCellUpdate settings.

  16. If the map service was configured to read S-63 encrypted datasets, copy the S-63 user-permit.txt file from the backup controlfiles folder to the new controlfiles folder.
  17. Delete the .senc files from <ArcGIS Server installation drive>\arcgisserver\directories\maritimeserver\maritimechartservice\sencs.

    The .senc files must be rebuilt after upgrading. Deleting them will significantly increase creation performance.

  18. Repeat steps 11–17 for each map service with the Maritime Chart Service capabilities configured.
  19. Log in as an administrator by launching https://gisserver.domain.com:6443/arcgis/manager.
  20. Click Site.
  21. Click Extensions in the GIS Server window.
  22. Click X at the end of the MaritimeServer.soe to delete the extension.
  23. Click Add Extension.
  24. Click Browse and browse to <ArcGIS Install Directory>\MaritimeServer\Server<version>\Bin.
  25. Click MaritimeServer.soe.
  26. Click Add.

    MaritimeServer.soe is now successfully added.

  27. Click the Services tab.
  28. Click your map service.
  29. In the Map Services window, click Capabilities.
  30. Check the Maritime Chart Service check box.
  31. Verify that the datasetsDirectory, controlFilesDirectory, and workingDirectory properties point to the correct locations.

    For more information on these properties, see Modifying Maritime Chart Service capabilities properties.

  32. Click Save and Restart to make Maritime Chart Service available.
    注：

    To confirm if the Maritime Chart Service capabilities are working properly in the map service, verify that the .senc files in the sencs folder are current.

  33. Repeat steps 27–32 for each map service with Maritime Chart Service capabilities configured.
注：

Since the map service URL has changed at 10.5, you will need to:

  • Modify any existing maps referencing this service
  • Redeploy your Web AppBuilder application using maritime widgets

For more information on the latest maritime widgets, see GitHub.

For what's new, see What's new for ArcGIS for Maritime: Server.

注：

Now that you have upgraded ArcGIS for Maritime: Server, you can deploy Products on Demand. For more information, see Deploying Products on Demand (PoD)

Upgrading the Maritime Chart Service

警告：

If you are upgrading from an existing 10.3, or 10.4, version of Mapping and Charting Solutions for Server, you must delete your existing MaritimeServer.soe first. This is due to the new URL path at 10.5.

For more information, see Upgrading from earlier versions of Maritime Chart Service.

To upgrade from a Maritime Chart Service version 10.5 or greater, you will need to upgrade the software, Maritime Server SOE, and map services with Maritime Chart Service configured.

注：

If the Maritime Chart Service has never been installed on this machine, see Installing Mapping and Charting Solutions for Server.

  1. Log in to the machine as a user with administrative privileges.
  2. Start the Mapping and Charting Solutions for Server setup program.
  3. Click Next on the Welcome to the Mapping and Charting Solutions for Server Setup Program pane of the installation wizard.

    The License Agreement dialog box appears.

  4. Review the license agreement and accept it, or exit if you do not agree with the terms.
  5. Click Next.
  6. The Maritime feature should already be selected. If not, click the drop-down arrow and choose This feature will be installed on local hard drive.
  7. Click Next.

    The Ready to Install the Program dialog box appears.

  8. Click Install to begin installation.
  9. Once the installation is complete, click Finish.
  10. Repeat steps 1–9 for each machine in a multiple-machine site.
  11. Log in as a Server Manager by launching https://gisserver.domain.com:6443/arcgis/manager.
  12. Click Services.
  13. Stop the map service with Maritime Chart Service capabilities configured.
  14. Move the controlfiles folder from <ArcGIS Server installation drive>\arcgisserver\directories\maritimeserver\maritimechartservice\controlfiles to a backup location.
  15. Copy the new controlfiles folder from <ArcGIS Install Directory>\MaritimeServer\Server<version>\maritimechartservice to <ArcGIS Server installation drive>\arcgisserver\directories\maritimeserver\maritimechartservice.
    注：

    Consolidate any modifications from the backup control files with the new control files. For example, coordinate systems and AutoCellUpdate settings.

  16. If the map service was configured to read S-63 encrypted datasets, copy the S-63 user-permit.txt file from the backup controlfiles folder to the new controlfiles folder.
  17. Delete the .senc files from <ArcGIS Server installation drive>\arcgisserver\directories\maritimeserver\maritimechartservice\sencs.

    The .senc files must be rebuilt after upgrading. Deleting them will significantly increase creation performance.

  18. Repeat steps 11–17 for each map service with the Maritime Chart Service capabilities configured.
  19. Log in as an administrator by launching https://gisserver.domain.com:6443/arcgis/admin.
  20. Click Uploads.
  21. Click Upload.
  22. Click Choose File and browse to the MaritimeServer.soe file located in <ArcGIS Install Directory>\MaritimeServer\Server<version>\Bin.
  23. Click Upload to add the SOE to the site.
  24. Copy the Item ID for use in the Update step.
  25. Click Home.
  26. Click Services.
  27. Click Types.
  28. Click Extensions.
  29. Click Update and paste the Item ID.
  30. Click Update.
  31. Return to ArcGIS Server Manager https://gisserver.domain.com:6443/arcgis/manager.
  32. Click Services.
  33. Click the map service for which you want to enable the Maritime Chart Service.
  34. Click Capabilities.
  35. Verify that the datasetsDirectory, controlFilesDirectory, and workingDirectory properties point to the correct locations.

    For more information about these properties, see the Modifying Maritime Chart Service capabilities properties section of Setting Maritime Chart Service properties.

  36. Click Save and Restart to make the Maritime Chart Service available.
    注：

    To confirm if the Maritime Chart Service capabilities are working properly in the map service, verify that the .senc files in the sencs folder are current.

  37. Repeat steps 31–36 for each map service with Maritime Chart Service capabilities configured.
注：

You will need to redeploy your existing Web AppBuilder application using Maritime widgets to take advantage of the new Display Parameters.

For more information on the latest maritime widgets, see GitHub.

For what's new, see What's new for ArcGIS for Maritime: Server.

注：

If you haven’t already deployed Products on Demand, see Deploying Products on Demand (PoD) for more information. Otherwise, see the following section: Upgrading Products on Demand.

Upgrading Products on Demand

注：

If ArcGIS for Maritime: Server has never been installed on this machine, see Installing Mapping and Charting Solutions for Server.

Before starting your upgrade process, you should backup your existing PoD deployment. Doing so will simplify the new deployment by preserving some of your settings. It is critical to backup your existing deployment if you made customization to the web application or PoD configuration files.

Now that you have backed up your existing PoD deployment you can now follow the Deploying Products on Demand (PoD) steps to redeploy the new version of PoD. Once complete you will need to merge any previous configuration changes.

Common software upgrade questions

If I have a multiple-machine server deployment, do I need to upgrade each machine?

Yes. All machines in the multiple-machine deployment must be upgraded to the same version of ArcGIS Server and Mapping and Charting Solutions for Server.

Do I need to uninstall a previous version of Mapping and Charting Solutions for Server before installing Mapping and Charting Solutions for Server?

No. If you already have Mapping and Charting Solutions for Server installed, the setup will automatically upgrade it to 10.8.

Do I need to back up anything before installing Mapping and Charting Solutions for Server?

  • Maritime: Server—See the Upgrading the Maritime Chart Service section.
  • Defense Mapping for Server—All services and site configuration changes are preserved when upgrading.
  • Production Mapping for Server—All services and site configuration changes are preserved when upgrading.

Do I need to stop my GIS services and ArcGIS Server before installing Mapping and Charting Solutions for Server 10.8?

The setup will restart ArcGIS Server after the installation.

If I upgrade to ArcGIS Server 10.8, do I also need to upgrade Mapping and Charting Solutions for Server?

Yes. The version of Mapping and Charting Solutions for Server must match ArcGIS Server. You cannot run mismatched versions of software.
