什么是全局设置？

ArcGIS GeoEvent Server 中提供的全局设置可用于根据各种部署环境正确配置 GeoEvent Server。例如，您可能想要配置 Internet 访问的代理设置、设置自动备份的频率，或增加支持的最大信息大小。

全局设置概览

以下概述了 GeoEvent Server 中提供的全局设置。某些设置可能需要来自 IT 部门或第三方供应商的其他信息才能正确配置。如果设置不正确，可能会导致功能减少和/或数据丢失。

自动备份设置

自动备份设置可用于启用或禁用整个 GeoEvent Server 配置的计划备份。自动备份默认处于启用状态，会在每天中午 12 时进行备份，最多存储 10 天的配置。在站点 > GeoEvent > 数据存储下配置自动备份文件夹。可以计划每日、每周或每月运行一次备份。最大备份设置可用于存储配置的数量。如果达到配置的最大数，GeoEvent Server 将自动开始删除最旧的文件。

Geofence Manager 设置

Geofence Manager 设置可用于配置针对地理围栏考虑的空间运算符、几何精度和数据存储的选项。

  • 第一个 GeoEvent 触发 Enter - 与空间过滤处理器（如过滤器、GeoTagger、事件检测器等）配合使用，以确定地理围栏内部是否出现定义追踪的第一个 GeoEvent，以及是否应触发 Enter 条件。默认值为 true。
  • 第一个 GeoEvent 触发 Exit - 与空间过滤处理器（过滤器、GeoTagger、事件检测器等）配合使用，以确定地理围栏外部是否出现定义追踪的第一个 GeoEvent，以及是否应触发 Exit 条件。默认值为 false。
  • 地理围栏加速程度 - 向空间评估授予的资源优先级。该值为 0 到 3 之间的非负整数。值越高表示使用速度越快，但初始加载时间更长，内存使用量也更多。默认值为 3。
    • 0 - 不加速
    • 1 - 轻度加速（消耗的内存量最少）
    • 2 - 中等加速（添加地理围栏消耗的内存最多，花费时间最长，但工作速度最快）
    • 3 - 快速加速（添加地理围栏消耗的内存最多，花费时间最长，但工作速度最快）
  • 地理围栏清除已启用 - 确定是否自动移除过期的地理围栏（以 END_TIME 标记字段为基础）。默认值为 false。
  • 地理围栏清除间隔（秒）- 与地理围栏清除已启用设置配合使用，用于确定检查过期地理围栏的间隔。默认值为 300 秒。
  • 空间索引最高细节级别 - 存储地理围栏几何的精度级别。值越高，生成的索引对于较小地理范围（较小地理围栏）越高效，但同时消耗的内存也越多。该值应介于 4 到 32 之间，默认值为 16。

几何处理器设置

管理员可利用几何处理器设置启用或禁用可选线性单位，以便用于 GeoEvent Server 中的可用处理器。

HTTP 代理设置

管理员可利用 HTTP 代理设置配置 GeoEvent Server，从而通过 HTTP 代理访问在线资源。这一过程通常是内部站点用户必须先验证身份，而后才能连接到 Internet。

  • 从 HTTP 代理中排除的主机 - 应该不通过 HTTP 代理访问的主机。通常情况下该设置定义内部主机。该属性的值是由 ''|'' 字符分隔的主机列表。此外，可使用通配符 ''*'’ 进行模式匹配。例如，http.nonProxyHosts="*.foo.com|localhost" 表示即使指定了代理服务器，也应直接访问 foo.com 域中的各台主机和本地主机。
  • HTTP 代理主机 - HTTP 代理服务器的主机名或地址。
  • HTTP 代理密码 - 在代理需要身份验证时向服务器提供的密码。
  • HTTP 代理端口 - HTTP 代理服务器的端口号。
  • HTTP 代理用户 - 在 HTTP 代理需要身份验证时向服务器传输的用户。

HTTPS 代理设置

管理员可利用 HTTP 代理设置配置 GeoEvent Server，从而通过 HTTPS 代理访问在线资源。这一过程通常是内部站点用户必须先验证身份，而后才能连接到 Internet。这些设置与 HTTP 代理设置的功能相同，但是使用 SSL 通信确保安全性。

  • 从 HTTPS 代理中排除的主机 - 应该不通过 HTTPS 代理访问的主机。通常情况下该设置定义内部主机。该属性的值是由 ''|'' 字符分隔的主机列表。此外，可使用通配符 ''*'’ 进行模式匹配。例如，https.nonProxyHosts="*.foo.com|localhost" 表示即使指定了代理服务器，也应直接访问 foo.com 域中的各台主机和本地主机。
  • HTTPS 代理主机 - HTTPS 代理服务器的主机名或地址。
  • HTTPS 代理密码 - 在代理需要身份验证时向服务器提供的密码。
  • HTTPS 代理端口 - HTTPS 代理服务器的端口号。
  • HTTPS 代理用户 - 在 HTTPS 代理需要身份验证时向服务器传输的用户。

事件点管理器设置

事件点管理器设置可定义事件检测器处理器内使用的关闭事件和开放事件的最大数量。默认情况下，事件检测器处理器可以保留 1000 个开放事件和 1000 个关闭事件的当前状态。增加该值可监控其他 TRACK_ID 标记事件，但会消耗更多内存使用量。

输入和输出设置

输入和输出设置可用于配置输入缓存区容量 (MB)。该值确定内存中可针对输入保留的最大消息大小（以 MB 为单位）。大批量的事件或含高度复杂几何的事件有时可能会超过默认设置 20。应将该值设置在 1 到 2047 之间。对该设置所做的任何更改都不会反映在现有输入中，建议重新启动 ArcGIS GeoEvent Server 服务。

SMS 设置

SMS 设置为大多数手机运营商提供短消息服务 (SMS) 端点。发送文本消息时，将使用这些设置自动填充输出，以包括 number@carrier.com（例如，针对 Verizon，将为 5551234567@vtext.com）。可以更新端点和添加新运营商。

SMTP 设置

管理员可利用 SMTP 设置配置特定于组织的简单邮件传输协议 (SMTP) 连接详细信息。该设置与发送电子邮件输出连接器配合使用，应按照组织的系统管理员或 IT 部门进行配置。

  • 默认 SMTP 身份验证 - 用于连接到 SMTP 服务器的身份验证方法（匿名、SSL 或 TLS）。
  • 默认 SMTP 密码 - 在身份验证方法需要凭据时向服务器提供的密码。
  • 默认 SMTP 端口 - SMTP 服务器的端口号。
  • 默认 SMTP 服务器主机 - SMTP 服务器的主机名或地址。
  • 默认 SMTP 用户名 - 在身份验证方法需要凭据时向服务器提供的用户名。

Spatiotemporal Big Data Store 设置

时空大数据存储设置定义创建新时空大数据存储数据源时的默认设置。可在 GeoEvent Manager 中的发布对话框内更改值。

  • 默认碎片数 - 时空大数据存储的数据将被划分的默认碎片数。建议将该值设置为时空大数据存储集群中的节点数，但是该值不得超过节点数。可接受的值为 1 到 N，默认值为 3。
  • 默认复制因子 - 发布到时空大数据存储时使用的默认复制因子。可接受的值为 0 到 N，其中 N 是时空大数据存储计算机的最大数量。

流服务设置

流服务设置可用于从多方面调整流服务连接详细信息和消息大小。

  • 客户端 ping 间隔（秒）- 定义服务器向所有客户端发送“ping”，以确保这些客户端处于运行状态并告知它们服务器处于运行状态（即使当前没有数据流经）的频率（以秒为单位）。客户端应回复“ping”以保持通信（大多数浏览器自动进行处理）。客户端不应将超时期间设置为小于此值，否则可能会自行断开连接。值必须介于 1 到 2,073,600（24 天）之间。
  • 聚类消息生命期（秒）- 定义消息为等待客户端进行使用而可在 ArcGIS 聚类中保留的时间（以秒为单位）。如果聚类中的消息未能在此时间段内得到处理，则会遭到清除。值必须介于 0 到 2,073,600（24 天）之间。
  • 最大 Web 套接字文本消息大小（字节）- 定义通过 WebSocket 发送文本消息时的最大大小（以字节为单位）。超过此大小的文本消息无法传送。值必须介于 1 到 2,147,483,647 (2GB) 之间。
  • 相关要素刷新间隔（秒）- 定义服务器应定期查询要素服务以查看空间过滤器中要素的频率（以秒为单位）。适用于在发布流服务时选择了相关要素服务，并从客户端添加了空间过滤器的情况。
  • 支持线程计数 - 用于管理所有客户端连接的线程数。该设置可提供性能更高的隔离，从而避免缓慢的客户端影响其他客户端。通常情况下使用的线程越多，消耗的服务器资源越多。值必须介于 1 到 100 之间。
  • WebSocket 缓冲区大小（字节）- 由 WebSocket 分配用于保持数据传输状态（数据已发送，但接收方尚未读取）的大小（以字节为单位）。值必须介于 1 到 2,147,483,647 (2GB) 之间。
  • 允许的最大 WebSocket 空闲时间（秒）- 与空闲客户端断开连接前，服务器可保持空闲客户端处于连接状态的最长时间（以秒为单位）。值必须介于 1 到 2,073,600（24 天）之间。
