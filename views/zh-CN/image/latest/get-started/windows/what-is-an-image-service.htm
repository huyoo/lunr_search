什么是影像服务？

在本主题中

  1. 创建影像服务
  2. 发布影像服务
  3. 缓存影像服务

影像服务是一种处理和大范围共享影像集合的方法。其为存储、处理、分析和共享影像和栅格产品提供了很大的灵活性和可扩展性。

影像服务既灵活又强大，并且包含各种日期类型，可以满足您的需求和并适用于各种应用程序。如果具有多个输入影像，则可以选择创建一个影像图层作为影像镶嵌或影像集合，或者为每个影像输入创建一个影像图层。您还可以从单个影像输入创建一个影像图层。

影像服务可以从单个影像创建，该影像可以具有多个波段或多个维度，并且大小可大可小。可以从大量相似影像中创建影像服务。影像服务可以由覆盖州或县的正射镶嵌集合组成，每个正射镶嵌都具有不同的元数据和采集日期。您还可以将飞机或无人机直接捕获的大量影像作为正射镶嵌影像进行快速访问，以提供对单个影像和元数据的访问权限。影像服务还可以包括大量的数字地形模型或覆盖国家/地区或整个世界的卫星影像存档。此外，数据科学家可以使用多维栅格设置影像服务，以便对结果感兴趣的许多用户可以进行交互式分析和共享。

在所有这些情况下，影像服务都会使通过桌面或 Web 应用程序连接的用户可以快速访问影像和栅格。当用户访问影像服务时，ArcGIS Image Server 可以快速执行所需的处理过程，并将所需的影像动态镶嵌在一起。这些应用程序可以通过无缝镶嵌访问影像，也可以查询每个影像的元数据以仅显示一种数据选择，或者跨数据集进行处理以查找差异或执行分类。这些过程可以包括重新投影、正射校正、裁剪、应用复杂影像处理算法等等。

这些操作在分布式处理、存储和共享环境中执行，您可以配置单机或多机部署。此外，您可以将不同的角色注册并分配给不同的影像服务器计算机（例如栅格分析和影像托管服务器角色），以及在栅格分析部署中完成分布式栅格数据存储。

影像服务具有很多优势：

  • 提供对 ArcGIS Pro、Web 地图客户端、其他桌面和移动应用程序及 OGC Web 地图服务 (WMS)、Web 覆盖服务 (WCS)、Web 地图切片服务 (WMTS) 和 Keyhole 标记语言 (KML) 服务中的托管影像和栅格数据的快速访问。可以控制哪些用户组和利益相关者对哪些影像服务具有访问权限，以及他们可以如何使用这些影像服务。
  • 管理庞大的影像集合，在镶嵌数据集中对其进行编译，并以系统且简单的方式提供更新。
  • 通过动态影像服务可以快速动态处理影像。动态影像服务可基于单个源按需创建多个图像产品，而无需存储和保留预处理数据。可以针对要处理的图像更改函数和参数，结果将在平移和缩放影像时实时显示。
通过将您的图像资产设置为以这种方式进行访问，贵组织的用户便可以在其图像利用、分析和地图制作工作流中应用影像和栅格数据。

创建影像服务

发布源影像、已处理影像和增值栅格产品时，将创建影像服务。发布影像服务后，客户端可以使用 ArcGIS Server 连接或 REST 连接到影像服务。还可以发布具有 WMS 或 WCS 功能的影像服务，以允许客户端可以访问更多种应用程序和设备中的影像服务。

创建影像服务时，需要考虑它的使用和共享方式。要考虑的基本问题包括：

  • 影像服务是由一个还是多个栅格数据集组成的？ 如果具有多个栅格，它是影像镶嵌、影像集合还是多个影像图层？
  • 栅格数据是作为 GIS 图层的影像背景还是作为分析的输入来查看？
  • 影像是否具有多个波段以及是否需要增强？
  • 是否需要影像处理？
要了解创建影像服务时要考虑的重要因素，请参阅准备影像服务。

发布影像服务

影像服务允许您可以通过 Web 服务提供和共享影像和栅格数据，从而使数据可用于 Web 客户端、桌面应用程序和现场应用程序。

作为影像服务的发布者，您可以确定使用该影像服务的利益相关者可以使用哪些功能（例如动态查看和定义影像处理模板）。

从影像服务的重要概念开始，在一系列主题中提供了有关准备、发布和处理影像服务的相关信息。

缓存影像服务

影像服务缓存可改善客户端应用程序中影像服务的性能。缓存影像服务可以快速地显示影像，因为 ArcGIS Server 不必动态生成影像。相反，会在服务器缓存目录中生成、索引并存储影像切片。只要有请求，服务器就会根据平移位置和缩放级别来分发这些影像切片。

对影像服务进行缓存时，将生成一个影像服务，并且可根据其用途来进行访问。一种用途是以切片服务形式提供对影像的最快访问。另一种用途是提供对数据的访问，以用于查询、下载、访问各个项并在处理和分析中使用。

从什么是影像服务缓存？开始，在一系列主题中提供了有关准备和创建缓存影像服务的相关信息。

有关缓存图像服务（包括工作流）的其他资源和相关信息，请参阅提供缓存影像和创建缓存影像服务。
