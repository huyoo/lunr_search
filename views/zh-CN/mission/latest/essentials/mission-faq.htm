什么是 ArcGIS Mission？

ArcGIS Mission 是一款地理空间通信和态势感知工具。您可以使用此工具创建、共享和监控业务材料，同时此工具支持实时对等通信。

有关详细信息，请参阅什么是 ArcGIS Mission。

Mission 包含哪些组件？

ArcGIS Mission 由 web 组件、移动组件和服务器组件构成。 ArcGIS Mission Manager 是用于创建和组织任务的 web 应用程序。ArcGIS Mission Responder 是允许外业任务成员参与任务的移动应用程序。 ArcGIS Mission Server 是允许 Manager 和 Responder 进行通信的服务器角色。

ArcGIS Mission 是否可用于 ArcGIS Online？

否。ArcGIS Mission 1.0 在 ArcGIS Online 中不可用，仅适用于 ArcGIS Enterprise 10.8 及更高版本。

ArcGIS Mission Manager 是否专用于特定角色？

ArcGIS Mission Manager 的用户可以分为三种不同的角色：编辑者、所有者和管理员，这三种角色均由您的权限决定。这三种角色均可成为任务成员，可以查看、协作并积极参与任务。有关详细信息，请参阅 ArcGIS Mission Manager 角色。

为什么我无法发送地理消息？

要发送地理消息，必须在任务地图上绘制标记图形。您也可以选择在其中添加文本信息（例如，标题、消息、地理消息的接受者），甚至添加附件（例如，影像）。有关详细信息，请参阅发送消息。

我可以将自己添加到任务吗？

默认情况下，任务所有者将自动添加为任务的成员。如果需要，具有管理员角色的门户用户可以轻松将其自己添加到任务。要完成此操作，导航到所需任务的“任务详细信息”，然后单击添加我以将其自己添加到任务。成为任务成员之后，管理员可以对任务的多个部分执行编辑。有关详细信息，请参阅概览。

创建任务后是否可以更改我的任务地图？

是。通过所需任务的“任务详细信息”视图中的“地图”选项卡，可以编辑任务地图以添加或移除附加参考图层。有关详细信息，请参阅任务地图。

为什么尽管我已被添加到任务，但仍无法从我的 Responder 应用程序中看到任务？

任务是否处于活动状态？ 如果任务处于草稿或完成状态，则在任务列表中不再可见。有关详细信息，请参阅任务状态。

为什么我的当前位置未显示在实时地图上？

请确保您已连接到任务。Responder 可以打开并参与任务，尽管您当前未传输您的位置，或未从其他使用 Responder 的用户处接受位置数据。

我如何登录到 Responder？

下载应用程序后，您必须输入组织的任务门户 URL 并使用帐户信息登录。有关详细信息，请参阅访问 Responder。

我可以在 Responder 中更改团队或将自己添加到其他团队吗？

不可以。Responder 无法编辑任何任务配置或详细信息。任何对此信息的更改都必须由作为任务成员的任务所有者或管理员通过 Manager 应用程序进行。

ArcGIS Mission Server 是否需要单独的安装或许可？

是。 ArcGIS Mission Server 是一个单独的安装程序，也需要单独的许可才能够使用。

我正在尝试静默安装，但是命令提示符持续返回错误。应该怎么办?

您是否正确输入了命令行参数？ 参数需要区分大小写，必须正确输入，否则将返回错误。

安装 ArcGIS Mission Server 之前有哪些要求？

简要回答，如果您具有 ArcGIS Enterprise 10.8 基础安装，则很可能已经满足 ArcGIS Mission Server 的硬件和软件要求。有关详细信息，请参阅系统要求。

我可以在 Responder 中更改团队或将自己添加到其他团队吗？

不可以。Responder 无法编辑任何任务配置或详细信息。任何对此信息的更改都必须由作为任务成员的任务所有者或管理员通过 Manager 应用程序进行。

是否可以使用 API 对 ArcGIS Mission 进行自定义？

在当前版本中，ArcGIS Mission REST API 不可用，但在将来的版本中将可用。
