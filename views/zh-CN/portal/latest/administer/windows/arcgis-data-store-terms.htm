以下部分定义了与 ArcGIS Data Store 相关的术语。由于 ArcGIS Data Store 是 ArcGIS Enterprise 部署的一部分，因此此处也包含其他术语以阐明整体 ArcGIS Enterprise 环境中的数据存储。

ArcGIS Data Store

ArcGIS Data Store 是一个软件组件，可用于为门户的托管要素图层、托管场景图层和时空要素图层配置数据存储。

请注意数据存储针对任何数据存储位置而言都是通用的术语。可以通过 ArcGIS Data Store 来配置数据存储类型的子集。通过 ArcGIS Data Store 配置的数据存储始终作为 ArcGIS Enterprise 部署的一部分使用并由 ArcGIS 托管，且会注册到您门户的托管服务器。

ArcGIS Data Store 帐户

在 Microsoft Windows 计算机上，ArcGIS Data Store 账户是您在运行 ArcGIS Data Store 服务时所使用的登录账户。安装 ArcGIS Data Store 时指定此帐户。

在 Linux 计算机上，ArcGIS Data Store 账户是您在安装 ArcGIS Data Store 时用于登录计算机的账户。

有关详细信息，请参阅 ArcGIS Data Store 帐户。

ArcGIS Data Store 配置存储

ArcGIS Data Store 配置存储中包含了有关其注册到的 ArcGIS GIS Server 站点、数据存储中的其他计算机和数据存储属性的信息。

每种类型的 ArcGIS Data Store 都有自己的配置存储，且其中都包含了特定于其自身类型的信息。例如，如果您的 ArcGIS Enterprise 部署具有关系数据存储、切片缓存数据存储和时空大数据存储，那么您的部署将具有三个 ArcGIS Data Store 配置存储。

ArcGIS Data Store 目录

这是您为存储 ArcGIS Data Store 运行所需文件指定的目录。该目录必须存储在 ArcGIS Data Store 安装本地。

最初，该目录存储关系数据存储的备份文件，但是一旦您创建了关系数据存储，则应该转而将其配置为在共享网络位置创建备份文件。

ArcGIS Enterprise

ArcGIS Enterprise 是用于创建 Web GIS 的产品线。它包括 ArcGIS Server、Portal for ArcGIS、ArcGIS Data Store 和 ArcGIS Web Adaptor。

请参阅什么是 ArcGIS Enterprise？ 获取更多信息，该内容位于 Portal for ArcGIS 管理员指南中。

ArcGIS GeoAnalytics Server

许可 ArcGIS Server 站点以充当特定角色。ArcGIS GeoAnalytics Server 是一个可用于通过分布式计算执行大数据分析联合服务器。

在 ArcGIS Data Store 环境下，您需要一个面向 Map Viewer 或 ArcGIS Pro 中 GeoAnalytics 工具 的数据输出的时空大数据存储。如果您拥有联合 Map Viewer，则这些工具在您的门户的 ArcGIS GeoAnalytics Server 中可用。

请参阅什么是 ArcGIS GeoAnalytics Server？ 以获取详细信息。

ArcGIS GeoEvent Server

许可 ArcGIS Server 站点以充当特定角色。ArcGIS GeoEvent Server 是独立或联合服务器站点，可用于流式传输实时数据。

在 ArcGIS Data Store 环境下，联合 ArcGIS GeoEvent Server 可将数据从流服务存档到门户托管服务器上运行的要素图层中。存档的数据将存储在时空大数据存储中。请注意要素图层不是托管要素图层，当您从门户中删除该要素图层时，既不会删除要素服务，也不会删除存储在时空大数据存储中的数据。

有关存档数据的详细信息，请参阅 GeoEvent Server 帮助中的管理时空大数据存储。

ArcGIS GIS Server

（也称为 GIS Server。）

许可 ArcGIS Server 站点以充当特定角色。ArcGIS GIS Server 可用于 GIS 资源，如地图服务、要素服务、地理处理服务（工具）、地理编码服务（定位器）、地理数据服务和场景服务。

门户的托管服务器被许可为 ArcGIS GIS Server 站点才能发布托管 web 图层所需的服务类型。

ArcGIS Image Server

许可 ArcGIS Server 站点以充当特定角色。ArcGIS Image Server 是独立或联合服务器站点，可用于缓存或动态影像服务。

在 ArcGIS Data Store 的环境下，如果您想要将托管影像图层发布到门户中，则使用 ArcGIS Enterprise 所基于的 ArcGIS Data Store 部署必须具有联合 ArcGIS Image Server。

数据存储

在 ArcGIS 环境下，数据存储是您保存 ArcGIS 所用数据的任何位置。

在 ArcGIS Server 的特定环境下，数据存储是注册到 ArcGIS Server 站点的文件、数据库、地理数据库或云存储位置。将这些位置注册到 ArcGIS Server 站点可确保您发布的服务有权访问数据。

注册到 ArcGIS Server 的用户维护数据存储指向数据源；因此数据将被保留在该数据存储中，并且不能被复制到任何其他位置。当将服务发布到 ArcGIS Server 站点后，填充服务的数据将来自于注册的源数据。

通过 ArcGIS Data Store 实施的数据存储也会注册到 ArcGIS Server，但它们由 ArcGIS 进行维护。当您发布某些托管 web 图层时，ArcGIS 会将数据复制到这些数据存储其中之一。

请注意，ArcGIS Enterprise 门户中的 web 图层是运行在 ArcGIS Server 站点的服务。如果在发布服务时数据被复制到数据存储中，则在门户中生成的 web 图层将是托管 web 图层。有关详细信息，请参阅 Portal for ArcGIS 用户帮助中的服务和门户项目。

GIS Server

请参阅 ArcGIS GIS Server。

联合服务器

联合服务器是您集成到门户中的 ArcGIS Server 站点。联合服务器与门户共享身份验证，允许您在门户中自动为发布到联合服务器的服务创建项目。

有关详细信息，请参阅关于将门户与 ArcGIS Server 配合使用。

GeoAnalytics 工具

GeoAnalytics 工具 用于使用分布式计算对大量数据执行要素分析。可以使用 GeoAnalytics 工具 在空间和时间环境中查找模式并聚合数据。

Map Viewer 中的 GeoAnalytics 工具 在与门户联合的 GeoAnalytics Server 上作为地理处理服务运行。GeoAnalytics 工具 也可以用作 ArcGIS Pro 中的地理处理工具。

有关详细信息，请参阅 Portal for ArcGIS 用户帮助中的标准和 GeoAnalytics 工具 要素分析比较以及在 Map Viewer 中使用 GeoAnalytics 工具。

托管 web 图层

托管 Web 图层是门户中的项目，它与运行在门户的托管服务器中的服务相对应。*

*其中一个例外情况就是托管影像图层；他们运行在联合 ArcGIS Image Server 而不是托管服务器中。

提示：

如果图层所有者或门户管理员打开门户中的项目页面，则（托管）会包括在概述选项卡的图层类型标注中。在 ArcGIS Pro 中查看托管图层时，会像在“目录”窗格或窗口中查看时一样对其进行标注。

在您发布服务时，填充服务的数据以及产生的托管 web 图层将被复制到 ArcGIS Data Store 中。托管 web 图层要求您创建一个特定类型的数据存储以保存这些数据。请参阅创建与 ArcGIS Server 配合使用的门户和 ArcGIS Data Store 站点中的表格，以确定每种类型的托管 web 图层所需要的数据存储类型。*

*托管影像图层数据将被复制到注册文件或云存储数据存储中，而不是通过 ArcGIS Data Store 创建的数据存储。

当您从门户中删除托管 web 图层时，服务即会从 ArcGIS Server 站点中删除，同时数据也会从数据存储中删除。

有关托管 web 图层的详细信息，请参阅 Portal for ArcGIS 用户帮助中的服务和门户项目和托管 web 图层。

托管服务器

托管服务器是与门户联合的 GIS Server 站点，同时也是被设置为托管发布到门户的 web 图层的服务器。

有关详细信息，请参阅关于将门户与 ArcGIS Server 配合使用。

位置追踪服务

此要素服务用于记录移动用户的位置。填充位置追踪服务的数据存储在时空大数据存储中。

关系数据存储

关系数据存储是门户的托管服务器用于存储托管要素图层数据的数据库。

时空大数据存储

时空大数据存储是具有全文搜索功能的分布式可扩展存储结构。它允许您存储同时包含空间和时间组件的数据。请参阅需要使用 ArcGIS Data Store 的应用程序和功能。

切片缓存数据存储

切片缓存数据存储是一组用于存储托管场景图层缓存的数据库。将针对门户中的每个托管场景图层创建一个数据库。
