管理许可

作为具有管理许可权限的管理员或成员，您可以使用 ArcGIS Enterprise 门户指定哪些成员可以访问尚未包含在其分配的用户类型中的许可应用程序。例如，您可以为以下获得许可的应用程序分配附加许可：ArcGIS Pro、Navigator for ArcGIS、AppStudio for ArcGIS Developer Edition、Drone2Map for ArcGIS、ArcGIS Business Analyst web 应用程序、ArcGIS Community Analyst、GeoPlanner for ArcGIS 和 ArcGIS Insights。您也可以为单独购买的应用程序包或用户类型扩展模块分配附加许可。您可以查看许可和用户类型详细信息，还可以每次为单个产品配置附加许可或一次为多个产品配置附加许可。

导入许可

可以通过从 My Esri 下载的单个 .json 文件来管理许可。如果已对组织的许可进行了更改，您会得到一个包含全部许可的新的 .json 文件，此文件将覆盖旧的许可文件。您可以通过门户或命令行导入新的许可文件。

在门户中导入许可

您可以在门户中导入新的许可文件。

  1. 验证您是否以组织管理员或者具备许可权限的成员身份登录。
  2. 单击站点顶部的组织，然后单击许可选项卡。
  3. 单击导入许可。
  4. 选择 .json 许可文件，然后单击打开。
  5. 单击导入。

新的许可将反映在您的许可页面，您可以在此将许可分配给成员。

注：

当导入新的许可文件时，门户中的所有用户类型和应用程序许可都将被新许可文件覆盖。例如，如果您的门户中的某些应用程序不在新的许可文件中，这些应用程序会在导入新的许可文件时自动移除。

使用命令行导入许可

可以使用 importlicense 命令行实用程序导入新的许可文件。

要使用 importlicense 命令行实用程序，请运行包含 -h 或 -help 的 importlicense.bat 脚本，如下所示：

<Portal for ArcGIS Installation Directory>\tools\importlicense\importlicense.bat -help

该实用程序的使用示例： importlicense.bat [-u <arg>] [-p <arg>] [-lf <arg>]

选项描述

-u; --username

用于登录到门户的具有管理权限的帐号用户名。

-p; --password

为 username 参数指定的帐户密码。

-lf; --licenseFile

门户许可文件的路径。

-h; --help

显示工具帮助并退出。

查看许可和用户类型详细信息

许可页面中包含有关贵组织的附加许可和用户类型的详细信息，其中涉及可帮助您在管理许可时做出决策的兼容性提示。

提示：

要对可用的且分配至成员的许可数和用户类型进行预览，请参阅组织的概述选项卡上的附加许可和成员卡片。

  1. 验证您是否以组织管理员或者具备许可权限的成员身份登录。
  2. 单击站点顶部的组织，然后单击许可选项卡。
  3. 单击用户类型选项卡，可查看组织中已分配的可用用户类型，以及每种用户类型所包含的许可、兼容的附加许可和兼容角色。
  4. 单击附加许可选项卡以查看在您的组织中可用的已分配附加许可，也可以查看与每个许可兼容的用户类型。
  5. 在 ArcGIS Pro 部分，单击许可活动可查看当前正在使用 ArcGIS Pro 的成员、他们上次使用该应用程序的时间以及成员是否已检出许可（如果在组织级别允许离线使用）。您可以按名称搜索成员，进而过滤列表。

    上次使用日期旁边的绿点表示该成员在过去 30 分钟内使用过 ArcGIS Pro。红点和断开连接标注表示该成员已检出 ArcGIS Pro 许可以便离线使用。

为一个产品配置附加许可

如果您要为一个特定产品配置附加许可，请完成以下工作流。使用此工作流，您可以一次为一个或更多成员分配/取消分配相同的产品。

  1. 验证您是否以组织管理员或者具备许可权限的成员身份登录。
  2. 单击站点顶部的组织，然后单击许可选项卡。
  3. 找到要更改的附加许可。如有必要，请按名称搜索许可。单击管理。
  4. 在显示的窗口中，找到要管理其许可的一个或多个成员。要搜索组织成员列表，请在搜索框中输入成员的姓名。使用过滤器缩小成员列表的搜索范围。例如，您可以过滤列表，以仅显示具有特定用户类型和角色的成员或特定群组中已分配 ArcGIS Pro 许可的成员。
  5. 如果您要分配许可，请单击每个成员旁边的管理。要将指定的附加许可分配给页面上显示的所有指定成员，请单击管理页面中的全部。

    如因许可与用户类型不兼容而无法将其分配给特定成员，您可以在成员选项卡上更新用户类型。

  6. 如果要取消分配许可，关闭成员旁边的许可分配。要从所有适用成员中取消分配指定附加许可，打开过滤器以仅显示已分配该许可的成员，然后单击管理页面中的全部。

    您无法取消已检出许可用户的 ArcGIS Pro 或 Drone2Map for ArcGIS 许可。

  7. 关闭窗口以返回到许可页面。

仅限 ArcGIS Pro

有时，您可能需要结束一个成员的活动 ArcGIS Pro 会话，以释放附加许可以供其他成员使用。

  1. 验证您是否以组织管理员或者具备许可权限的成员身份登录。
  2. 单击站点顶部的组织，然后单击许可选项卡。
  3. 在 ArcGIS Pro 部分，单击许可活动。
  4. 在显示的窗口中，搜索成员列表并单击要查看其活动详细信息的成员所在行。
  5. 单击结束会话。

    如果成员设置了用于 ArcGIS Pro 内工作站的别名，您可以利用该别名来帮助确认要关闭的会话。在 ArcGIS Pro 中，成员会在应用程序关闭时开始接收到会话结束的通知。该成员可最多同时进行三个 ArcGIS Pro 会话。

为多个产品配置附加许可

如果您要一次为多个产品配置附加许可，请完成以下工作流。您可以使用此工作流更改一个成员或一组选定成员的附加许可。

  1. 验证您是否以组织管理员或者具备许可权限的成员身份登录。
  2. 单击站点顶部的组织，然后单击成员选项卡。
  3. 按姓名或用户名搜索成员，然后应用过滤器以缩小成员列表范围。根据需要对列表进行排序，例如按姓名或最近登录日期进行排序。
  4. 单击成员所在行中的更多选项按钮 ，然后单击管理附加许可。
    提示：

    要一次更改多个成员，请选中要为其分配或取消分配相同附加许可的每个成员的名称旁边的框。一次最多可以选择 100 个成员。在搜索和过滤组织中的所有成员时，即使跨多个页面，也会维护选定集。如有必要，请在修改完初始搜索或过滤后单击已选定成员下拉菜单查看并修订选择。在成员列表上方，单击管理附加许可。

  5. 在出现的窗口中，选择或取消选择要更改的附加许可。
    • 如果选择了多个成员，则只能分配与所有选定成员的用户类型兼容的附加许可。如有必要，请单击选定成员下拉菜单，移除具有不兼容用户类型的任何成员。要查看与组织的用户类型兼容的内容，请转到许可选项卡并单击用户类型。
    • 如果成员已检出许可，您无法取消分配其 ArcGIS Pro 或 Drone2Map for ArcGIS 许可。
  6. 单击保存。

释放检出许可

如果用户检出可用于离线环境的许可，并且使用该许可的设备丢失或发生损坏，您需要使用 ArcGIS Portal Directory 来释放许可。这将释放未完成的许可，并且允许用户在在线环境中检出新许可或使用 ArcGIS Pro 或 Drone2Map。

  1. 打开 Web 浏览器并以组织管理员的身份登录到 ArcGIS Portal Directory。URL 地址格式为 https://portal.domain.com:7443/arcgis/portaladmin。
  2. 单击主页 > 许可 > 释放许可。
  3. 输入检出许可的用户的用户名（例如，jcho）。
  4. 单击释放许可。

有关管理许可的注意事项

  • 如果为成员分配的用户类型包括某一应用程序，则您无需为该成员分配该应用程序的附加许可。例如，分配了 GIS Professional 用户类型的成员已拥有 ArcGIS Pro 许可，因此您无需将 ArcGIS Pro 分配给该成员作为附加许可。如果您不确定成员现能够访问哪些应用程序，请查看其个人资料以了解其用户类型包括的许可以及已分配的附加许可。（转至组织页面，然后单击成员选项卡。查找您要查看其个人资料的成员，单击更多选项按钮 ，然后单击查看个人资料。）
  • 首先需要为您的组织提供 ArcGIS Pro 或其他特定产品（例如 Navigator for ArcGIS），然后您才能在许可选项卡上看到这些产品，并可以管理这些产品的附加许可。
  • 需要进一步控制离线使用 ArcGIS Pro 的组织可以阻止离线使用组织内的所有 ArcGIS Pro 许可。要针对您的组织执行此操作，具有许可权限的管理员或成员必须在许可选项卡的 ArcGIS Pro 部分中启用禁止成员离线使用 ArcGIS Pro 设置。启用此设置后，将允许已离线使用 ArcGIS Pro 的用户检回许可，但是未来任何离线使用 ArcGIS Pro 许可的尝试都将会产生错误。
  • 仅已配置应用程序的组织的管理员可使用此功能。自定义管理员角色必须具有管理成员许可权限才能使用此功能。
  • 每个产品在许可页面上都有一张相应的卡片，其中会显示可用于分配和兼容用户类型的附加许可数量。
  • 您的组织也许会超出可用的许可数量，使得可用数量为负数。如果您向成员分配许可但稍后获得的许可授权少于分配的数量，便会发生这样的情况，例如当您更改维护协议或正在使用试用许可时。组织中的任何许可过期后，您都必须取消分配过期的许可，这样就可以仅向您的组织成员分配有效许可。用户类型已过期的成员将无法登录。附加许可已过期的成员将无法使用许可。
