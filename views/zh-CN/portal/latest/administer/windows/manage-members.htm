管理成员

在本主题中

  1. 修改个人资料
  2. 更改用户类型
  3. 更改成员角色
  4. 重置密码
  5. 禁用成员
  6. 删除成员

在您将成员添加到组织后，可管理他们的帐户。管理成员包括修改个人资料、重置密码、禁用成员以及删除成员。 管理成员也包括更改其角色或用户类型。

注：

尽管组织中的匿名用户可以搜索并使用公开共享的项目，但他们没有组织账户，因此无法加入群组或创建并共享内容。如果您想授予这些成员附加权限，您可以向他们分配角色和用户类型，这将影响他们在门户中可执行的操作。

管理成员时可以执行的操作取决于您在组织中的权限。

修改个人资料

想要更新成员的描述性信息、缩略图、个人资料可见性、语言、单位或电子邮件地址时，请修改其个人资料。

  1. 验证您是否登录到组织以及是否具备更新成员帐户信息的权限。
  2. 单击站点顶部的组织，然后单击成员选项卡。
  3. 按姓名或用户名搜索成员，然后应用过滤器以缩小成员列表范围。根据需要对列表进行排序，例如按姓名或最近登录日期进行排序。
  4. 对于要修改其个人资料的成员，单击更多选项按钮 ，然后单击查看个人资料。
  5. 在成员的个人资料页面上，单击编辑个人资料。您现在可以更改成员的名称、电子邮件地址、描述、缩略图、个人资料可见性、语言、单位和其他个人资料信息。
  6. 单击保存，保存对成员个人资料的更改。

更改用户类型

用户类型可以确定成员可用的权限和应用程序。分配后，具有正确权限的用户可以更改用户类型。

  1. 验证您是否登录到组织以及是否具备更改成员角色、管理许可和更新成员帐户信息的管理权限。
  2. 单击站点顶部的组织，然后单击成员选项卡。
  3. 按姓名或用户名搜索成员，然后应用过滤器以缩小成员列表范围。根据需要对列表进行排序，例如按姓名或最近登录日期进行排序。
  4. 对于要修改其用户类型的成员，单击更多选项按钮 ，然后单击管理用户类型。
    提示：

    如需同时更改多个成员的用户类型，请选中想要更改的每个成员名字旁边的复选框。一次最多可以选择 100 个成员。在搜索和过滤组织中的所有成员时，即使跨多个页面，也会维护选定集。如有必要，请在修改完初始搜索或过滤后单击已选定成员下拉菜单查看并修订选择。在成员列表上方，单击管理用户类型

  5. 在随即显示的窗口中，更改用户类型，选择角色（如有必要）并单击保存。
    注：

    如果成员符合相应要求，可将其更改为功能更少的用户类型。例如，可在成员符合以下条件时将其从 Creator 更改为 Viewer：

    • 成员不拥有内容或群组。
    • 成员未被分配与其新用户类型不兼容的附加许可。
    • 成员不属于已启用项目更新功能的群组。
    如果不满足上述条件，则更改其用户类型之前，管理员必须重新分配该成员的内容和群组、撤销附加许可，然后从这些群组中移除该成员。

更改成员角色

角色可以定义分配给成员的权限集。分配角色后，可以由管理员和具有更改成员角色权限的成员对角色进行更改。分配的角色必须与分配给成员的用户类型兼容。

  1. 验证您是否已登录组织，以及是否有权限更改成员角色。
    注：

    将分配给成员的角色更改为默认的管理员，或从默认的管理员角色更改为其他角色时，都需要以默认管理员角色登录。

  2. 单击站点顶部的组织，然后单击成员选项卡。
  3. 按姓名或用户名搜索成员，然后应用过滤器以缩小成员列表范围。根据需要对列表进行排序，例如按姓名或最近登录日期进行排序。
  4. 在您想要更改角色的成员所在行中，单击角色下拉箭头并选择新角色。

重置密码

具有更新成员帐户信息权限的组织成员可以重置成员的密码。必须与成员共享系统提供的临时密码，以便其进行登录。成员通过临时密码成功登录后，系统将提示其更改密码。如果您在成员处于登录状态时重置其密码，则成员将立即登出。

注：

您无法重置企业登录帐户的密码。

  1. 验证您是否登录到组织以及是否具备更新成员帐户信息的权限。
  2. 单击站点顶部的组织，然后单击成员选项卡。
  3. 按姓名或用户名搜索成员，然后应用过滤器以缩小成员列表范围。根据需要对列表进行排序，例如按姓名或最近登录日期进行排序。
  4. 对于要重置其密码的成员，单击更多选项按钮 ，然后单击重置密码。
  5. 通知成员其新的临时密码。

成员通过临时密码成功登录后，系统会立刻提示其更改密码。

禁用成员

如果您是您所在组织的管理员，或具备相应的权限，则可以从组织中禁用成员。

禁用成员将阻止该成员使用组织资源。此操作在您将成员的项目移动给其他成员时非常有用。已禁用成员无法登录组织、使用组织资源、创建内容或管理站点。但他们仍是组织成员，并且计入组织用户人数。

如果该成员拥有内容或群组，您需要移交项目所有权给其他成员，然后才能删除该成员。移交所有项目之后，您可以从组织中删除禁用的成员。要了解有关如何在门户中管理成员的详细信息，请参阅管理门户访问。

您可以单独禁用组织成员，或选定的一组成员（一次最多 100 人）。

  1. 验证您是否登录到组织以及是否具备禁用成员的权限。
  2. 单击站点顶部的组织，然后单击成员选项卡。
  3. 按姓名或用户名搜索成员，然后应用过滤器以缩小成员列表范围。根据需要对列表进行排序，例如按姓名或最近登录日期进行排序。
  4. 选中想要禁用的每个成员名字旁边的复选框。在成员列表上方，单击禁用一位成员（如选定多个成员，则单击禁用多位成员）。

    仅当选定的成员至少登录到组织一次时，您才能够禁用成员。

    提示：

    一次最多可以选择 100 个成员。在搜索和过滤组织中的所有成员时，即使跨多个页面，也会维护选定集。如有必要，请在修改完初始搜索或过滤后单击所选成员下拉菜单查看并修改您的选择。

  5. 要启用一位或多位已禁用的成员，请选择成员，然后单击成员列表上方的启用成员。

删除成员

如果您是您所在组织的管理员，或具备相应的权限，则可在想要从组织中移除单个成员的帐户时删除该成员。也可以使用命令行实用程序批量删除成员。只有管理员可以删除其他管理员。

如果该成员拥有内容或群组，您需要移交项目所有权给其他成员，然后才能删除该成员。可针对每个成员单独执行此操作，也可使用命令行实用程序批量执行此操作。

如果已为该成员分配了许可，则必须在删除成员前撤销许可。对于某些 Esri 产品，例如 ArcGIS Pro 或 Drone2Map for ArcGIS，用户必须先检入许可，之后才能撤消这些许可。

如果直接从标识存储中删除成员，则该成员会保留在门户中。需要在门户中手动删除成员。如果该成员拥有内容或群组，则在删除该成员之前，您需要删除该成员的项目或移交项目所有权给其他成员。

存在于门户的内置标识存储中的成员帐户将被永久性删除且无法恢复。企业帐户将从门户取消注册，并保留在标识存储中。如有必要，可通过将帐户重新添加至门户恢复企业帐户。

提示：

如果要快速评估组织中哪些用户为门户成员，可使用 ListUsers 命令行实用程序生成可列出门户中所有成员的文本文件。使用此实用程序评估组织内的哪些用户可从门户中移除。输出文本文件也可用作对下述 DeleteUsers 命令行实用程序的输入。有关详细说明，请参阅列出成员。

  1. 验证您是否登录到组织以及是否具备删除成员的权限。
  2. 单击站点顶部的组织，然后单击成员选项卡。
  3. 按姓名或用户名搜索成员，然后应用过滤器以缩小成员列表范围。根据需要对列表进行排序，例如按姓名或最近登录日期进行排序。
  4. 对于要从组织中移除的成员，单击更多选项按钮 ，然后单击删除成员。
  5. 单击弹出窗口中的删除成员按钮确认要移除该成员。
    6. 注：
    只有管理员可以删除其他管理员。

使用命令行实用程序批量删除成员

如需一次从门户中移除大量成员，则比较适合使用命令行实用程序删除成员。您可以定期执行此操作，以使总成员数处于控制范围内。如果您最近升级了门户，并必须将指定用户的数量减至授权文件所允许的最大值，也可以执行此操作。有关此方案的详细信息，请参阅指定用户许可的强化。

您将使用随软件一起安装的 DeleteUsers命令行实用程序从门户中批量移除成员。此工具位于 <Portal for ArcGIS installation location>\tools\accountmanagement 目录中。此 工具以文本文件作为输入，且必须在安装门户的计算机上运行。

注：

工具只能由内置管理员帐户执行；您不能使用企业管理员帐户。您使用的内置帐户可以是初始管理员帐户，该帐户在配置门户或配置其他已添加至默认门户管理员角色的内置帐户时建立。如果您已删除初始管理员帐户，并且没有任何其他可用的内置管理员帐户，您将需要创建一个管理员帐户以便执行该工具。有关说明，请参阅向门户添加成员的内置门户帐户部分。

  1. 创建一份文本文件，其中需包含要从门户中删除的成员的用户名。在单独的一行上列出每个成员的用户名。示例如下：

    sarah
robert
james
qing

    注：

    您必须指定该成员的用户名。请勿使用成员的全称；否则，工具将忽略使用该成员全称的所有条目。另请注意，在文本文件中键入的用户名必须与其存储在门户中时的大小写一致。可通过运行 ListUsers 命令行实用程序或核对门户网站上的组织页面来查看用户名存储的大小写形式。

  2. 保存文本文件。
  3. 通过将文本文件指定为工具输入来运行 DeleteUsers 命令行工具，例如，DeleteUsers --file c:\scripts\memberstodelete.txt。

    提示：

    确保命令行选项和文件名称的大小写使用正确。
