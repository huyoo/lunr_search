共享的最佳做法

在本主题中

  1. 将正确的内容共享给所需的人
  2. 帮助人员找到您共享的内容
  3. 建立信任
  4. 保护共享的内容

web 已经对我们如何获取信息、如何与他人建立联系以及如何完成日常工作产生了影响。GIS 也因为网站而不断发生变化 - 现在我们更容易获取内容、创建各种地图和应用，以及分享结果。但是，web 也为寻找内容和验证权威性增添了新的挑战。如果您有一组引人入胜的地图、图层、场景、分析和图层。当您共享此内容时，如何确保人们可以找到他们需要的内容并相信此内容是可靠的并且仍可供其使用？

将正确的内容共享给所需的人

当您与其他人共享您的项目时，请考虑以下问题：

  • 哪些人有权访问此内容和采用何种格式？
  • 一个项目的受众会随着时间的推移而改变吗？
  • 该项目是否由其他项目组成？
  • 该项目是否为可编辑的托管要素图层？

哪些人有权访问此内容和采用何种格式？

共享项目之前，需要考虑您的共享权限。 然后需要考虑形式和您的受众。您想要共享地图或应用程序、在网站中嵌入地图或者将文件添加到群组吗？ 例如，如果要与可能没有 ArcGIS Pro 或 ArcMap 但能访问 Internet 的社区成员共享数据，可以将数据发布为 web 图层，然后创建故事地图并在网站中嵌入应用程序。

确定格式后，即可与合适的受众共享内容。有时您需要与所有人共享项目，但很少需要与所有人共享您的全部内容。如果与所有人共享所有项目，在最好的情况下，也会加大人们找到所需内容的难度。最坏的情况是，它可能会无意间向公众泄露敏感或专有信息。要将正确的内容提供给合适的人，那么与之共享内容的人应因项目而异。共享项目时，您具有以下几种选项：

  • 群组 - 对于许多项目，您的受众为特定群组人员。创建群组，在群组内共享相关项目，然后向群组中添加需要访问此内容的人员。 如果群组成员需要为群组贡献内容，您可将群组配置为同时允许该操作。
  • 组织 - 如果组织中的每位成员都需要使用您的项目，请将其与组织共享。请注意，如果该项目是可编辑的要素图层，则与组织共享该项目时，组织中的每位成员都将有编辑更改要素图层数据的权限。
  • 所有人（公众）-如果您需要允许除您的门户组织之外的人员访问您的项目，请将其与所有人共享。 请注意，与所有人共享可编辑的要素图层时意味着所有人都可以对图层中的数据进行编辑。

项目的受众会随着时间的推移而改变吗？

您可能需要在不同的时间与不同的受众共享您的项目。例如，在与公众共享应用程序（例如故事地图）之前，您需要创建该应用程序，您希望对其进行审查以给出反馈：首先由组织内部的若干同事进行审查以给出反馈，然后由组织内的更多受众进行审查以给出反馈。在这种情况下，建议执行以下步骤：

  1. 创建群组，该群组对群组成员可见。与群组共享您的应用程序项目以及应用程序中的所有图层、地图和场景，然后邀请审查人员（组织中）加入该群组。向审查人员发送应用程序的链接。
  2. 初始审核完成且应用程序处于较好状态后，即可与组织共享此应用程序，以便其他人提供反馈。
  3. 如果准备好与公众共享应用程序，请更改项目（和应用程序中使用的资源）以及群组的共享属性，以便公众对其进行访问。

该项目是否由其他项目组成？

某些项目包含其他源项目。应用程序包含地图、场景或群组等项目。地图和场景包含图层项目。确保在共享由其他项目组成的项目时，您与相同受众共享所有项目。

例如，如果要与所有人共享图库应用程序，则要确保群组、群组中的所有项目以及应用程序都是公共的。如果您与所有人共享应用程序但与您的组织共享该群组，则组织之外的任何人将不会在应用程序中看到任何内容。如果您与所有人共享群组和应用程序，但未与所有人共享全部项目，则图库中将缺少项目。

该项目是否为可编辑的托管要素图层？

仅与需要编辑数据的人共享可编辑图层。如果组织中有多个编辑者想要访问图层以进行更新，请创建一个群组，邀请编辑者加入该群组，然后在群组中共享可编辑的托管要素图层。如果在可编辑托管要素图层中也具有非编辑者需要查看的内容，请从托管要素图层创建托管要素图层视图，然后与更多受众（其他群组、组织或任何人）共享该视图。

有关控制访问可编辑数据的详细信息，请参阅在地图中使用图层的最佳做法。

帮助人员找到您共享的内容

确定内容所需采用的格式以及具有访问此内容权限的人员后，即可通过向项目添加相关且完整的详细信息，来帮助与您共享内容的人员找到该项目。要帮助人员找到您的内容，可以采用以下几种方式：

  • 改善搜索结果
  • 添加具有信息性的项目详细信息
  • 添加具有代表性且美观的缩略图

改善搜索结果

在搜索框中键入关键字时，ArcGIS Enterprise 将对关键字和每个项目的如下信息进行比较：

  • 标题
  • 标签
  • 摘要

因此，要改善您的搜索结果，请确保这些项目详细信息包含您预期用户将用来搜索特定数据类型的各种关键字。标签和摘要中所使用的关键字需保持一致，以提高用户查找类似内容的机会。

改善搜索结果的另一种方法是完善项目的状态。下一章节中将对此方法进行讨论。

添加具有信息性的项目详细信息

当用户获得搜索结果列表或在其浏览内容页面或图库时，他们需要确定哪些项目满足自己的需求。为了帮助用户找到正确的内容，请清晰明确地描述项目详细信息。请花一些时间来创建信息性标题、摘要、描述和标签，以便他人能够了解内容的用途。可以使用概述选项卡上的项目信息部分来帮助标识缺失或不完整的项目信息，并确定改进方法。项目信息状态越完善，项目在搜索结果列表中显示的位置就越靠前。

请确保其中包含了准确的访问路径、使用限制、制作者名单和空间范围。还可以为项目分配一个或多个类别，从而方便其他人查找您的项目，并了解其用途和可靠性。

最后，请确保对有关您的项目的任何评论做出响应。可以主动添加评论，来推广地图或应用程序中的特定功能。例如，可以建议用户查看您添加到地图的新的航空影像。

添加具有代表性且美观的缩略图

可以准确表示项目内容的美观的缩略图将有助于您的项目在搜索结果列表或图库中脱颖而出。请查看网站默认添加的图像。如果该图像看起来不够吸引人或不够精确，请将其替换成您自己的图像。

为获得最佳效果，请添加不小于 600 像素（宽）x 400 像素（高）或纵横比为 3:2 的 web 文件格式（如 PNG、JPEG 或 GIF）的图像。平移和缩放您希望在缩略图中显示的内容。根据图像文件的大小、分辨率以及自定义缩略图时的缩放程度，在保存该图像时可能需要重新采样和缩放。如果您添加 GIF 或 JPEG 格式的图像，则该图像在保存时将转换至 PNG 格式。

建立信任

您已经共享您的内容并尽可能地确保用户可以找到该内容并确定其是否满足他们的需求。现在，如何让用户相信这些数据值得信赖？ 以下是一些帮助用户信任您与他们共享的内容的方法：

  • 提供描述性配置文件
  • 识别权威内容

提供描述性配置文件

请利用个人资料来确立您在地理信息、地图设计、应用程序开发等方面的权威性。有用的描述性信息包括您的姓名、所属组织、联系信息、专业领域和感兴趣领域。添加您或您组织的图像，将使得您的描述更加个性化。

识别权威内容

在您的门户内，管理员和内容管理者（有权查看所有内容并对其进行更新的成员）可以标注内容权威性。当门户成员查找内容时，可以对权威内容进行过滤。

以下步骤描述了您可以在组织中实施的工作流程，以便在标注内容之前识别并确保内容具有权威性：

  1. 创建群组以将项目推荐为权威项目。群组成员可在该群组内共享其认为具有权威性的项目。
  2. 配置群组以允许成员向群组内贡献内容。
  3. 决定该群组对整个组织开放还是仅对部分成员开放，可通过两个群组设置组合来实现，即，谁可以查看此群组？ 和谁可以加入此群组?。
  4. 内容管理者或管理员需要检查群组中的项目以确定其是否真正具有权威性。如果是权威项目，管理者或管理员可以向项目添加权威徽标。如果不是，管理者或管理员可以将该项目从群组中移除。拒绝某个项目时，管理者或管理员可以在项目的详细信息页面上提供注释，以指出它不被视为权威项目的原因。这样可使所有者有机会提升项目的质量。
  5. 随着时间的推移，曾经具有权威性的项目可能不再权威。例如，如果要添加的数据更准确，则可以取代现有内容。为了让用户保持对权威徽章的信任，内容管理者应定期检查指定为权威的项目以确保其依然具有权威性。

保护共享的内容

您希望确保您共享的项目仍然可供需要的用户使用。为此，您可以启用项目的删除保护功能，以防止其被意外删除。您可能还希望限制使用安全服务项目；如果运行服务项目的服务器过载，其性能将会受到影响，使其对所有用户的有用性降低。

防止意外删除

与其他人共享项目后，这些用户会在自己的地图和应用程序中使用这些项目。如果删除共享项目，依赖于项目数据的地图和应用程序将无法访问数据，因此可能导致这些地图和应用程序无法使用。为防止您或管理员意外删除共享项目（例如，当在内容页面选择多个项目以进行删除时），请考虑对共享项目启用删除保护。

限制使用安全服务

如果想要与公众共享安全服务项目（例如，作为公共 web 应用程序的一部分），可将凭据和项目一同存储，以使公众无需登录即可访问您的应用程序。可能还需要限制使用以控制访问服务的次数和访问服务的用户。可以指定级别限制并进一步限制使用、指定可以访问服务的特定 referrer URL 或 IP（例如，贵门户的 URL）。

有关详细信息，请参阅 ArcGIS Server web 服务。
