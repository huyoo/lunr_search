创建影像图层

在本主题中

  1. 输入影像的支持栅格类型
  2. 支持的数据存储
  3. 输出影像图层项目的配置
  4. 创建影像图层

创建用于发布和共享影像服务的影像图层。影像图层可以使用支持的栅格数据类型进行创建、写入栅格存储，并作为影像服务发布。

要创建影像图层，门户管理员必须正确配置 ArcGIS Enterprise 以进行影像托管和栅格分析，同时您必须拥有创建内容和发布影像服务的权限。 如果创建影像图层的选项不可用，请联系门户管理员确定是否对您的门户进行配置以进行影像托管或栅格分析，并确认您是否具有足够的权限来发布影像服务。

输入影像的支持栅格类型

可以使用以下栅格数据类型创建影像图层：

  • ASTER
  • DMCII
  • DubaiSat-2
  • GeoEye-1
  • GF-1 PMS
  • GF-1 WFV
  • GF-2 PMS
  • GRIB
  • HDF
  • IKONOS
  • KOMPSAT-2
  • KOMPSAT-3
  • Landsat 1-5 MSS
  • Landsat 4-5 TM
  • Landsat 7 ETM+
  • Landsat 8
  • NetCDF
  • Pleiades-1
  • QuickBird
  • RapidEye
  • 栅格数据集
  • Sentinel-2
  • SkySat
  • SPOT 5
  • SPOT 6
  • SPOT 7
  • UAV/UAS
  • WordView-1
  • WordView-2
  • WordView-3
  • WordView-4
  • ZY3-SASMAC

用作输入的影像数据可以来自本地文件夹或数据存储。输出可以是托管影像图层或引用已注册数据源的影像图层。企业不对引用已注册数据源的影像图层进行管理。删除引用的影像图层时，数据不会从数据存储中删除。

支持的数据存储

支持通过以下工作流在栅格分析和影像托管服务器上注册的数据存储：

  • 在 ArcGIS Server Manager 中注册的数据存储（针对 10.7 版本）
  • 添加到门户的数据存储项目（自 10.7.1 版本起）

支持以下类型的数据存储：

  • 本地文件夹
  • 云存储
通过 ArcGIS Enterprise 门户将数据存储注册到栅格分析和影像托管服务器。可以使用门户的添加项目对话框来添加新数据存储。数据存储项目对话框可以提供有关所有联合服务器的信息。将数据存储注册到门户的影像托管服务器；如果门户仅具有栅格分析服务器而没有影像托管服务器，请将其注册到栅格分析服务器。如果您的门户网站同时具有栅格分析服务器和影像托管服务器，则需要同时注册到两者的服务器。如果您不了解将哪个 ArcGIS Image Server 站点配置为门户的栅格分析或影像托管服务器，请联系您的门户管理员。

注：

对于 10.7 版本：

  • 如果门户管理员为门户配置了影像托管服务器和栅格分析服务器，您必须使用 ArcGIS Server Manager 将数据存储注册到影像托管服务器和栅格分析服务器。
  • 请勿将这些文件夹或云存储配置为栅格存储。栅格存储可用于数据输出而非输入，因此不支持在此工作流中用作输入。如果文件夹或云存储也注册为栅格存储，则此文件夹或云存储将不会列出，并且无法用于此工作流。

注：

自 10.7.1 版本起：

要使用数据位置作为“创建影像图层”向导中的输入，您必须将数据位置作为数据存储项目注册到门户。

如果您的 ArcGIS Enterprise 配置具有不同的光栅分析影像服务器站点和影像托管服务器站点，请确保将数据存储项注册到影像托管服务器和栅格分析服务器。

输出影像图层项目的配置

将输入数据放入本地文件夹或已注册数据存储后，确定数据在影像图层中的传送方式。创建的影像图层类型取决于输入位置和输出配置。输出影像图层将为以下托管影像图层或 ArcGIS Server 影像服务：

  • 当输入数据来自已注册数据存储，并且创建包含所有输入影像集合的单个影像图层时，门户将处理影像并创建影像图层项目。
  • 当输入数据来自已注册数据存储并且创建包含单个文件中所有输入影像的镶嵌的单个影像图层时，门户会将数据复制到门户的栅格存储、处理图像并创建托管影像图层项目。
  • 为已注册数据存储中的每个影像创建一个影像图层时，数据将保留在已注册数据存储位置中，并且门户会在其中创建多个影像图层项目（多个 ArcGIS Server 影像服务）。
  • 输入数据位于本地文件夹时，门户会始终将数据复制到门户的栅格存储并创建托管影像图层。

创建包含指定的所有影像的单个托管影像图层，或为每个影像创建影像图层。

创建影像图层

如果希望将所有输入影像数据发布为一个影像图层，则可以将影像作为影像集合或镶嵌包括在内。如果具有多个输入影像，则可以选择创建一个影像图层作为影像镶嵌或影像集合，或者为每个影像输入创建一个影像图层。您还可以从单个影像输入创建一个影像图层。

如果希望针对单个影像属性查询生成的托管影像图层，请选择将影像分组到集合中。如果数据输入位置为本地文件夹，则门户会将数据复制至栅格存储并创建托管影像图层。

选择图层类型（步骤 1）

如果希望为每个输入影像文件创建影像图层，请按照下列步骤操作。如果数据输入位置为本地文件夹，则门户会将数据复制至栅格存储并创建托管影像图层。如果数据输入位置为已注册数据存储，则数据将保留在数据存储中。

  1. 登录您的门户组织，然后单击内容 > 我的内容。
  2. 单击创建 > 影像图层。
  3. 选择影像的传送方式：
    • 一个影像 - 根据单个影像创建一个影像图层
    • 影像镶嵌 - 通过镶嵌多个影像来创建一个影像图层
    • 影像集合 - 创建一个管理由多个影像所构成集合的影像图层
    • 多个影像图层 - 针对每个输入影像创建一个影像图层

定义影像（步骤 2）

定义输入栅格类型和影像图层的属性。然后从本地存储或数据存储中选择影像。

  1. 从上方列表的受支持的栅格数据类型的下拉列表中选择输入栅格类型。栅格数据类型必须与输入影像数据的栅格类型相匹配。
  2. 单击属性按钮设置所选栅格类型的属性。
    3. 每种栅格类型均包含特定于传感器和影像集合参数的唯一元数据。栅格类型元数据用于以一致的方式显示、校正和处理影像。影像属性参数可帮助您根据工程要求构建信息影像图层。栅格类型属性可定义为以下三类：
    • 常规
    • 处理模板
    • 波段映射
  3. 单击常规选项卡。

    源类型

    选择源类型以与影像的预期用途相匹配。

    • 通用 - 镶嵌数据集没有指定的数据类型。这是默认设置。
    • 高程 - 镶嵌数据集包含高程数据，例如数字高程模型 (DEM)、数字地形模型 (DTM) 或激光雷达数据。
    • 专题 - 专题数据具有离散值，例如土地覆被或类地图。您可以在辅助信息选项卡中提供相应的栅格属性表（DBF 版本 5）或色彩映射表文件 (.clr)。
      注：

      色彩映射表为包含空格分隔值的文本文件，扩展名为 .clr，文件中的每一行均包含四个整数值，以将像素值映射到颜色，这些值将按照像素值（整数）、红色 (0-255)、绿色 (0-255) 和蓝色 (0-255) 的顺序排列。以下是一些示例：

      0 255 0 0

      1 253 0 2

      ...

      255 0 220 253

    • 已处理 - 镶嵌数据集已经过地理配准和色彩平衡。

    重采样类型

    设置请求中要使用的重采样方法。方法包括以下内容：

    • 最邻近法（对于离散数据，建议使用此方法）。
    • 双线性插值法（对于连续数据，建议使用此方法）。
    • 三次卷积插值法（对于连续数据，建议使用此方法）。
    • 众数法（对于离散数据，建议使用此方法）。

    适用于所有输入。

    输出空间参考

    设置影像图层的坐标系。默认为 WGS 1984 Web 墨卡托（辅助球体）

  4. 单击处理选项卡。

    处理选项与上下文相关。选项通过在创建影像图层页面中选择的输入栅格类型值启用。例如，如果栅格类型包括多光谱和全色影像，则可以选择创建全色锐化影像图层。

    • 处理模板 - 确定处理影像数据的方式以创建影像图层。可用选项取决于输入栅格数据类型，并且可以包括提取波段、全色锐化、正射校正、拉伸等。
    • 对每个栅格项目应用默认拉伸 - 根据项目的统计数据，针对每个栅格项目将裁剪百分比拉伸应用为默认值。
    • 构建概视图以优化访问性能 - 将影像图层作为服务提供时，建议构建概视图来提高显示速度并降低 CPU 使用率。
    • 使用辐射度构建轮廓线（移除边界上的黑色像素）- 消除导致无缝镶嵌的空数据值边界像素。移除空数据边界像素会为每个影像项目生成更合适的统计数据，从而产生更好的影像增强和分析处理效果。
      • 选择此属性后，可以指定轮廓线设置。
    • 正射校正影像使用 - 使用影像服务或数据存储中的平均高程或 DEM。
    • 全色锐化类型 - 输入栅格类型包括重合的全色和多光谱影像数据时，可为包含影像图层的每个波段选择全色锐化方法和相关处理权重。
      • 选择此属性后，可以指定全色锐化设置。
    5. 所选属性设置可启用其他属性设置选项。例如，如果您选择源类型的专题，辅助选项卡会变为可用。
  5. 单击波段映射选项卡可指定用于处理影像图层的自定义波段组合。

    该表包含波段索引号、部分电磁频谱和波段波长间隔。

    对于卫星影像，波段映射选项卡包含只读波段映射索引以供参考。

  6. 针对影像图层单击选择输入影像，并选择以下任一输入位置：
    • 如果数据位于本地计算机，请单击自本地选项卡，并将本地影像文件拖动到虚线区域。也可使用浏览按钮浏览至本地影像文件，然后单击上传。系统将显示一个表，其中包含所有附带名称、大小和上传状态的所有文件。文件将上传至栅格分析服务器。
    • 如果数据位于已注册数据存储中，单击自数据存储选项卡将显示两个用于选择输入影像的选项：数据存储或包含云 URL 或数据存储的文件夹/文件路径的 .CSV 文件。请使用其中一个选项。
      • 要选择从中进行发布的已注册数据存储，请单击数据存储，展开包含要发布数据的数据存储，然后选中包含要发布数据的文件夹旁边的复选框。您可以从多个数据存储中选择多个文件夹。所选文件夹在所选影像列表中列出。要排除所选文件夹，请取消选中该复选框，或将鼠标悬停在所选影像列表中的文件夹名称上，然后单击删除按钮。
      • 要使用存储在 .CSV 文件中的已注册数据存储的列表，请单击包含云 URL 或数据存储的文件夹/文件路径的 .CSV 文件，然后将 .CSV 文件拖动到窗口中，或者浏览至 .CSV 文件并将其选中。
        注：

        如果使用数据存储重复创建影像图层工作流，则使用 .CSV 选项可以高效地指定输入影像。将数据存储和文件的路径记录在 .CSV 文件中。上传 .CSV 文件将填充路径（无需其他步骤），可以直接从这些路径进行发布。在使用此过程之前，请验证您是否有权访问这些路径，如果没有，则复制和发布作业将失败。

  7. 单击下一步。

    无需等待文件上传结束即可继续执行下一步操作。

设置项目详细信息（步骤 3）

定义影像图层的详细信息，并创建影像图层。

  1. 提供影像图层项目的详细信息。即使文件尚未上传结束，您也可以提供这些详细信息。
    1. 为标题选项提供文本，并为待创建的多个影像图层的标题定义命名约定。命名约定可以包括基本标题的前缀和后缀。
    2. 为托管影像图层定义标签。
    3. 可选择提供图层所包含内容的摘要。

      如果影像不相关，则对于多个图层可能很难实现此操作。例如，如果所有影像均位于相同位置，则可以提供适用于所有图层的摘要信息。但是，如果您发布的影像不相关，则可能需要进入各个图层的详细信息页面，以便在其发布后提供摘要。

    4. 选择用于保存托管影像图层项目的文件夹。
  2. 单击创建。

    系统将显示图层的创建进度，其中包括上传文件和创建影像图层项目。

  3. 可选择单击查看项目详细信息链接以显示图层的项目详细信息页面。

图层创建完成后，在内容页面中将显示图层的项目详细信息页面。
