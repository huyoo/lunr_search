什么是 ArcGIS Companion？

ArcGIS Companion 是适用于 iOS 和 Android 的原生移动应用程序，可提供对 ArcGIS 组织、内容和配置文件的访问。可以使用该应用程序来搜索、浏览以及管理 ArcGIS 内容、成员和群组，并与其他人共享和协作。借助 Companion，可以选择要打开项目（例如地图、场景、应用程序和图层）的位置。 例如，您可以在 Explorer for ArcGIS 中打开 Web 地图。 了解组织中最受欢迎的最新内容，并随时了解最新的 Esri 新闻、博客和推文。管理员可以执行常见任务，例如重置成员密码、启用和禁用成员、更新用户配置文件、向门户中添加成员、为成员分配群组以及更改角色。

Companion 适用于 ArcGIS 组织的所有成员以及属于 ArcGIS 开发者计划的开发人员。提供门户网站支持的所有语言。

要开始使用，请从以下位置在您的移动设备上安装 ArcGIS Companion：

  • Google Play 或 Amazon Appstore（适用于 Android）
  • App Store（适用于 iPad 和 iPhone）

在 ArcGIS Enterprise 中，什么是强度较弱的密码？

可能不会接受强度较弱的密码。如果密码为常用密码（例如 password1）或者包含重复字符或序列字符（例如，aaaabbbb 或 1234abcd），则密码将视为强度较弱。

谁可以使用分布式协作？

分布式协作适用于 10.5 及更高版本的所有 ArcGIS Enterprise 部署。您的组织必须至少具有 ArcGIS Enterprise基础部署才能使用协作。

我的组织为什么只能使用 HTTPS？

加密敏感信息是使用 HTTPS 的主要原因。HTTPS 使用传输层安全性 (TLS) 或安全套接字层 (SSL) 加密协议来确保网络通信的安全性。ArcGIS 平台使用 TLS，这是一种比 SSL 更加先进和安全的加密协议。当您在 URL 地址中使用 HTTPS 通过 Internet 发送信息时，只有预期的接收者能够理解该信息。此加密非常重要，因为您通过 Internet 发送的信息通常在许多计算机之间传送，然后才能到达目标服务器。如果未使用有效的 TLS 或 SSL 证书对密码之类的敏感信息进行加密，您与服务器之间的任何计算机都能看到此信息。在使用有效的 TLS 或 SSL 证书的情况下，只有发送此信息的服务器能够读取此信息，任何人都无法读取此信息。此做法可保护此信息免遭身份盗用等恶意活动的影响。

默认情况下，ArcGIS Enterprise 为所有通信强制使用 HTTPS，但是您可以通过安全设置来调整安全协议以满足组织的需求。 还建议您在所有本地服务上启用 TLS。当您将图层添加到地图或者将图层作为项目添加时，建议您使用 HTTPS URL。HTTPS 不仅有助于保护数据，还可以减轻浏览器的混合内容问题。如今，许多网站只能通过 HTTPS 进行访问。HTTPS 可确保您在浏览器与 ArcGIS Enterprise 之间的数据和所有其他通信都经过加密。

有关 TLS、SSL、HTTPS 和 Internet 安全性的详细信息，请访问 Trust Center。Esri 创建的 Trust Center 作为与 ArcGIS 平台有关的安全性、隐私性和合规性信息的资源。Trust Center 提供与组织的产品安全、安全警报、安全合规性、安全性最佳做法等内容有关的信息。

如何使用嵌入式凭据将安全服务的数据源 URL 协议更改为 HTTPS？

如果您拥有使用嵌入式凭据管理安全服务（例如 ArcGIS Server 要素图层或者打印或地理编码服务）的权限，则可以在安全服务项目的项目页面的设置选项卡上将数据源的 URL 协议从 http 替换为 https。建议仅使用 HTTPS 访问所有服务，以在通过 Internet 传输时加密服务中的信息。有关详细信息，请参阅我的组织为什么只能使用 HTTPS？

将可配置应用程序标注为成熟意味着什么？

当一个可配置应用程序能够被经过改进的同等应用程序替换时，该应用程序将更改为成熟状态。当可配置应用程序进入成熟状态时，通过可配置应用程序创建的现有应用程序可继续使用并且用户仍然可以对其进行访问，同时您仍然可以编辑和更新您的应用程序。您还可以通过成熟的可配置应用程序创建新应用程序；但是，不建议这样做，原因在于 Esri 不会添加新要素或修复成熟应用程序中的问题。例如，成熟应用程序不会更新以支持浏览器更改或 ArcGIS 中影响应用程序功能的更改。不过，如果您想要根据成熟的可配置应用程序创建应用程序，则可以通过单击成熟应用程序项目页面上的创建 Web 应用程序按钮来完成此操作。

成员何时可以查看其组织的管理员联系信息？

您可以选择列为门户联系方的管理员。添加后，指定管理员的详细信息将在组织页面上对所有非管理员组织成员可见。

地图中存储的要素与要素服务图层中存储的要素有何区别？

地图中存储的要素来自于创建地图注释图层的过程或来自于上传的文件，只能由地图创建者进行编辑。要素服务图层中的要素来自 web 服务，可以在服务所有者允许的情况下由用户编辑。

向地图中添加要素的最佳方法是什么？

视情况而定。如果您拥有大量数据，可采用的有效方法是创建要素服务并将其作为图层添加至地图。 如果希望其他人编辑要素及其属性信息，也应添加要素图层。默认情况下，任何查看您地图的用户均可编辑您的要素。

如果只想添加少量要素，应使用 Map Viewer 添加地图注释图层。 Map Viewer 提供了多种模板供您选取形状和符号。但是，添加大量要素时此方法并不实用，因为必须在 Map Viewer 中创建每个要素 - 例如，您不能上传含有预定义要素的文件。这些图层为只读图层，因此其他人无法更改要素或编辑相关的属性信息。

如果要素存储在带分隔符的文本文件（.txt 或 .csv）或 GPS 交换格式文件 (.gpx) 中，则可将它们导入您的地图中。添加存储在计算机上的文件中的要素时，这种方法非常方便。将这些要素添加到地图后，可以更改符号系统并配置弹出窗口。

如果您的文件包含许多要素，应该将其置于 web 服务器上并使用从 Web 添加图层通过 URL 引用此文件，而不要直接将其导入地图中。

如何将单个要素图层另存为项目？

要通过多图层要素图层保存单个要素图层，请完成以下步骤：

  1. 打开多图层要素图层，然后转到可视化选项卡。
  2. 从图层下拉菜单中选择要保存的图层。
  3. 单击另存为新图层在我的内容中将图层的副本另存为新项目。
  4. 输入标题、标签和摘要，或者选择其他文件夹以保存图层。
  5. 选择使用当前图层创建新项目，然后单击保存。

如何辨别地图是否已启用时间？

如果在地图底部看到时间滑块，则表明该地图包含一个或多个时态图层，您可以通过播放地图来查看信息如何随时间变化。

“保存”与“另存为”之间有何区别？

保存操作会更新您拥有的现有地图。另存为操作则会创建地图副本，同时会显示原始的保存地图 窗口，您可以在其中更新标题、标签等。您可以为不属于您的任何地图保存副本，但不能更新不属于您的现有地图。

使用 ArcGIS API 构建地图与使用 Map Viewer 构建地图有何区别？

可将 Map Viewer 视为这样一个“画布”：您可在其中轻松聚合自己感兴趣的各种图层。创建完地图之后，您可与其他门户的用户共享此地图或将其嵌入应用程序。可以使用 ArcGIS API 编写此应用程序，如果您的编程经验不足，可以使用预配置的应用程序，例如门户的 Web 应用程序模板。

ArcGIS API 也可用于从头开始构建聚合应用程序，而无需启动 Map Viewer。这需要由开发人员进行更多的代码和图层管理。

如何在组织的底图库中使图层作为底图？

默认情况下，Map Viewer 中的底图库会使用 ArcGIS Online 中的底图，但是，您的门户管理员可以将 Map Viewer 配置为使用其他群组来填充底图库。如果您的门户使用自定义底图库，门户管理员允许其他用户向其贡献内容。如果您是具有内容创建和共享权限的角色的成员，则可以创建底图并将其添加到底图库中。要使图层可供自定义底图库使用，请按照以下步骤执行操作：

  1. 请联系您的门户管理员，确认底图库使用允许其成员贡献内容的自定义群组。如果是这样，加入该群组或让您的管理员将您添加到该群组中。
  2. 登录您的组织并打开 Map Viewer。
  3. 将图层作为底图添加到地图。您可以添加 ArcGIS Server web 服务、OGC 图层或切片图层。必须将该图层共享到底图库群组和您的组织。
  4. 保存地图。为地图指定一个简洁的名称，以便您的同类门户成员了解地图包含的内容。
  5. 单击共享，然后将地图共享给您的底图库群组和组织。

什么是场景？

场景是一种包括图层、样式和配置集合的 3D 制图表达，允许通过 Internet 与其他用户共享符号化地理空间内容。您可以在 ArcGIS Enterprise 门户中通过 ArcGIS Pro 或 Scene Viewer 创建 3D 场景。

什么是 Scene Viewer？

Scene Viewer 是内置在 ArcGIS Enterprise 门户中用于创建 3D 场景并与 3D 场景进行交互的应用程序。Scene Viewer 可以与支持 WebGL（内置于大多数现代化浏览器中的网络技术标准，用于渲染 3D 图形）的桌面 Web 浏览器结合使用。

如何打开场景？

您可以使用 Scene Viewer 在门户网站中查看场景，或登录您的组织并创建场景。您也可以在 ArcGIS Pro 中打开场景。

如何创建场景？

您可以登录组织以使用 Scene Viewer 在门户中创建场景，也可以使用 ArcGIS Pro 创建和共享场景。

场景和地图之间有何区别？

场景与地图相似的地方在于它们都是可在浏览器中查看的数据图层的聚合。场景与地图的不同之处在于场景可以显示 3D 环境中的数据，并且允许任何方向和角度的导航。

需要使用 ArcGIS Pro 创建场景吗？

否。您可以使用 Scene Viewer 通过聚合共享给组织的图层来创建场景。

如何在 Scene Viewer 中使用 2D 数据？

可以将 2D 数据转换到 3D 环境中以更好地理解数据。例如，可以通过应用大小和高度从 2D 符号创建 3D 符号。可以缩放和旋转表面以从不同角度查看数据。还可以为重叠 2D 图层添加高程，从而更方便地查看各图层中的数据。

Scene Viewer 和 CityEngine Web Viewer 之间有哪些区别？

Scene Viewer 和 CityEngine Web Viewer 是 ArcGIS Enterprise 中两个不同的应用程序，均具有独特的功能和用途。

在 Scene Viewer 中，您可以执行以下操作：

  • 查看在 Scene Viewer 或 ArcGIS Pro 中创建的场景。Scene Viewer 不支持 CityEngine web 场景。
  • 创作场景：例如，可以添加和移除图层、修改符号系统或者捕获幻灯片。
  • 显示门户图层的集合，例如场景图层、要素图层、影像图层或切片图层。
  • 查看在球形地球视图（全球场景）或平面视图（局部场景）中使用底图在全球范围内进行渲染的场景。
  • 逐步执行数据加载和影像图形渲染时导航场景。

在 CityEngine Web Viewer 中，可执行以下操作：

  • 显示从 CityEngine 或 ArcScene 导出的静态 CityEngine web 场景 (.3ws)。CityEngine Web Viewer 不支持场景。
  • 查看使用笛卡尔坐标系在较小的局部范围内进行渲染的 CityEngine web 场景。
  • 评论和比较方案：例如，可以使用卷帘工具来查看两个方案之间的区别。
  • CityEngine Web Viewer 加载 CityEngine web 场景作为打开时的初始下载，并且之后不会加载任何流数据。
  • CityEngine Web Viewer 是一个查看工具，不支持创作，例如添加底图、更改符号系统或者配置图层。

可以在要素图层中共享 3D 几何吗？

可以。可以共享 3D 点、线和面。

可以在场景中共享自己的地形模型吗？

会。您可以通过 ArcGIS Server 共享自定义地形。使用 ArcGIS Desktop 10.3 或更高版本创建缓存高程影像服务并通过 ArcGIS Server 共享服务。您也可以使用 ArcGIS Pro 通过高程数据构建切片包以创建 ArcGIS Enterprise 上的托管高程图层。 您可以添加这些高程服务和图层到场景。

我的 Story Map 应用程序为什么不显示我嵌入其中的应用程序或 web 内容？

如果您的门户被配置为仅通过 HTTPS 进行通信，则您在门户中创建的故事地图需始终通过 HTTPS 进行访问。如果您嵌入的应用程序或 web 内容是通过 HTTP 进行访问，则嵌入的项目不会显示在 Story Map 中。

哪些 ArcGIS 应用程序支持底图库中的矢量底图？

矢量底图可以在 Map Viewer、Scene Viewer、ArcGIS Pro、Collector for ArcGIS、Workforce for ArcGIS、Explorer for ArcGIS、使用 Web AppBuilder 创建的应用程序、 Ortho Maker、 以及包含底图库的可配置应用程序中使用。要确定矢量底图在特定的 ArcGIS 应用程序中是否可用，请参阅应用程序文档。

有关矢量底图及其与栅格底图比较的详细信息，请参阅切片图层。

我可以上传多大的文件？

您可以通过 web 浏览器将最大为 200 GB 的文件上传到我的内容。

提示：

要发布大型切片包，建议您使用共享包地理处理工具，以便使用多核进行处理。

可以共享哪些内容？

您可以共享支持类型的地图、应用程序、图层、工具和文件。

能否共享地图或应用程序的 URL？

可以。可通过 URL 直接访问内容项目、搜索结果和群组。

如果删除用于发布托管图层的源项目，会发生何种变化？

如果删除了用于发布托管图层的源项目，图层将继续按预期进行绘制。但是，需要访问源数据的部分功能可能不再按照预期工作。图层间的特定功能可能有所不同。受影响的功能包括：

  • 如果删除服务定义文件，则 ArcMap 中的覆盖现有服务选项可能不再按照预期工作。
  • 如果删除 shapefile、文件地理数据库或 CSV 文件，则托管要素图层项目页面上的覆盖选项将不再可用。
  • 如果删除切片包 (.tpk)，自其中发布的托管切片图层将不再可离线获取。

用于发布托管图层的源项目为托管图层本身时，无法删除用作源项目的托管图层，除非删除自其中发布的所有托管图层。

我如何更新托管要素图层中的数据？

如果在托管要素图层上启用编辑，那么可在 Map Viewer 中编辑要素及其显示。如果将图层保存在 web 地图中并在应用程序中使用地图，则可以通过应用程序编辑托管要素图层中的要素。

要更新文件地理数据库、shapefile、GeoJSON 或包含经纬度信息的 CSV 文件或 Microsoft Excel 文件中发布的托管要素图层中的数据，可使用托管要素图层项目页面中的覆盖选项，将数据替换为更新的源文件中的数据。

如果您从 ArcMap 或 ArcGIS Pro 中发布了托管要素图层，则将覆盖相同应用程序中的相同地图中的服务。

网站环境内的 Web 地图和 Web 应用程序有何区别？

Web 地图是一组覆盖特定地理区域的信息图层和弹出窗口。地图是通用的，因此可以在浏览器、移动设备或桌面应用程序中查看地图。可使用特定于应用程序的工具来更改地图范围、查找地点以及查看有关位置的详细数据。

可通过定义感兴趣区域、选择底图、添加数据图层和配置弹出窗口来创建 web 地图。您可以保存地图并与所有人或您所在的特定群组共享地图。

Web 应用程序是将地图、数据和工具组合在一起以实现目标用途的网站，例如用于查找选举的投票站。应用程序可以像嵌入在博客中的可导航地图图像一样简单，也可以像 GPS 导航可视化一样复杂。

Web 应用程序可以基于模板（包含在门户中），或使用 ArcGIS API 从头开始开发。这两种类型的应用程序都可以插入到 ArcGIS 地图中。通常，应用程序基于地图中的信息构造，并使用特定的配置和自定义内容进行补充。应用程序可作为内容的一部分在系统中托管，也可以独立管理并注册到系统中。

当以 ArcGIS Online 底图为背景进行查看时，如何控制重新投影托管要素图层的方式？

当您向 ArcGIS 应用程序（例如 Map Viewer、Scene Viewer、ArcGIS Pro 和 ArcMap）添加图层时，应用程序会对图层进行动态重新投影以便数据与底图对齐。ArcGIS 根据底图的坐标系选择适当的默认变换以用于动态重新投影。有时，默认的选择达不到您所要求的精度，看起来像要素朝某个方向稍微移动了一些。或者默认选择可能与您使用的变换不一致，而您使用的变换是您的组织 GIS 最佳实践的一部分。

当您从 ArcMap 或 ArcGIS Pro 发布托管要素图层，并且您也知道用户将使用包含 ArcGIS Online 底图的地图中的托管要素图层时，您可以执行以下操作之一：

  • 选项 1：保留数据的投影，但定义一个可以在执行托管要素图层动态重新投影时门户能够使用的地理变换。

    这样，您的源数据和托管的要素图层数据将保留在相同的投影中。您需要定义当需要进行动态重新投影时应使用的地理变换。

    以下步骤总结了保留数据原始投影的方式，并定义了在 ArcGIS Online 底图中查看图层时控制图层重新投影方式的地理变换。

    1. 在 ArcMap 或 ArcGIS Pro 中，打开要作为托管要素图层发布的地图。
    2. 打开地图的数据框属性，然后设置数据框坐标系以匹配地图中源数据的坐标系并应用更改。
      • 对于 ArcMap，请参阅指定坐标系。
      • 对于 ArcGIS Pro，请参阅指定坐标系。
    3. 定义地理变换。这涉及到定义变换将要投影的内容以及用于执行变换的投影。选择投影到 WGS 1984。

    当您发布时，托管的要素图层将存储您设置的地理变换，并在重新投影图层时以 ArcGIS Online 底图为背景进行查看。

    注：

    如果 ArcGIS Enterprise 不支持您定义的图层变换，则其可以使用默认变换，或者无法重新投影。ArcGIS Enterprise 支持在 ArcMap 和 ArcGIS Pro 中默认安装的变换。

  • 选项 2：配置地图，以便在将要素类数据复制到关系数据存储时能将其重新投影到 WGS 1984 Web Mercator（辅助球体）中。

    使用此选项时，托管要素图层数据将存储在使用与 ArcGIS Online 底图相同坐标系的关系数据存储中；因此在以这些底图为背景查看时不再需要动态重新投影。 但是，这意味着托管的要素图层数据和数据源中的数据现在处于两个不同的投影中。

    以下步骤总结了如何配置地图才能在发布过程中重新投影托管要素图层数据，以便将其存储在 WGS 1984 Web Mercator（辅助球体）中：

    1. 在 ArcMap 或 ArcGIS Pro 中，打开要作为托管要素图层发布的地图。
    2. 将数据框坐标系更改为 WGS 1984 Web 墨卡托（辅助球体）。这就是将存储托管要素图层数据的投影。
      • 对于 ArcMap，请参阅指定坐标系。
      • 对于 ArcGIS Pro，请参阅指定坐标系。
    3. 定义地理转换。选择地图中源数据使用的投影。

    发布时，复制到关系数据存储的数据存储在 WGS 1984 Web Mercator（辅助球体）中。

什么是 ArcGIS Living Atlas of the World？

ArcGIS Living Atlas of the World 是 Esri 提供的一种权威的、不断更新的即用型全球地理信息集合。其中包括影像、底图、人口统计和生活方式、景观、边界和地点、交通、地球观测数据、城市系统、海洋，以及可与您自己的数据相结合来创建地图、场景和应用程序并执行分析的历史地图。

什么是订阅者内容？

订阅者内容是即用型地图图层、分析工具和 Esri 发布的需要 ArcGIS Online 组织订阅帐户才能访问的服务的集合。这包括来自 Esri 的图层，例如 NAIP 影像、景观分析图层和历史地图。订阅者内容不消耗 ArcGIS Online 配额。

什么是高级内容？

高级内容是 ArcGIS Online 订阅者内容的一种类型。高级内容是即用型地图图层、分析工具、应用程序和 Esri 发布的通过 ArcGIS Online 组织帐户才能访问的服务集合，使用时需要消耗配额。Esri 提供的高级内容图层包括人口统计和生活方式地图，以及用于地理编码、Geoenrichment、网络分析和空间分析的工具。

我如何授权我的同事修改我共享给他们的内容？

通过将项目共享给一个启用了项目更新功能的群组，您可以允许组织内的其他人更新您的地图、应用程序、图层和文件以及他们的项目详细信息。当您将项目共享给一个启用了此功能的群组时，即表示您允许群组成员更新共享给该群组的任何项目。其中包括修改项目详细信息或者更新有关内容。

在许多情况下，允许组织的其他成员更新您共享的内容会非常实用。例如，此功能可便于轮班员工团队成员之间分担工作，如更新重要 web 地图（添加或移除图层）、更改符号、更新地图描述等。另外一种常见情景是使编辑团队成员均能够编辑公共可见的托管要素图层，而无需针对所有人启用图层编辑功能。

要允许其他人更新您共享的项目，请执行以下操作：

  1. 创建一个启用了项目更新功能的群组，具体步骤如下：
    1. 确保具备创建带有更新功能的群组的权限。
    2. 创建新群组。对于成员可以更新群组中的哪些项目? 设置，请选择所有项目（群组成员资格受限于组织）。
      注：

      当您创建新群组，且群组中的成员资格仅对受到邀请或其请求已获批准的用户开放时，此选项可用。

    3. 将您自己与要合作的同事添加到群组。
      注：

      您只能邀请具有创建、更新和删除内容权限的组织成员。

  2. 使用访问和更新功能选项与群组共享项目。

    您仍然是项目的所有者，但其他群组成员可以更新项目，这包括更改项目详细信息和更新内容。

注：

只有项目所有者（或管理员）才能对项目执行以下操作（不是所有操作都适用于所有项目类型）：删除、共享、移动、更改所有者、更改删除保护、发布、注册应用程序、覆盖托管要素图层中的数据以及管理托管切片图层中的切片。
