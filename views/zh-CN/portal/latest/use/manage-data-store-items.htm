管理数据存储项目

在本主题中

  1. 从 ArcGIS Server 站点中添加或移除
  2. 删除数据存储
  3. 更新数据库密码
  4. 更新云数据存储项目的信息

作为数据存储项目的所有者或门户管理员，您可以管理数据存储设置以帮助门户成员发现并使用数据存储。包括更新项目的概述信息、共享数据存储、更改项目的状态、启用项目的删除保护，以及定义数据存储中的数据要应用到的空间范围。

您还可以更改对数据存储具有访问权限的联合服务器，或者删除数据存储项目（如果不再需要）。

如果数据存储项目访问数据库，则用于访问数据库的密码将可能会定期更改。如果密码更改，则必须更新数据存储项目所使用的密码。

您可能还需要修正云数据存储项目的信息。

从 ArcGIS Server 站点中添加或移除

创建数据存储项目后，可以允许多个联合 ArcGIS Server 站点访问数据存储。例如，如果门户管理员将其他 GIS Server 和 Image Server 站点与门户联合，则可以在数据存储项目的设置选项卡中将数据存储添加到新站点。

如果您不希望在将数据存储项目注册到的一个或多个 ArcGIS Server 站点中使用该数据存储项目，则可以将其移除。例如，当您添加了数据存储项目后，可能已将数据存储注册到 4 个联合 ArcGIS Server 站点，但是您未意识到其中一个站点并非 ArcGIS GIS Server 站点。可以从错误站点中移除数据存储，这将取消注册此站点的数据库连接。

注：

如果站点上没有正在运行的批量发布的服务，则只能从数据存储中移除 ArcGIS Server 站点。对于其他所有 web 图层，移除 ArcGIS Server 站点将会断开服务与数据之间的连接，这可能导致以后出现问题。例如，如果您删除了数据库数据存储，并在以后更改了访问数据库的帐户的密码，则使用已删除的数据存储的服务将停止运行。这是因为服务将使用旧密码，因此将无法再访问数据。由于您删除了数据存储项目，因此无法更新密码。

请按照以下步骤将 ArcGIS Server 站点添加到数据存储项目或从数据存储项目移除站点：

  1. 登录到门户并打开数据存储项目的详细信息页面。
  2. 单击设置选项卡。
  3. 向下滚动到服务器部分，然后单击编辑服务器。

    您可能需要等待验证 ArcGIS Server 站点状态后才能进行编辑。

  4. 选中或取消选中每个联合服务器旁边的复选框可以更改站点对数据存储项目的访问权限。
  5. 单击保存以应用更改。

删除数据存储

在极少数情况下，您可能决定不再需要门户中的特定数据存储项目。例如，您可能已注册一个数据存储，用于为短期工程发布图层。完成工程后，您需要删除图层和数据存储项目。

删除数据存储项目不会删除数据或数据源位置。但是，由于数据存储项目与从其发布的服务和图层之间存在依赖关系，因此，在删除从数据存储中发布的所有 web 图层并从数据存储中移除所有联合服务器之前，将无法删除数据存储项目。

要删除数据存储项目，请执行以下操作：

  1. 对于数据库数据存储，必须删除所有批量发布的 web 图层，这些图层引用了数据存储项目中的数据。
  2. 从数据存储中移除所有联合服务器。
  3. 从门户中删除数据存储项目。

更新数据库密码

在大多数组织中，必须定期重置密码以维护安全性。如果数据存储项目用于访问数据库的帐户的密码更改，则必须在门户中更新密码。更新后的密码将发送到已添加到数据存储项目的各个联合服务器。

要更新数据库数据存储项目的密码，请按照以下步骤进行操作：

  1. 打开 ArcGIS Pro 或 ArcCatalog，然后打开用于创建数据存储项目的数据库连接文件 (.sde) 的数据库连接属性。
  2. 更新数据库连接文件中的密码并保存更改。
  3. 从相应计算机登录到门户，该计算机具有在上述步骤中更改的 .sde 文件的访问权限。
  4. 打开数据存储项目的详细信息页面，然后单击设置选项卡。
  5. 向下滚动到数据库部分，然后单击更新密码。
  6. 对于需要更新的连接，单击导入包含已更新密码的连接文件。
  7. 选择在步骤 1 和 2 中更新的 .sde 文件，然后单击验证以确认联合服务器是否能够访问数据存储。
  8. 单击保存以应用更改。

更新云数据存储项目的信息

当您创建云数据存储时，ArcGIS 将使用您提供的凭据（例如 IAM 角色或访问密钥）对您的连接进行身份验证。将无法确定您指定的位置中是否存在云存储位置。因此，可能您创建了云数据存储项目并错误地输入了存储段名称或选择了错误的地区。当您尝试从数据存储发布影像图层时，会收到一条消息，提示该数据存储不可访问。要更正此问题，您可以更新运数据存储项目的连接信息。

您无法更改云提供商，但是可以更改其他设置。但是，您应该仅更新其他设置，以更正问题或更新凭据。如果您将设置更改为已经发布图层的云存储的另一个位置（例如，您制定了一个其他 Blob 存储位置）且数据不在所述的另一个位置，您的图层可能停止运行。

请注意，您也可以更新文件夹数据存储的信息。但是，与更改云存储的位置类似，如果更改文件夹数据存储的文件路径并且不存在现有图层使用的影像数据，则现有图层可能会遇到问题。

要更正云数据存储项目的信息，请按照以下步骤操作：

  1. 作为数据存储所有者登录到门户。
  2. 打开云数据存储项目的详细信息页面，然后单击设置选项卡。
  3. 向下滚动到数据存储（云）部分，然后单击更新密码。您可以为云存储编辑以下内容：
    • Amazon Web Services 和 Alibaba - 您可以编辑访问密钥、保密密钥、密码、地区、容器名称和文件夹。
    • Microsoft Azure - 您可以编辑帐户名称、帐户密钥、环境、容器名称和文件夹。

    如果您已从此云数据存储中发布影像服务，则请勿更改位置（例如，地区、环境、容器名称或文件夹）。

  4. 单击验证以确认联合服务器可以访问数据存储。
  5. 单击保存以应用更改。
