元数据

在本主题中

  1. 查看元数据
  2. 编辑元数据
  3. 元数据样式、标准和专用标准
  4. 常见问题解答
  5. 元数据注意事项

元数据为描述项目的信息。在 ArcGIS Enterprise 门户中，在项目页面内创建、编辑和查看项目的元数据。项目详细信息包括标题、类型、来源、作者、上次修改日期、缩略图与标签。它也包括额外的信息，例如摘要和描述、项目准确性与项目新近性、以及与使用和共享项目相关的限制。此信息将帮助他人发现并验证项目的实用性。在 ArcGIS 中，元数据与所描述的项目一起保存。元数据与项目一起复制、移动和删除。

默认管理员角色的成员可以启用元数据，允许门户成员利用内置元数据编辑器来包括所有项目类型的其他基于标准的元数据。门户中的所有项目（包括 web 地图、web 场景和 web 应用程序）都可以包含元数据。门户管理员选择元数据样式，用以配置门户编辑和查看元数据的体验。除确定可以进行查看和编辑的信息外，元数据样式还可以标识您所遵循的元数据标准和用于验证项目元数据是否符合标准的方案。无论应用何种样式，元数据总是以 ArcGIS 元数据格式进行存储。这意味着即使门户管理员切换元数据样式，元数据也不会丢失。

内置元数据查看器可用于查看与项目关联的元数据。如果门户管理员在门户中启用了元数据，则同一查看器可允许项目所有者和管理员编辑元数据。 为了让您能够访问查看器，以下情况下，元数据按钮将出现在项目页面上：

  • 如果门户管理员已启用元数据，则项目所有者和管理员可以使用该元数据按钮以添加和编辑元数据。
  • 项目包含元数据后，任何有权访问该项目的人都可以使用该元数据按钮以查看项目的元数据。无论以何种方式添加元数据（直接将其添加到项目中或者从 ArcGIS Pro 或 ArcMap 发布数据时将其包含在内），都会出现这种情况。即使禁用了元数据，您仍然可以查看这些包含元数据项目的元数据。

此外，对于所有包含关联元数据的托管要素图层，还会在图层列表中显示元数据链接。通过该链接，任何有权访问托管要素图层项目页面的用户都可以在为门户配置的元数据样式中查看单个图层的元数据。 同样，如果某表随元数据一同发布，您将看到表列表中的元数据链接。

随项目提供基于标准的元数据可以帮助人们更加了解项目并决定是否使用它。

查看元数据

项目所有者向项目中添加元数据后，所有者可以在元数据按钮中选择查看或编辑。对项目拥有访问权限的所有其他门户管理员都可以看到元数据按钮，并且可以使用该按钮查看为门户配置的元数据样式中的元数据。

此外，对于所有包含关联元数据的托管要素图层，还会在图层列表中显示元数据链接。通过该链接，任何有权访问托管要素图层项目页面的用户都可以在为门户配置的元数据样式中查看单个图层的元数据。 同样，如果某表随元数据一同发布，您将看到表列表中的元数据链接。

编辑元数据

如果门户管理员已启用元数据，则项目所有者可以为其项目包括其他基于标准的元数据。项目所有者可使用项目上的内置元数据编辑器包括其他元数据并将其存储为 ArcGIS 元数据格式。ArcGIS 元数据格式包含可存储由 ArcGIS 支持的所有元数据标准内的所有内容的字段，这些内容包括所有 FGDC CSDGM 元数据内容、所有 ISO 19139 元数据内容、所有 North America Profile 元数据内容以及所有 INSPIRE 元数据内容。ArcGIS 格式也存储没有包括在这些官方元数据标准和专用标准内的项目属性，例如缩略图。

首次打开项目编辑器时，编辑器由源于以下项目页面字段的现有元数据所填充：title、tags、snippet、description、accessinformation、licenseinfo 和 extent。如果上传本地元数据 XML 文件，您可选择使用由 XML 文件导入的元数据覆盖项目详细信息，或使用现有的项目详细信息。

按照以下步骤编辑项目详细信息以包括项目元数据。

注：

目前，元数据编辑仅适用于托管要素图层的项目级别。可以使用图层列表中的元数据链接查看所有与托管要素图层相关联的元数据，但无法进行编辑。

  1. 验证您是否已登录，是否具备创建内容的权限，以及您的门户管理员是否已为门户启用元数据。
  2. 从内容页面上我的内容选项卡中，单击将包括元数据的项目的标题。
  3. 单击元数据，并提供用于描述项目的元数据内容。带有星号 (*) 的字段是正在使用的元数据格式的必填字段。
    提示：

    您可通过打开元数据 > 专用标准选项卡来查看门户的元数据样式。

  4. 在您当前编辑会话中，您有以下选项：
    • 验证，用于验证您是否填充了元数据样式的必填字段。您将收到一条消息，通知您文档是否有效或哪些必填字段没有被填充，并附上那些字段的链接。
    • 保存，用于保存更改并保持元数据编辑器开启。
    • 保存并关闭，用于保存更改并关闭元数据编辑器。
    • 保存本地副本，将当前的元数据文档（其中包括在编辑会话期间所做的任何更改）下载为采用 ArcGIS 元数据格式的 XML 文件，且不对其应用任何元数据样式。
    • 打开，用于导入您计算机中现有的元数据 XML 文件。单击打开，单击浏览按钮，然后选择计算机上的文件。元数据 XML 文件需要采用 ArcGIS 元数据格式。

      选中复选框以用项目详细信息填充元数据。如不选中复选框，文件中的元数据将会更新项目详细信息。

    • 关闭，用于在不保存的情况下关闭编辑会话。
    • 删除，用于删除添加到项目中的任何元数据，并在他人查看项目时隐藏元数据按钮。删除选项仅在项目包含元数据时可用。它不能删除项目详细信息。

元数据样式、标准和专用标准

元数据样式配置了编辑和查看元数据内容的体验。元数据样式类似于对项目的元数据应用过滤器。样式控制元数据的查看方式以及元数据编辑器中出现的页面的查看方式。基于标准的元数据样式旨在支持元数据的标准或专用标准。样式决定按照标准或专用标准验证元数据的方式。

默认的项目页面可用于查看和编辑项目的一组简单的元数据属性。信息被编入索引且可搜索，并且可以被跨平台的应用程序所使用。项目详细信息中的信息简单有效，适合任何不需要遵守特定元数据标准的人员。当您的门户启用元数据时，项目详细信息属性将与项目基于标准的元数据保持同步。

元数据样式

下面提供的元数据样式支持不同的元数据：

  • FGDC CSDGM 元数据 - 使用此样式可以查看和编辑遵循 FGDC 数字地理空间元数据内容标准 (CSDGM) 指导原则的元数据，并使用 CSDGM XML DTD 进行验证。
  • INSPIRE Metadata Directive - 使用此样式可以查看和编辑符合 INSPIRE 实施规则的完整 ISO 19139 元数据文档，并使用 ISO 19139 XML 模式进行验证。
  • ISO 19139 Metadata Implementation Specification - 使用此样式可以查看和编辑所有符合 ISO 标准 19139，即 Geographic information — Metadata — XML schema implementation 的完整元数据文档，并使用标准的 XML 模式进行验证。用此样式可创建符合 ISO 标准 19115，即 Geographic information - Metadata 的元数据。
  • ISO 19139 Metadata Implementation Specification GML3.2 - 除文件使用 GML 3.2 命名空间外，此样式与上文提及的样式相同，因此可使用引用 GML 3.2 命名空间的 ISO 19139 XML 模式版本对其进行验证。例如，如果您计划使用 NOAA NCDDC XML 模式验证元数据文件，则请使用此样式。
  • North America Profile of ISO 19115 2003 - 使用此样式可以查看和编辑符合 North American Profile of ISO 19115:2003 – Geographic information – Metadata 的完整元数据文档，并使用 ISO 19139 XML 模式进行验证。

元数据标准和专用标准

元数据标准是标识用于描述地理空间资源（如地图、地图服务、矢量数据、影像），甚至一些与空间工作相关的非空间资源（如表格和工具）的内容的文档。元数据标准还可提供一个描述内容存储格式的 XML 模式。通常，标准 XML 格式是使用 XML 模式或文档类型定义 (DTD) 来定义的。标准通常由国家或国际标准组织批准。

许多地理空间元数据标准均为国际标准组织 (ISO) 委员会所创建，包括作为内容标准的 ISO 19115 Geographic Information—Metadata 和 ISO 19119 Geographic Information—Metadata—Services。相比之下，ISO 19139 Geographic Information—Metadata—Implementation Specification 提供一组“XML 模式”，这些模式定义存储 ISO 19115 和 ISO 19119 元数据内容应采用的 XML 格式。

联邦地理数据委员会 (FGDC) 颁布了 Content Standard for Digital Geospatial Metadata (CSDGM)，该标准已在美国使用多年，通常被称为 FGDC 元数据标准。它是另一个内容标准示例。虽然，已有一种被接受的格式用于以 XML 格式存储这些内容，但是，还有几种其他文件格式常用于呈现信息。

元数据专用标准是用于修改元数据标准的文档。专用标准可以减少标准所定义的元数据元素的总数。专用标准可以进一步限制元数据元素是否可选，使其从可选元素变为必选元素；但是，专用标准不能将必选元素变为可选元素。专用标准还可以进一步限制元数据元素中所允许的值。元数据专用标准可以由标准团体、机构或组织批准代替元数据标准。由美国和加拿大共同创建并批准的 North American Profile of ISO 19115:2003 就是元数据专用标准的一个示例。

无论出于自愿还是职责所在，如果计划创建符合某标准或专用标准的元数据，请获取该标准或专用标准文档的副本。对于 ISO 标准，必须从 ISO 或与 ISO 合作的国家组织（如美国的 ANSI）处购买文档。专用标准则可以从创建该标准的组织处获得。

常见问题解答

以下为有关 ArcGIS Enterprise 门户中的元数据的常见问题。 如果您认为其他用户可能也会经常遇到您所遇到的问题，请单击页面底部的反馈链接，将该问题作为候选问题进行提交。

ArcGIS Enterprise 门户中支持哪些元数据样式？

目前，ArcGIS Enterprise 支持以下基于标准的元数据样式：FGDC CSDGM Metadata、INSPIRE Metadata Directive、ISO 19139 Metadata Implementation Specification GML3.2、ISO 19139 Metadata Implementation Specification 和 North America Profile of ISO 19115 2003。

ArcGIS Enterprise 门户如何支持元数据？

您可以创建并共享受 ArcGIS Enterprise 门户支持的所有项目的元数据，其中包括 web 地图、web 场景、web 应用程序、图层、上传的文件和工具等等。

可使用基于 web 的编辑器来创建和编辑元数据，并将元数据作为单独浏览器选项卡中的 HTML 进行查看。通过门户管理员选择的样式来查看元数据。目前，管理员可以配置一种元数据样式，以用于创建、编辑和查看元数据。管理员可以在不丢失元数据的情况下更改样式。

基于 web 的编辑器使用 ArcGIS 元数据格式。可用于在 web 编辑器中进行编辑的字段由门户管理员所配置的样式决定。 ArcGIS 元数据格式包含可以存储 ArcGIS 支持的所有元数据标准的全部内容的字段。

在发布托管图层时是否自动包括元数据？

是，从 ArcMap 向 ArcGIS Enterprise 门户发布托管图层，或发布在 ArcMap 中创建的服务定义文件时，元数据将自动用于图层项目。如果从 ArcGIS Pro 发布，则您发布的地图或图层的元数据将包括在托管要素图层中。

对于托管要素图层来说，从 ArcGIS Pro 发布时也会包括图层级元数据。

  • 在 ArcMap 中，当您将地图文档发布为托管图层时，地图文档的所有元数据都会被自动包括在内，并可用于托管图层项目。各个图层的元数据不会被包括在托管图层项目的元数据内。您可以在 ArcCatalog 或门户中对托管图层元数据进行编辑。要素类的元数据文档可以用于更新托管图层的元数据，方法为在 web 编辑器中打开并保存文档，或将文档导入到 ArcMap 中。
  • 当您发布来自托管要素图层的托管 WFS 图层时，托管 WFS 图层会继承托管要素图层的元数据。对托管要素图层元数据进行的后续编辑不会影响托管 WFS 图层元数据，对托管 WFS 图层元数据进行的编辑也不会影响托管要素图层元数据。
  • 在 ArcGIS Enterprise 门户中，如果发布的 shapefile 包含元数据，或发布的文件地理数据库包含带有元数据的要素类或表格，则该元数据将包含在托管要素图层中，并且您可以在门户中要素图层的项目页面上查看每个图层的元数据。
  • 如果您作为托管要素图层发布的 ArcGIS Pro 2.2 地图包含具有元数据的要素类或表，或者元数据与地图中的图层一起保存，则元数据将包括在托管要素图层中。您可以在门户中的要素图层项目页面上查看每个图层的元数据。
  • 从托管要素图层创建托管要素图层视图时，该视图将继承托管要素图层及其图层的元数据。

如何将元数据导入到企业级门户中？

  • 有关如何从 ArcGIS Pro 或 ArcMap 发布元数据的相关信息，请参阅上一个问题和答案。请注意，此时，您只能通过覆盖图层来编辑图层级元数据。从切片包发布时，元数据不会发布到图层项目。
  • 可使用 web 编辑器更新所有项目的元数据。在 ArcGIS Pro 和 ArcMap 中，您可以编辑这些应用程序中可用项目的元数据。
  • 可使用以下其中一种方法将门户中的所有项目均更新为包括元数据：
    • 在 ArcGIS Pro 中复制或导入元数据。
    • 使用 ArcMap 中的 XSLT 变换地理处理工具和以下任一样式表，将元数据内容导出到采用 ArcGIS 元数据格式（exact copy of.xslt、remove local storage info.xslt 或 remove local storage info strict.xslt）的独立元数据 XML 文件。元数据 XML 文件可以在 web 编辑器中打开，并用于更新托管图层的元数据。

ArcGIS Pro 如何支持元数据？

ArcGIS Pro 支持对可在 ArcGIS Pro 中访问的、您拥有的任何项目的基于标准的元数据进行编辑。有关详细信息，请参阅 ArcGIS Pro 帮助中的查看和编辑元数据。

当我注册 ArcGIS Server 服务时，元数据是否会成为项目的一部分？

否。注册 ArcGIS Server 服务作为门户的项目时，只有项目详细信息将自动包括在内。 此时，现有元数据不会成为项目的一部分。这些项目的元数据可以通过使用 web 编辑器、ArcGIS Pro 或 ArcMap 进行更新。

如何从企业级门户中导出元数据？

要下载项目的元数据，单击 web 编辑器内的保存本地副本。 该本地副本包括在编辑会话期间所做的尚未保存至项目的所有更改。

元数据是否可被搜索？

是，目前内容搜索使用以下项目详细信息中的特定字段：title、tags、snippet、description、accessinformation。您在元数据编辑器或项目页面中对这些字段所做的更改会自动在两处进行同步。与要素图层中的图层相关联的元数据不可搜索。

从我的托管要素图层导出数据时是否包含元数据？

是的，当您以 shapefile 或文件地理数据库格式从托管要素图层导出数据时，将自动包含与图层关联的所有元数据。

是否可以导入除 ArcGIS 元数据格式之外的元数据格式?

否，不能导入其他元数据格式。目前，只有 ArcGIS 元数据格式可以在元数据编辑器中打开。Web 编辑器只打开 ArcGIS 元数据格式的元数据文档。如果您尝试打开其他格式的文档，编辑器将显示一条错误消息，且不会打开元数据。您可以利用 ArcMap 中的 XSLT 变换地理处理工具将其他元数据格式转换为 ArcGIS 元数据格式。使用 ArcGIS Pro，您可以将基于标准的元数据格式导入到项目中。作为此过程的一部分，元数据将转换为 ArcGIS 元数据格式。

将项目添加至内容页面上“我的内容”选项卡时，是否提取元数据？

发布文件地理数据库或 shapefile 或者从ArcGIS Pro 或 ArcMap 发布托管要素图层时，数据中包含的所有元数据都将自动用于图层。如果 shapefile 或文件地理数据库仅包含一个图层并具有元数据，则项目和图层都将包含关联的元数据。如果 shapefile 或文件地理数据库包含多个图层，则元数据仅适用于包含元数据的图层，不与项目相关联。

管理员是否可以切换门户元数据的样式？

可以，门户管理员可以切换门户的元数据样式而不会丢失任何元数据。 然而，之前用于编辑的样式可能不会含有更新后样式的所有元素。项目所有者可以利用编辑器将元数据更新到最新的样式。门户所配置的最新样式将于查看元数据时出现，除非您的元数据样式已根据基于 ISO 的标准设置，并且将样式切换到不同的 ISO 专用标准。 这种情况下，元数据将以制作的样式被查看。这时，您无法使用编辑器切换样式。

元数据注意事项

  • 目前，可将门户配置为使用一次元数据样式。
  • ArcGIS 元数据格式是唯一支持导入元数据的格式。
  • 从 ArcMap 发布数据中已包含元数据的托管 web 图层时，如果门户未针对元数据进行配置，可在门户管理员启用了门户的元数据功能后查看该元数据。为此，请打开托管 web 图层的项目页面，打开元数据编辑器，然后单击保存。该操作将使用门户管理员在样式中配置的元数据来填充元数据编辑器。
  • 目前，title 字段未在内置编辑器与项目页面间保持同步。此类情况包括上传元数据 XML 文件。第一次打开编辑器时，字段会包含项目页面中的标题。您在编辑器内对标题所做（和保存）的任何更改都会被保存到基于标准的元数据中。它们不会保存到项目页面中。
  • 目前，您对托管 WFS 图层元数据进行的编辑不会更新基础 WFS 服务的功能文件。
  • 目前只能查看图层级元数据，并且无法进行编辑。
