服务器配置详细信息

站点旨在使用 ArcGIS Enterprise 部署内的内容和服务，使您可通过 ArcGIS Enterprise 站点创建的站点和页面访问这一内容。可用于站点的内容和服务类型包括：Enterprise 门户中的托管图层、来自与 ArcGIS Enterprise 联合的服务器的服务，以及已作为项目注册到门户的服务。

注：

站点 仅支持 ArcGIS Server 10.1 或更高版本中的服务。

以下提供的一些最佳做法可用于组织和管理服务及数据，以便以最佳的状态用于站点。

服务属性

最大记录计数应少于 5000

当发布服务时，默认最大记录计数设置为 1000 或 2000，以便提供服务器到客户端间的最佳性能。最大记录计数确定了在单次请求中可以返回的最大要素数量。如果将服务的此计数设置得过高，客户端可能会在单次请求中请求全部数据，从而造成生成速度缓慢，通过 Internet 发送的内容过多。

如果您图层或表格中的总要素数量刚刚超过 2000 或在 4000 左右，则将该值增加到默认范围以上会有所效果。

请注意，无论最大记录计数是多少， 站点 都会一次查询出所有数据的 1000 条记录，将其整合在一起，并支持以 CSV、KML、SHP 或 GeoJSON 的格式下载文件。

OGC 链接

服务发布者对特定服务启用 WMS、WFS 和 WCS 链接后，会将这些链接添加到各个站点项目视图的 API 部分。如果发布者在发布时没有启用这些功能，则可以编辑服务以对其进行启用。请注意，OGC 链接将仅显示在 ArcGIS Server 10.2 或更高版本的服务中。

无需要素服务

站点 从地图服务中查询要素的方法与其在要素服务中查询要素的方法相同。除非您的其他需求需要启用“要素服务”，否则最好将其关闭。

比例可变选项无关紧要

站点 的工作方式是向您的地图或要素服务发送请求，并在不考虑范围的情况下查询数据。尽管出于制图考虑，您可能不想以全球比例显示地址点，但 站点 仍将查询出所有数据，并提供高级的数据可视化效果来显示位置概要或者以可过滤的网格可视化来表现要素的密度。这些可视化不可自定义。

站点的下载支持

对于所有上传的文件（图像、PDF 等）和所有托管图层，只要它们支持提取功能，即启用了下载功能。如果您共享的项目来自不支持提取的服务，则下载将被禁用，同时出现工具提示。

组织服务

大型服务将超时

与将所有数据组织到一个服务中相比，将数据组织到多个服务中更加快速。为了能够在站点中获得最佳性能，我们建议每个服务中包括的图层不要超过 20 个。

支持编码值属性域

与前一部分相反，生成 open-machine-readable 的格式时，有一种地理数据库行为受到支持。在 站点中查看数据时，将支持编码值域，下载数据时，原始值将由编码值替换。

项目配置

配置要用于站点的项目时，请牢记以下最佳做法。

项目标题为数据集名称

站点 使用“使用条款”字段作为数据许可。

Enterprise 门户中的每个项目都具有这样一个提供使用条款信息的部分。 站点 将此字段解释为数据的许可。向数据应用许可对数据使用者了解其对客户使用方式和使用创建人来说十分重要。

许多提供者使用指向其使用的现有数据策略的链接，而其他人则生成 Creative Commons 许可。您可通过 Creative Commons License Picker 启用项目详细信息这一部分，以便输入 HTML 和 license picker 生成的代码。
