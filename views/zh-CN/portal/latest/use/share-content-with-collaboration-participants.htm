与协作群组共享内容

在本主题中

  1. 共享 web 应用程序
  2. 共享地图
  3. 共享场景
  4. 共享图层和视图
  5. 共享文件
  6. 共享 ArcGIS Insights 项目

支持的项目类型可以在协作中的参与者之间共享。这些项目类型包括：

  • Web 应用程序
  • 地图
  • 场景
  • 图层和视图
  • 文件
  • Insights

与参与者共享内容后，将在参与的管理员文件夹下创建一个新文件夹。此文件夹用于导入内容和存储项目，这些内容和项目随后会共享至与协作工作空间关联的群组。该文件夹将基于协作名称命名。

注：

从 ArcGIS Enterprise 10.6.1 开始，可使用 ArcGIS Enterprise 版本（即等于或高于发送方版本）与协作参与者共享 web 应用程序。可从 ArcGIS Enterprise 将 web 应用程序共享至 ArcGIS Online，但是无法将 web 应用程序从 ArcGIS Online 共享至 ArcGIS Enterprise。

更改项目的共享方式时，还要确保修改相关项目的共享设置。例如，如果 web 地图之前与所有人共享，但您将其修改为与组织共享，则应相应地更改 web 地图中包含的所有业务图层中的设置。

共享 web 应用程序

可通过以下方式与协作参与者共享 web 应用程序：

可配置的应用程序模板

与协作参与者共享可配置应用程序模板时，会在协作中创建一个相应的 web 应用程序项目。组织的可共享主题属性也将与应用程序共享。您应该与协作群组共享所有 web 应用程序的相关项目，例如 web 地图、徽标等图像、图层和源数据。这样做可以确保协作参与者能够收到 web 应用程序的副本和应用程序所使用项目的副本。Web 应用程序中的要素图层将根据工作区设置进行共享。应用程序中包括的所有实用程序资源（其中包括地理编码、打印或 geoenrichment 服务）都将更新为使用目标 URL 中相应的实用程序服务。

从 ArcGIS Enterprise 10.8 开始，可以在 ArcGIS Enterprise 10.8 参与者之间或 ArcGIS Online 组织中共享基于群组的应用程序以及使用 ArcGIS Indoors 模板创建的应用程序。使用 ArcGIS Indoors 必须配置 ArcGIS Indoors 应用程序的接收者。

Web AppBuilder 应用程序

与协作参与者共享 Web AppBuilder 应用程序时，会在协作中创建一个相应的 web 应用程序项目。组织的可共享主题属性也将与应用程序共享。您应该与协作群组共享所有 web 应用程序的相关项目，例如 web 地图、徽标等图像、图层和源数据。这样做可以确保协作参与者能够收到 web 应用程序的副本和应用程序所使用项目的副本。应用程序中包括的所有实用程序资源（其中包括地理编码、打印或 geoenrichment 服务）都将更新为使用接收方门户中的相应的实用程序服务。

Web AppBuilder for ArcGIS 可用于使用即用型微件或自定义微件构建自定义 2D 或 3D web 应用程序。共享 Web AppBuilder 应用程序的接收方可以选择将相应的代码下载到他们的门户或在门户中创建应用程序模板。

注：

自定义微件无法通过协作共享。其中包括注册为项目类型 AppBuilder Extension 的所有已注册微件项目。

通过 URL 引用的其他 web 应用程序

通过提供 URL，您可以将现有应用程序作为项目添加到门户中。门户将根据现有 URL 引用应用程序。您需要确保已向接收参与者授予了访问应用程序的权限。

共享地图

与协作参与者共享包含托管图层的 web 地图时，将在协作中创建相应的 web 地图项目。还应与协作群组共享所有 web 地图中托管图层的相关项目和创建这些图层的源数据。例如，如果共享的 web 地图包含从 shapefile 发布的托管要素图层，则将托管要素图层和 shapefile 共享至与协作工作空间关联的群组。通过这种方式，协作参与者将收到 web 地图副本、托管要素图层副本和原始 shapefile 副本。

共享场景

共享 Web 场景时，只要协作群组配置为共享为副本，Web 场景中包含的要素图层就会共享为副本。

共享图层和视图

托管要素图层、托管要素图层视图和联合要素图层可以在协作中进行参考共享。这将为接收者门户创建一个新项目，该门户将引用回到原始门户中的实时图层。

或者，托管要素图层、托管要素图层视图和联合要素图层可以作为副本与其他协作参与者共享。这将复制数据并在接收者门户中创建新的托管要素图层。对原始图层进行的编辑将按指定的间隔与复制的图层同步。

在设置或加入协作工作空间时，管理员需要选择是否将图层作为参考或者副本共享。当要素图层作为参考或副本共享时，只有原始门户的指定成员才能够编辑要素图层。

将托管要素图层、托管要素图层视图和联合要素图层作为参考共享

对托管要素图层、托管要素图层视图和联合要素图层进行参考共享将为协作接收者创建新的图层项目，但不会复制数据。接收者门户中的图层项目将引用回到原始要素图层或要素图层视图。要访问参考的图层，成员必须具有访问原始门户的权限，或者需要与所有人共享图层。如果图层是安全的，则成员将需要在原始门户中作为成员（用户类型至少为 Viewer）进行身份验证，该成员可以通过共享设置访问共享图层。

将托管要素图层、托管要素图层视图和联合要素图层作为副本共享

可将托管要素图层、托管要素图层视图和联合要素图层作为副本共享。当将图层共享为副本时，数据将复制到接收者门户，并发布新项目（托管要素图层）。复制的项目将由接收门户托管，该门户由图层的服务 URL 表示。原始要素图层的所有者可编辑图层，并通过同步间隔将更改同步给所有参与者。客户将选择在协作工作空间中同步编辑内容的间隔。要将图层共享为副本，请确保在托管图层或视图上启用了同步，或者已经相应地配置了联合图层。

如果加入的工作空间的访问权限包括发送内容，则向工作空间群组添加的项目将发送至从工作空间接收内容的组织。

有关更多详细信息，请参阅有关将要素图层和视图共享为副本和查看同步状态。

共享 ArcGIS Server 服务

如果将 ArcGIS Server 服务项目与其源自的门户以及与协作工作空间关联的群组共享，则在向地图添加服务时，系统仍会提示其他参与者成员输入凭据。要避免输入凭据的提示，必须在源门户中与所有人以及与协作工作空间关联的群组共享服务。

与协作参与者共享的 ArcGIS Server 服务将在参与者门户中创建参考原始服务的项目；服务和对应数据不会复制到每个参与门户中。如果参与者共享要素服务，则仅该服务的所有者可以编辑原始要素服务。

要素图层作为数据副本共享后，任何后续的方案更改（如启用附件）都将不会与其他参与者门户同步。原始托管要素图层上的 基于所有权的访问控制设置将不会包含在复制数据中。

共享文件

与协作参与者共享的文件（例如 PDF）将在协作参与者门户中创建对应的文件项目。当同步内容与项目源时，将覆盖以访客参与者身份对这些文件做出的任何更改。这同样适用于与项目关联的元数据。

只要移动地图包 (.mmpk) 不属于“esri_nav”，便可以将其与协作参与者共享。

注：

项目评论不会同步给协作参与者。

共享 ArcGIS Insights 项目

自 ArcGIS Enterprise 10.7 起，ArcGIS Insights 可以与协作参与者共享工作簿、页面、模型和主题。

先决条件

在将任何 Insights 项目共享给协作之前，必须满足以下条件：

  • Insights 必须获取 ArcGIS Enterprise 和 ArcGIS Online 的正确许可。有关更多许可详细信息，请参阅如何配置门户以支持 Insights。
  • ArcGIS Enterprise 接收者必须安装 Insights 才能打开收到的工作簿、页面、模型和主题。
  • 参与者必须安装 Insights 3.2.1 或更高版本才能共享和接收 Insights 内容。项目仅能被所使用的 Insights 版本与发送者当前版本相同或高于当前版本的接收者共享。
注：
您将能够将 Insights 项目从 ArcGIS Enterprise 共享到 ArcGIS Online；但是，Insights 项目无法从 ArcGIS Online 共享到 ArcGIS Enterprise。

ArcGIS Insights 工作簿和页面

将 Insights 工作簿和页面与协作参与者共享将会为其提供对共享项目的只读权限，从而允许接收者浏览您的分析。当您与协作参与者共享工作簿时，您对原始项目所做的任何后续编辑都将在下一次协作同步后提供给接收者。您与接收者共享的页面将不会接收到任何持续进行的编辑；如果您希望向接收者提供最新的编辑，则需要将该页面重新与协作参与者共享。

参与者可能无需 Insights 许可即可查看共享工作簿或页面；但是，如果您的 Insights 项目或其内容受保护，协作参与者必须能够在 ArcGIS Enterprise 门户中作为已与其共享源工作簿或页面的成员进行身份验证。

注：

如果工作簿包含直接从 Insights 支持的关系数据库添加的数据，接收者必须能够在原始门户中作为有权访问关系数据库连接项目的成员进行身份验证。

您可以通过查看 ArcGIS Insights 文档找到有关共享工作簿或共享页面最佳做法的详细信息。

ArcGIS Insights 模型

参与者可以通过共享 Insights 模型在模型中工作，以浏览和分析其自己的数据。与协作参与者共享模型时，将复制该项目并将其添加到每个参与者的 ArcGIS Enterprise 或 ArcGIS Online 组织。这些新项目可作为 Insights 项目快照，这表示对工作簿或页面进行的任何后续编辑均不会反映在接收者的模型中。接收者可以重新配置共享模型，以使用其自己的数据和参数对其进行运行。

注：

接收组织中的用户必须获得 Insights 的许可才能打开和编辑接收的模型。

您可以通过查看 ArcGIS Insights 文档找到有关共享模型最佳做法的详细信息。

Insights 主题

可以共享主题并将其应用于模型，以便为协作参与者维护一致的设计标准。

您可以通过查看 ArcGIS Insights 文档找到有关共享主题最佳做法的详细信息。
