使用此网站时遇到了最大值限制。

网站具有下列最大值限制：

  • 每个用户的群组数量：512
  • 上传文件大小：200 GB
  • 缩略图大小：1 MB
  • 登录时间：两周
  • 通过 Web 添加项目的 URL 字符：1024

尝试向门户添加新成员时出现软件授权错误。

对 Portal for ArcGIS 进行许可，以允许特定的成员数量。如果门户的成员数量已达许可的最大数量，则无法添加新的帐户，并且将显示消息由于软件授权错误无法创建新帐户。请联系 Portal for ArcGIS 管理员获取帮助。 门户管理员必须要获得新的许可或移除现有成员才能添加新成员。

尝试登录我的门户时出现软件授权错误。

对 Portal for ArcGIS 进行许可，以允许特定的成员数量。如果门户成员超出该数量，下次尝试登录门户时会显示以下消息：由于软件授权错误无法登录。请联系 Portal for ArcGIS 管理员获取帮助。 门户管理员必须要获得新的许可或从门户中移除成员。

搜索图层添加地图时，仅列出 web 图层。

您可以搜索并添加要素图层、切片图层、影像图层、KML 图层以及 OGC WMS、WMTS 和 WFS 图层。不支持其他图层类型（例如图层包）。

在搜索要添加到地图中的图层时，搜索结果中并未列出我的组织的所有图层。

如果选中了在地图区域内框，则搜索结果中将仅列出与当前地图范围相交的图层。

弹出窗口中的日期和时间与基础数据中的日期和时间不匹配。

服务器以协调世界时间 (UTC) 存储日期。而 Web 浏览器会将日期转换为本地时间。例如，如果您在加利福尼亚州查看弹出窗口中的时间字段，且此时为标准夏令时，您将看到早于数据时间 8 小时的时间 (UTC-8)。此偏移量可能也会影响日期。如果在弹出窗口中查看 7/7/2017 12:00 AM 时的数据，您会看到 7/6/2017 5:00 PM。

日期在整个地图中的格式有所不同。

在 Map Viewer 中查看日期时，您可能会看到地图中不同的部分具有不同的格式。地图制作工具可用于配置弹出窗口、表格和标签中显示的格式。对于地图的其他部分，例如时间滑块、与您的帐户相关联的语言将决定日期格式。

时间不会显示在日期字段中。

仅短日期格式会显示时间（例如，12/31/1999 和 31/12/1999）。而其他日期格式不会将时间作为日期字段的一部分显示。如果要在日期字段中显示时间，则需要在弹出窗口中配置日期字段以使用其中一种短日期格式（并选中复选框才能显示时间）。

无法在可编辑的要素图层中编辑要素的几何。

如果可编辑的要素图层包含 m 值，则无法编辑现有要素的几何。如果要编辑图层中现有要素的几何，则图层所有者必须将包含 m 值的数据移至独立图层（该图层可以位于同一服务中）。

打印地图时，某些图层丢失。

根据管理员配置打印服务的方式，某些类型的图层可能不会显示在打印的地图上。有关详细信息，请与您的管理员联系。此外，如果使用浏览器的打印按钮打印地图，也可能会缺失许多其他图层和徽标。使用 Map Viewer 中的打印按钮可打印最完整且格式良好的地图。

我的地图图层未绘制完全。

对于 Map Viewer 而言，图层中包括的要素数量可能过多，以至于无法同时显示所有要素。发生这种情况时，Map Viewer 会显示一则警告，指示图层未绘制完全。要查看省略的要素，请放大并四处移动地图。

了解在地图中使用图层的最佳做法

新建 web 地图的缩略图未生成或者无法正确显示。

如果您的 web 地图包含使用 HTTPS 的 ArcGIS Server 服务，则可能会遇到此类问题。打印服务用于生成缩略图。如果带有打印服务（用于生成缩略图）的计算机不信任托管 HTTPS 服务的 ArcGIS Server 站点，则缩略图不会生成或者无法正确显示。要解决此问题，请联系门户管理员，并让管理员在计算机与站点之间建立信任。

地图缺少图例。

将不会显示底图或 OGC WMS 和 OGC WMTS 图层的图例。也不会显示外部无法访问的图层的图例，以及地图作者所隐藏的图层的图例。

该地图图例与在要素图层中创建的图例不同。

要素图层的图例可能与 web 地图中保存的图例不匹配。当您使用与原始图层不同的符号系统在 web 地图中保存图层时，通常会发生这种情况。保存地图时，符号系统将保存到 web 地图。因此，如果您将不同的符号系统保存到图层并希望在 web 地图中显示图层的图例，则必须从 web 地图中移除该图层并将其添加回来。

我无法对地图内容中的图层重新排序。

图层始终显示在底图之上，要素图层始终显示在切片集图层（地图图像、影像和切片图层）和 KML 图层之上。根据地图中图层类型的不同，您可能无法将所有图层上移或下移。例如，如果您有一个地图图像图层和三个要素图层，则可以对要素图层进行重新排序，但无法将地图图层移至其上方。

我丢失了正在处理的地图。

当您在 HTTP 中添加安全图层时或在地图过大以致浏览器缓存不足的情况下导航离开 Map Viewer 时，可能会导致丢失地图。为防止在添加大量数据时丢失地图，请在开始工作之前登录并定期保存地图（如果具有创建项目的权限）。如果计划添加安全图层，请将 Map Viewer URL 更改为 HTTPS 并首先添加安全图层。

Web 地图包含不再可用的图层，加载地图时会出现错误。如何移除此图层，以便使用另一图层来替换此图层？

保存 Web 地图。问题图层无法保存，您可以根据需要向地图添加其他可用图层。

时间滑块不会显示在具有时间感知型图层的地图中。

在 KML 文档、嵌入式地图和来自文件的图层中不支持时间感知型图层。上述情况下不会显示时间滑块。如果地图作者在图层上禁用了时间动画，也不会显示时间滑块。

没有可将我的图层用作底图的选项。

ArcGIS Server 服务（地图图像图层）、OGC WMS、OGC WMTS 和切片图层可用作底图。文件中的图层、KML 文档和在 Map Viewer 中创建的地图注释不能用作底图。

使用自己的底图时，在地图中会丢失一些底图的缩放级别。

出现上述情况的原因是地图显示的是当前底图（例如，世界地形底图）的缩放级别。要查看所添加底图的其他缩放级别，请保存地图，关闭 Map Viewer 页面（例如，转至“图库”），然后重新打开该地图。随即显示其他的缩放级别。您需要创建项目的权限才能保存地图。

由于底图不可用，所以不会显示任何地图内容。

Map Viewer 无法在没有工作底图的情况下在地图中显示图层，因为底图用于建立地图的坐标系。将底图随地图一起保存后，Map Viewer 将只使用该底图；如果您的底图不可用，它不会恢复成默认底图。只要已损坏底图和新底图的空间参考相同，就可以使用其他的底图来修复您的地图。对于作为切片图层的底图，已损坏底图和新底图的切片方案也必须相同。通过从图库中选择不同的底图或使用 URL 参数basemapUrl 修复您的地图。

将 OGC WMS 或 WMTS 图层添加到地图中时，会收到一条错误消息，该消息提示此图层的坐标系未与底图的坐标系匹配。

如果已向地图中添加了图层，然后再添加非 Web 墨卡托坐标系（Map Viewer 默认图库中的底图投影）的 OGC 图层，则通常会显示此错误消息。要解决此问题，请创建新地图，然后先添加 OGC 图层。如果您的图层在 GCS WGS84 投影坐标系中，则 Map Viewer 将使用 GCS WGS84 世界影像底图。如果您的 OGC WMS 或 WMTS 图层处于一个非 Web 墨卡托或 GCS WGS84 的坐标系中，则您的图层将用作底图。如果您的 WMTS 图层支持多个坐标系，您可以选择一个要使用的坐标系。

添加的航空影像在地图上不显示。

如果正在使用 Web 墨卡托投影底图（图库中的底图投影）而图层不在 Web 墨卡托投影内，采用航空影像的切片图层可能无法在地图上显示。要显示该图层，可将其用作底图或在与切片图层处于同一投影中的底图上方进行显示。

我的地图图层在 Map Viewer 中没有以缓存切片时采用的比例来显示。

Map Viewer 支持通过 ArcGIS 10.2 创建的图层中的最小和最大比例设置。可在 ArcGIS Desktop 中调整这些比例。

我在 Microsoft Internet Explorer 中访问 ArcGIS Server 服务和受 Web 层身份验证保护的 OGC 图层时遇到问题。

您可能需要将组织的受信任服务器的属性域添加到浏览器的受信任站点列表。有关为组织配置的受信任服务器，请联系您的组织管理员。

某些图层在地图中显示不正确。看起来像是 HTTP 和 HTTPS URL 的混合。这种混合的内容是否对地图显示造成负面影响？

视情况而定。该特定行为取决于您的浏览器和浏览器版本。

当您尝试在基于 HTTP 的 Map Viewer 中打开基于 HTTPS 的图层时，浏览器可能会通知您必须重新将页面（此情况下是 Map Viewer）加载到 HTTPS 中并重新添加图层。此方法适用于您添加到地图中的图层以及现有地图中的图层。某些浏览器可能会成功加载这些图层而不发送通知。

当您尝试在基于 HTTPS 的 Map Viewer 中打开基于 HTTP 的图层时，图层可能无法正确加载或在不同的浏览器或浏览器版本中表现各异。此方法适用于您添加到地图中的图层以及现有地图中的图层。某些浏览器可能会成功加载这些图层。

您可通过多个浏览器提供商了解有关混合内容的详细信息。

  • Chrome - 支持文章和博客文章
  • Firefox - 支持文章和Mozilla 开发人员网络文章
  • Internet Explorer - 博客文章

尝试向地图添加图层或打开现有地图时，出现消息提示 Map Viewer 无法与一个或多个图层建立安全连接。

可能仅为组织设置 HTTPS，但是您提供的 URL 为 HTTP。某些浏览器会通知您或阻止混合内容。可以尝试将图层 URL 切换至 HTTPS。关于在托管图层的服务器上配置“传输层安全性”(TLS)，您还可以联系管理员。TLS 是比“安全套接字层”(SSL) 加密协议更新且更安全的加密协议，能够提供跨网络安全通信。

地图中丢失了一些要素。

大数据集应发布为托管要素图层。将 shapefile、GeoJSON、CSV、TXT 或 GPX 文件直接添加至 Map Viewer 时，这些要素将存储于 web 地图中，这将增大每次打开 web 地图时必须下载的数据量。将数据发布为托管要素图层允许根据需要请求数据集中的要素，而非将其存储于 web 地图中。

使用门户登录后，如果添加包含地址的文本文件（.txt 或 .csv），则会将 4,000 个要素直接添加至地图。如果使用匿名用户或未登录的情况下添加具有地址的文本文件，可直接向地图中添加 250 个要素。对于包含的要素超过此值的数据集，必须将其发布为托管要素图层。

如果添加包含点数据的 shapefile 或 GeoJSON 文件，则可将 4,000 个要素直接添加至地图。如果添加包含面或线数据的 shapefile 或 GeoJSON 文件，则可将 2,000 个要素直接添加至地图。对于包含的要素超过此值的数据集，必须将其发布为托管要素图层。对于大于 10 MB 的 shapefile 和 GeoJSON 文件，必须将其发布为托管要素图层，不考虑其要素数量。

如果添加了 KML 文档，则会显示来自地标、网络链接、不具有刷新属性的地面叠加层、文件夹和扩展数据的要素。暂不支持其他要素。

我无法编辑已经添加到地图中的要素。

发布要素图层时，发布者将决定可通过要素图层执行哪些编辑操作（如果需要执行编辑操作）。请确保要素图层可编辑。而且，无法在 Map Viewer 中编辑 KML、OGC WMS 和 OGC WMTS 图层。

已知服务中包含要素数据，但我看不到配置弹出窗口的选项。

Map Viewer 不支持在 OGC WMS 和 OGC WMTS 图层上编辑或显示弹出窗口。不支持在 KML 图层上配置弹出窗口；KML 图层中的所有要素数据会自动显示（您无法禁用或配置这些数据）。

我无法移除地图中图层的弹出窗口。

您可以移除之前启用的地图图层的弹出窗口和来自带分隔符的文本文件或 GPX 文件的要素的弹出窗口。无法移除在 Map Viewer 中创建的地图注释图层或 KML 图层的弹出窗口。

我在 Map Viewer 中更新的图层上没有看到保存项目属性的选项。

您只能在您添加的图层上保存项目属性。

Web 上的 CSV 文件中的数值字段未能正确地导入到地图中。

如果要添加来自 web 的使用句号作为小数点的 CSV 文件，但是设置的系统语言却使用逗号作为小数点，则可能无法正确地显示数字字段。您文件中的小数点字符应与您系统语言所支持的格式相匹配。例如，如果您的系统设置为英语，则您的文件应使用句号作为小数点。如果您的系统设置为法语，则您的文件应使用逗号作为小数点。

我的组织具有可在公开共享的地图中使用的 Bing Maps 密钥。我可在地图中看到 Bing Maps 底图，但是公众看到的是 ArcGIS Online 底图。

确认您的组织是否已配置 Bing Maps 密钥以供公众使用。如果经过配置后公众仍无法在地图中看到 Bing Maps 底图，则请重新保存地图。

我在样式设置方面遇到问题。

请参阅样式设置注意事项列表。

仅搜索推荐的六个要素。

搜索图层中的要素时，搜索结果仅返回六个推荐的要素。这就可能导致列表中不包含您要查找的要素。

正在配置的故事地图不具有共享订阅者内容的设置。

您可以使用我的故事在故事地图中共享订阅者内容。

我无法通过托管要素图层发布托管切片图层。

如果托管要素图层配置为某些样式并保存到托管要素图层项目，则无法为从托管要素图层发布的托管切片图层生成地图切片。

此外，只有当门户具有采用托管服务器配置的关系数据存储 (ArcGIS Data Store) 时，才支持该工作流。

无法将 .txt 文件发布为托管要素图层。

CSV 文件必须已格式化，并另存为 .csv。不支持其他格式的文本文件。

我在上传 .tpk 文件并试图将其发布为托管切片图层时，收到了错误消息无法识别的服务类型：影像服务。

当在 ArcGIS Online 发布使用 LERC 压缩的切片方案所创建的切片包时，将生成托管高程图层。无法将其加载并发布到 ArcGIS Enterprise 门户。

我看不到通过使用 ArcGIS Desktop 发布到组织门户的选项。

为使用 ArcMap 或 ArcGIS Pro 发布到组织门户，必须满足以下条件：

  • ArcGIS Desktop 必须连接到正确的门户。默认情况下，ArcGIS Desktop 连接到 ArcGIS Online。使用 ArcGIS Administrator 从 ArcMap 连接到您的组织的门户。有关完整说明，请参阅将 ArcMap 与门户结合使用。在 ArcGIS Pro 中，将您的组织的门户添加到工程并将其设置为活动门户。
  • 您必须登录到您的组织的门户才能进行发布。在 ArcGIS Pro 中，活动门户以及您的登录状态显示在工程的顶部。在 ArcMap 中，单击文件菜单；如果您看到登录选项可用，则表示您并未登录。
  • 您必须是具有组织门户发布权限的角色的成员。通过登录到门户网站并选择姓名，可检查自己的成员角色。此操作将打开您的成员个人资料并显示您在组织中的角色。
  • 您的组织的门户必须已配置托管服务器。托管服务器允许门户成员将托管的切片、要素和场景图层发布至门户。如果不存在托管服务器，则不能发布到门户。请联系您的组织的门户管理员以确定是否已配置托管服务器。要了解有关托管 Web 图层的详细信息，请参阅关于托管 Web 图层。

发布托管切片图层需要花费较长时间。

通常，渲染特定地图切片所需的时间取决于要缓存的地图的空间范围、地图中的图层数量以及处理地图时系统上的负载。系统已设计为可随着负载增加自动进行缩放（通过添加计算机）。然而，当系统遇到过载时，可能会出现延迟。

发布上传到门户的服务定义 (.sd) 文件失败并显示消息无法发布项目。无法将该项目发布到 Online 服务。

ArcGIS Server Web 服务的服务定义文件与托管切片图层的服务定义文件不同。保存服务定义文件时所建立的连接可确定创建的服务定义文件类型。

如果在 ArcGIS Desktop 中保存服务定义文件，当您选择一个连接来保存可发布托管切片图层的服务定义文件时，必须选择我的托管服务。您无法连接到 ArcGIS Server 或选择无可用连接。

尝试在 ArcGIS Desktop 中打开托管图层的服务属性时，收到错误消息无法查看或编辑服务属性。请尝试直接通过服务器访问服务属性。。

如果图层是在门户网站中创建的，您将无法通过目录树中的我的托管服务节点查看服务属性。可通过连接到 ArcGIS Desktop 中的托管服务器查看服务属性，或直接在门户网站中查看图层属性。

我的托管要素图层不使用我在 ArcGIS Pro 或 ArcMap 中定义的符号系统。

Web 浏览器无法显示您在 ArcGIS Pro 或 ArcMap 中制作地图时使用的某些较为复杂的制图符号。大多数符号类型均可用，但某些符号会在发布时降级。有关受支持符号的详细信息，请参阅 ArcGIS Server 帮助中的创作地图以发布要素服务。在发布之前，对地图符号系统进行任何所需更改。

当我运行 GeoAnalytics 工具 创建用于在时空大数据存储中存储数据的托管要素图层时，我收到已配置节点均不可用消息。

时空大数据存储中至少一台计算机包含多个网络接口控制器 (NIC) 卡。每个卡使用不同的 IP 地址进行通信，并且未将数据存储配置为使用正确的 IP 地址。请联系 ArcGIS Data Store 管理员以解决此问题。

当我向 Map Viewer 添加要素图层或查询图层时，收到堆空间不足异常的错误消息。

如果要素图层中包括含较大和较复杂要素的多面体图层，则要素服务运行所在的 ArcGIS Server 站点可能会发生堆内存不足。请联系要素服务运行所在的 ArcGIS GIS Server 站点的管理员，因为管理员需要为站点中的每台机器增加 SOC 最大堆大小设置。SOC 最大堆大小设置在 ArcGIS Server 管理员目录中进行更改。

要确定要素图层中是否包含多面体图层，请在ArcGIS Server 服务目录中选中各个图层的几何类型。如果要素图层是您门户中的一个项目，您可以打开各图层的 ArcGIS Server 服务目录信息，方法如下：打开要素图层的详细信息页面，然后在概述选项卡的图层部分单击图层名称。

无法在 ArcGIS Desktop 中打开项目。

您必须安装 ArcGIS Desktop 9.3.1 或更高版本才能打开大多数项目。一些项目需要较新版本的 ArcGIS Desktop - ArcGIS Desktop 10 或更高版本才能打开 ArcGIS Desktop 加载项文件、地图包和 Web 地图。需要 ArcGIS 10.1 for Desktop 或更高版本才能打开地理处理包、要素、KML、定位器包、切片包、切片和 WMS 图层。

需要使用 ArcGIS Pro 1.1 或更高版本才能打开 ArcGIS Pro 加载项文件。

请注意，加载项可向后兼容，但不可向前兼容。例如，ArcGIS Pro 1.1 加载项可通过 ArcGIS Pro 1.1 或更高版本打开，而 ArcGIS Pro 1.2 加载项只能通过 ArcGIS Pro 1.2 或更高版本打开。

更新图层包或地图包项目详细信息时，即使选择了保持项目属性，图像也会更新。

更新图层包或地图包时，项目属性缩略图被重置为包中的缩略图。 即使选择将项目属性保留在服务器上，也会出现这种情况。更新项目后，您需要上传想要显示在项目页面上的图像。

打开图层包时系统提示我登录。

打开与群组共享但未公开的包时，系统会提示您（通过弹出对话框）输入门户网站的用户名和密码。此对话框包含保存登录信息的选项，用于在不重新输入用户名和密码的情况下打开共享包。

在 ArcMap 中打开 globe 图层将返回错误消息。

默认情况下，门户网站在 ArcMap 中打开 globe 图层时，您会收到一条错误消息。要成功打开这些项目，您必须在计算机上打开 ArcGlobe 并关闭 ArcMap。

通过在图层项目详细信息页面中单击在 ArcGIS Desktop 中打开选项来将图层添加到 ArcGIS Desktop 或 ArcGIS Explorer 中时，此图层不可用。例如，在 ArcMap 中，我看到添加了新图层，但无法对其进行绘制。而在图层项目详细信息页面中单击在 Map Viewer 中打开选项时，则可以在 Map Viewer 中打开和绘制图层，由此可知托管地图服务的服务器已启动和运行。

这是有关托管地图服务图层的 GIS Server 的 REST 配置问题。单击在 ArcGIS Desktop 中打开会导致自动生成的 LYR 文件使用 GIS Server 站点的内部 URL 而非外部 URL。同一问题还会导致地图服务的“REST 服务目录”页面上的在 ArcMap 中查看和在 ArcGIS Explorer 中查看链接失败。此问题只能由托管地图服务的 GIS Server 管理员解决。

如果想要使用存在此问题的 GIS Server 站点中的服务，您仍然可以在 ArcGIS Desktop 或 ArcGIS Explorer Desktop 中通过直接连接到 GIS Server 站点来向地图或 globe 中添加地图服务，步骤如下所示：

  1. 打开要使用的地图服务的详细信息页面，然后单击地图内容链接。
  2. 复制地图内容页面 URL 的第一部分，复制到（并包括）单词“services”，然后删除该字符串的 /rest 部分，因此其格式如下：https://gisserver:6080/arcgis/services。
  3. 打开目录窗口中的添加 ArcGIS Server 对话框，然后将字符串粘贴到服务器 URL 字段中。在 ArcGIS Explorer Desktop 中，打开添加内容对话框，然后将该字符串粘贴到 GIS 服务字段中。此时您已连接到 GIS Server 站点。
  4. 浏览至您要使用的地图服务。
  5. 您还可以联系 GIS Server 站点的所有者，使其了解此配置问题亟需解决。

尝试从 ArcGIS Desktop 登录我的门户时出现软件授权错误。

对 Portal for ArcGIS 进行许可，以允许特定的成员数量。如果门户成员超出该数量，下次尝试从 ArcGIS Desktop 登录门户时会显示以下消息：登录 Portal for ArcGIS 时出错。由于软件授权错误无法登录用户。请联系 Portal for ArcGIS 管理员获取帮助。 门户管理员必须要获得新的许可或从门户中移除成员。

当我在 ArcGIS Desktop 客户端打开图层文件（.lyr 或 .lyrx）或地图文档（.mxd 或 .mapx）时，一些图层会丢失。

您可能不具有基础数据源的访问权限。请联系图层文件或地图文档的所有者，使其与组织或其共享图层文件或地图文档的群组共享上述数据。
