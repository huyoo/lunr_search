网络追踪微件

在本主题中

  1. 配置网络追踪微件
  2. 使用网络追踪微件

“网络追踪”微件旨在调用任何地理处理服务并与其进行交互。输入应为点要素或要素，输出可以返回面、折线或点要素作为结果。例如，可以将此微件配置为使用某个地理处理服务，该服务将追踪几何网络，并返回中断区域和受影响的资产。要了解如何配置此微件以在几何网络上执行隔离追踪，请访问 Water Outage 模板。

配置网络追踪微件

可将此微件设置为应用程序启动时自动打开。要启用该功能，请在微件上单击在应用程序启动时自动打开此微件按钮 ，微件随即变成深绿色。

  1. 将鼠标悬停在微件上方，然后单击配置此微件按钮 以打开配置属性对话框。
  2. 或者，单击更改微件图标，然后用自己的图像替换此微件的默认图标。

    随即显示文件资源管理器窗口，供您选择可用作微件图标的本地图像文件。

  3. 选择设置按钮并提供地理处理服务的任务 URL。

配置设置

配置“网络追踪”微件时，需要考虑以下 4 个部分：工程设置、输入/输出和常规设置。

工程设置

工程设置是可选的。如果配置这些设置，则可以定义工程并保存追踪输入和输入结果。

工程设置选项卡上的选项如下：

  • 工程面图层 - 此面图层必须具有“创建”、“删除”和“更新”编辑功能，并包含名为 name 的字段（字符串类型字段）和名为 globalid 的字段（GlobalID 类型字段）。
  • 输出参数名称 - 任务 URL 中的输出面图层。
  • 工程点图层 - 此点图层必须具有“创建”、“删除”和“更新”编辑功能，并包含名为 inputtype 的字段（字符串类型字段）和名为 projectid 的字段（GUID 类型字段）。

输入和输出

输入/输出选项卡提供的选项可用于配置追踪结果的输入和输出设置。

  1. 对于每个输入参数，请将类型设置为标签、障碍或跳过，并指定工具提示和符号。
  2. 对于每个输出参数，请选择以下任一选项：
    1. 可见 - 在选中的追踪结果中选择可见图层。如果未选中可见，则结果图层将在追踪结果窗格中将输出参数显示为未选中。
    2. 可跳过 - 允许将此图层中的要素传递至跳过输入参数。
    3. CSV - 选择要导出到 .csv 文件的结果。
  3. 要保存输出参数，请选择保存至图层设置下的下拉箭头并选择图层。

    注：
    如果未配置工程设置，则保存至图层选项不可用。

  4. 对于每个输出参数，可以通过选择相应的符号来更新符号系统。
  5. 选择设置按钮 以更新输出结果的标注、工具提示、显示文本和比例。

常规设置

设置追踪结果的摘要并设置缩放设置。

  1. 将输入或输出摘要项目添加到摘要文本。
    1. 单击添加输入/输出计数按钮 以打开添加摘要项目对话框。
    2. 从下拉菜单中单击输入/输出参数。
    3. 根据所需摘要文本，选择运算符参数和字段参数。

      注：
      计数运算符基于图层级别，因此适用于所有输入/输出选项。总和、最小值、最大值和平均数运算符基于字段级别，并且仅当图层中存在数值字段时才会显示。

    4. 单击添加以将摘要项目添加到文本框。
    5. 完成后，单击取消关闭对话框。
    6. 如果需要，可禁用追踪后自动缩放设置。
  2. 如果要将地图缩放至追踪结果，请选择缩放选项。

使用网络追踪微件

配置完成后，自来水公司可以使用此微件来确定管道爆裂时的隔离区域。

  1. 打开网络追踪微件以显示输入选项卡。或者，可以配置自定义 URL，以在设置的位置处运行追踪。要使用自定义 URL 运行追踪，必须将微件设置为自动打开，然后针对自定义 URL 使用以下格式之一：
    • 使用地图的空间参考：https://<domain>/webappbuilder/apps/index.html?id=<itemid>?flags=[[x,y]]&runTrace=true
    • 使用与地图不同的空间参考：https://<domain>/webappbuilder/apps/index.html?id=<itemid>?flags=[[Lat,Long]]&wkid=<wkid>&runTrace=true
  2. 添加标记位置和障碍（如果需要）。

    注：
    按 Ctrl 键激活捕捉功能。

  3. 单击运行开始追踪。

    地图随即缩放至该范围并显示追踪结果。

  4. 查看摘要和输出结果。

  5. 单击保存。

    注：
    必须配置工程设置才能保存追踪。

  6. 在工程属性窗格中提供工程名称和任何附加信息。

  7. 单击确定完成保存结果。
