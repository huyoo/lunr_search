共享微件

在本主题中

  1. 配置“共享”微件
  2. 使用“共享”微件

“共享”微件允许通过以下方式共享应用程序：将应用程序发布到您的社交媒体帐户、发送包含链接的电子邮件或者将应用程序嵌入网站或博客中。该微件还为定义应用程序的 URL 参数提供了一种简便的方式。

配置“共享”微件

可将此微件设置为应用程序启动时自动打开。要启用该功能，请在微件上单击在应用程序启动时自动打开此微件按钮 ，微件随即变成深绿色。

要配置“共享”微件，可选择以下任一共享选项：电子邮件、Facebook、Twitter 和 Google+。

使用“共享”微件

  1. 单击“共享”微件图标将其打开。
  2. 复制共享此应用程序的链接下的短链接，然后将其粘贴到电子邮件、即时消息等中。还可使用提供的按钮将您的应用程序发布到 Facebook、Twitter 或 Google 帐户或者包括在电子邮件中。
  3. 或者，单击链接选项轻松定义短链接中包括的 URL 参数。请参阅使用 URL 参数。
    1. 选中此选项或复选框从以下选项中添加 URL 参数。可以在链接预览框中预览 URL 更改。
      • 当前地图范围 - 默认使用此选项。
      • 单击地图以定义具有缩放级别的地图中心 - 处于选中状态的选项下方将显示一个点图标。单击该图标，然后单击应用程序中要将地图居中的位置。将在地图上添加一个点标记，并基于地图缩放级别相应地更新链接预览中的 URL。
      • 单击地图以定义具有地图比例的地图中心 - 处于选中状态的选项下方将显示一个点图标。单击该图标，然后单击应用程序中要将地图居中的位置。将在地图上添加一个点标记，并基于地图比例相应地更新链接预览中的 URL。
      • 查找位置或要素并缩放至该位置或要素 - 在框中输入一个位置，例如 "Redlands, ca, 92373"，或者可搜索图层中的要素。
      • 查询要素并缩放至该要素 - 从下拉菜单中选择图层、字段和值，以查询某个要素并缩放至该要素。
      • 在地图上添加标记 - 单击点图标，然后单击地图中要添加标记的位置。在每个文本框中填写 WKID、弹出窗口标题、符号和标注属性。
      • 覆盖移动布局的默认屏幕尺寸 - 输入屏幕尺寸，以像素为单位。默认情况下，当屏幕显示的高度或宽度小于 600 像素时，将自动应用移动布局。
      • 设置应用程序的显示语言 - 从下拉菜单中选择一种语言。
      • 记住图层可见性 - 遵循当前地图上图层的可见性。

        警告：
        如果地图内容包含大量图层，请记住图层可见性会导致 URL 过长，由此造成服务器无法对其进行解析。如果可能，“共享”微件将尝试通过使用哈希算法对图层 ID 进行编码来缩短 URL，但是将无法保证 URL 可用。（如果对图层 ID 进行编码，则将在 URL　中使用参数 showLayersEncoded= 或 hideLayersEncoded=。）

        如果　URL　超过　2048　个字符，则将显示一条警告消息（因为大多数服务器对　URL　进行解析时，将限制　URL　中的字符不能超过　2048　个）。在这种情况下，建议忽略可见性信息。

      注：

      出于安全考虑，自 Portal 10.6 起，自动验证用户身份的选项已被移除。

    2. 单击 -> 返回到主面板。应更新短链接以反映上述步骤中对 URL 进行的更改。
  4. 在将此应用程序嵌入到网站中下方，复制 HTML 代码并将其粘贴到您的网站中。如果需要，可单击嵌入选项以选择要嵌入的应用程序的大小。
