Create a project from scratch

在本主题中

  1. Create a feature service
  2. Create a layer view in ArcGIS
  3. Create a QuickCapture project

This exercise explains how to create an ArcGIS QuickCapture project from scratch using ArcGIS Pro. You will create a pothole assessment project that will allow users to capture potholes of two sizes on different road surface types. As the user travels, they can capture the surface type of the road as a line feature, and when they come to a pothole, they can capture its location, distinguished by its size.

Create a feature service

To create a feature service in ArcGIS Pro, complete the following steps:

  1. Sign in to ArcGIS Pro and choose to create a new project using the Catalog template.
  2. Name the project Pothole Assessment and click OK.
  3. On the Catalog tab, select Databases.
  4. Right-click the Pothole Assessment.gdb file, hover over New and select Create new feature class.

    The Create feature class wizard opens.

  5. Do the following in the wizard:
    1. Name the feature class Pothole.
    2. For the feature class type, choose Point.

    3. Click Next.
    4. Add a new field called Size and leave it as the default Text data type.

    5. Add the following fields: UserName (data type Text), Time (data type Date), Direction (data type Double), Speed (data type Double), Latitude (data type Double), and Longitude (data type Double).

      Later, you will configure these fields to automatically populate when a record is captured.

    6. Accept the defaults for the remaining parameters.
  6. Create another feature class for the road assessment, and name it Surface, and assign the Line feature class type.
  7. Add a field called SurfaceType and leave the data type as Text.
  8. Add StartTime and EndTime fields and use the data type Date.
  9. Accept the defaults for the remaining parameters.
  10. Right-click the database and select Domains.
  11. On the Domains tab, create two new domains.
    1. Create a domain named PotholeSize with two coded values: Large (Large >25cm) and Small (Small <25cm}.

    2. Click Save.
    3. Create another domain named SurfaceType with four coded values: Bike Path, Footpath, Freeway, and Main Road.

    4. Click Save.

    Two domains are created.

  12. Add these domains to the appropriate feature class fields.
    1. In Catalog, browse to the Pothole Assessment.gdb file.
    2. Right-click the Pothole feature class, hover over Design, and select Fields to open the Fields tab.

      This tab shows the fields for the Pothole feature class.

    3. In the Domain column of the Size field, click in the intersecting box.

      A drop-down menu appears with the domains you created.

    4. Choose the Pothole Size domain.

    5. Click Save.
  13. Return to the feature class menu in the catalog and repeat step 13 for the SurfaceType feature class, but this time choose the Surface Type domain.
  14. Click Save.
  15. Select the two layers, right-click the selected layers, and choose Add to new map.
  16. In the Contents pane, right-click the Pothole layer and click Symbology.
  17. Click the drop-down menu and symbolize your layer by category using the Unique Values option.

  18. If Size does not automatically appear in the field 1 option, choose Size from the drop-down menu.
  19. Click the add all values button .

    The Pothole layer has unique symbology for small and large potholes.

  20. Repeat steps 17–20 for the SurfaceType layer, creating unique symbology for the different surface types.
  21. Click Save.
  22. Click the Share tab in ArcGIS Pro.
  23. Select Web Layer > Publish Web Layer.

  24. Type Potholes for the map name, summary, and tags.
  25. Select Feature for the layer type.
  26. Select an ArcGIS folder in which to save the layer.
  27. On the Configuration tab, click the edit tool next to Feature.
  28. Select the Enable editing and allow editors to box.

  29. Click Publish.

You now have a Pothole feature layer ready for use.

Create a layer view in ArcGIS

Creating a layer view from your hosted feature layer is recommended but optional. It is best suited for layers that you'll use in different applications. Using a layer view for your QuickCapture project means you can define symbology, or other properties, specifically suited to the project. In other applications, you may decide to use the original hosted feature layer or another layer view.

  1. Sign in to your ArcGIS organization.
  2. Browse to the folder where you saved your Pothole Assessment and click the hosted feature layer.
  3. Click Enable Attachments on the Pothole layer.

    This allows you to capture points with a photo attachment.

  4. Click Create View Layer.
  5. Name your layer Pothole Assessment Project.

    This will ensure that you build your QuickCapture project from the view that is designed for its use. You can modify the symbology for this view layer for use in the project, and the original symbology can still be viewed in other apps.

Create a QuickCapture project

The following steps describe how to create a QuickCapture project from your feature service. This section covers the creation and design process, as well as downloading and capturing points on your device.

  1. Open the QuickCapture website. Sign in to your ArcGIS organizational account.

    ArcGIS Online

    https://quickcapture.arcgis.com

    ArcGIS Enterprise

    https://www.arcgis.com/apps/quickcapture?portalUrl=https://host.domain.com/webadaptor/ (where host, domain, and webadaptor are replaced by your portal information)

  2. Click New Project.
  3. Scroll down to Pothole Assessment Project and select it.
  4. Select Next.

    A dialog box appears where you can confirm the thumbnail, title and data recovery email.

  5. Accept the default settings for the thumbnail and title. The data recovery email is there so that if something happens to the feature layer (for example, it gets deleted), the data collected can be recovered and sent to you. Enter your email address for this setting.
  6. Click Create.

    Your QuickCapture project appears with the automatically generated capture buttons.

  7. Click the Pothole group label and configure the appearance of the group.
    1. Change the label of the group to Pothole Size.
    2. Change the number of columns to 2.
    3. Keep the default state and colors.
    4. Click Save.
  8. Click the button labelled Large in the Pothole group and change the Label to Large >25cm.
  9. Hold down Shift and click the button labelled Small to add it to the selection. This will ensure that subsequent changes to the appearance and data will be applied to both pothole buttons.
    1. Change the size of the button to the smallest size.
    2. Keep the default shape and colors.
    3. Click the Data tab (next to the Appearance tab).
    4. For each capture field (other than Size), click the drop-down arrow and choose Device Variable. Select the appropriate variables from the drop-down menu.

      For example, the Latitude field should have the Latitude variable selected.

    5. Leave the data values for the Size field as they are coded values that are different for each button.
    6. Click Save.
  10. Deselect the Large >25cm button so that only the Small button is highlighted, Change the label to be Small <25cm.
  11. Optionally change the order of the buttons by dragging them to the desired order.

    You cannot drop them into a different group.

  12. Change the order of the Surface buttons to Main Road, Freeway, Footpath, and Bike Path.
  13. Select each button in the Surface Type group by holding down Shift while clicking on each button and on the Data tab, add variables to each of the fields (similar to how you added variables for the pothole buttons).
    注：

    You must fill in the capture fields with the variables or they will not automatically populate when the point is captured. This will leave blank spaces in your attribute table.

  14. Click the group border and change Number of columns to 2.
  15. Click Save.

    Your project preview looks similar to the following:

  16. Select both pothole buttons and click the Data tab (next to the Appearance tab).
  17. Enable the Take Photo toggle button.

    Two icons appear.

  18. Select the eye with the line through it.

    This changes the camera mode to automatic, which allows you to capture the point and a photo without manually taking a picture. This option doesn’t allow a preview of the photo, so make sure when capturing your point, that your device’s camera is pointing at the pothole. This setting is best used when you need to capture the point quickly, for example, if you are driving a car and the camera position is already pointing to the road. If the Take Photo toggle button does not appear, confirm that you enabled attachments on your feature service.

    Your project preview looks similar to the following:

  19. Click Save.

    Next you will use the exclusivity groups options in the Settings menu to modify the surface type buttons. Exclusive groups prevent multiple line or polygon buttons from capturing data at the same time. In this example we will create an exclusivity group so you can't capture multiple surface types simultaneously.

  20. Click the Settings button and select Exclusive groups.
  21. Click New Group and name it Surface Type.
  22. Select all the buttons in the Surface Type group on the project preview, and then click Done.
  23. Click Save.

You can now download your project in the mobile app and test it. For others to be able to use your project, share it in your ArcGIS organization.
