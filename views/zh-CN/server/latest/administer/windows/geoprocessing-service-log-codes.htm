地理处理服务日志编码

本主题列出了使用地理处理服务时将看到的日志消息的编码。每个编码列表后跟您从日志中看到的文本。尽管许多文本一目了然，但仍有一些文本添加了说明以提供进一步信息和疑难解答指导。

编码文本说明

20002

地理处理服务初始化已成功。

20003

地理处理服务初始化失败。

20004

作业目录初始化已成功。

20005

服务启动失败。请确保 <ArcGIS Server 目录路径>\arcgisjobs 目录可以访问。

20006

已成功构造地理处理服务实例。

20007

地理处理服务实例构造失败。

您还将看到显示关于工具和参数的错误或警告编码和消息的文本。

20008

已成功提交作业 <作业 id>。

20009

无法提交作业 <作业 id>。

20010

执行工具时出错。

您还将看到显示关于工具用法的错误或警告编码和消息的文本。

20013

启动配置期间出错:无法添加 <服务器> 类型的隐藏配置。

20014

启动配置期间出错:无法停止 <服务器> 类型的隐藏配置。

20015

启动配置期间出错:无法停止 <服务器> 类型的隐藏配置。

20016

启动配置期间出错:无法删除 <服务器> 类型的隐藏配置。

20017

未找到属性 JobsDirectory。

20018

当前许可不支持地理处理服务。

确保具有 ArcGIS Server Standard 或 ArcGIS Server Advanced 许可。

20019

尚未定义服务器的属性输出目录。

20020

创建目录失败：<目录>。

20021

从 <本地作业目录> 复制到 <共享作业目录> 失败。

20022

执行作业 <作业 id> 消息类型 <类型> 消息大小 <大小>。

消息类型可以是 BINARY、SOAP 或 REST。BINARY 的大小以字节为单位而 SOAP 和 REST 的大小为字符串长度。此编码还通知您关于执行工具所经历的时间。

要查看所经历的时间，通过选择日志 > 查看日志 > 列打开经历的时间列。

20023

加载作业 <作业 id> 消息类型 <类型> 消息大小 <大小>。

消息类型可以是 BINARY、SOAP 或 REST。BINARY 的大小以字节为单位而 SOAP 和 REST 的大小为字符串长度。此编码还通知您用来加载执行请求所经历的时间。

要查看所经历的时间，通过选择日志 > 查看日志 > 列打开经历的时间列。

20024

保存作业 <作业 id> 消息类型 <类型> 消息大小 <大小>。

消息类型可以是 BINARY、SOAP 或 REST。BINARY 的大小以字节为单位而 SOAP 和 REST 的大小为字符串长度。此编码还通知您用来保存执行结果所经历的时间。

要查看所经历的时间，通过选择日志 > 查看日志 > 列打开经历的时间列。

20025

调试消息。

20026

服务器输出目录 <输出目录> 不存在。

您指定的 ArcGIS Server 输出目录不存在。您可能会在更新服务器目录的位置后遇到此消息。可以通过在服务的属性中重新指定输出目录位置来解决此问题。

20027

无法创建输出文件夹 <输出文件夹>。

20028

服务器作业目录 <目录> 不存在。

您指定的 ArcGIS Server 作业目录不存在。您可能会在更新服务器目录的位置后遇到此消息。可以通过在服务的属性中重新指定输出目录位置来解决此问题。

20029

无法重命名作业文件夹 <作业文件夹>。

20030

打开地理处理结果失败 <结果文件的路径>。

20031

无效图层 <图层名称>。

20032

创建结果地图服务失败。
