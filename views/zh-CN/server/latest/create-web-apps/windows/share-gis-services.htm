发布和使用 ArcGIS Server 服务

在本主题中

  1. 使用 ArcGIS Desktop 或 ArcGIS Pro 创建内容并将其共享为 GIS 服务
  2. 通过 REST 端点 (URL) 使用 ArcGIS Server 服务

如果您的组织已部署独立 ArcGIS Server 站点，并且您要将其服务共享到 Web 上，本主题将概述执行此操作的工作流。以下部分将概述通过 REST 端点 (URL) 来创建和使用 GIS 服务所需的步骤

有关构建 web 应用程序的注意事项概述，请参阅在 web 上共享 GIS 内容的工作流。

使用 ArcGIS Desktop 或 ArcGIS Pro 创建内容并将其共享为 GIS 服务

可通过 ArcGIS Desktop 和 ArcGIS Pro 创建、共享和使用 GIS 服务。使用 ArcGIS Pro 和 ArcMap，您可通过信息捕获并保存为要素类、地理数据库或数据库中的表、地图、地理处理工具、图像文件和其他文件等形式对各种地理信息进行构建。您可以发布这些资源并将其共享为在 ArcGIS Server 站点上运行的 GIS Web 服务。可以通过 ArcGIS Server 服务目录获取 GIS 服务 URL。

一旦您的服务在 ArcGIS Server 站点上运行，您和其他用户即可从 ArcGIS Pro 和 ArcMap 连接到该站点并使用地图中的服务。

有关创作 GIS 服务的最佳做法和注意事项，请参阅地图服务规划和地图制作注意事项。

通过 REST 端点 (URL) 使用 ArcGIS Server 服务

如果您的组织使用 ArcGIS Server 的独立配置，或者当前需要使用服务的 REST 端点 (URL) 来构建 web 应用程序，则可通过以下选项来创建 web 应用程序并允许用户使用您的 GIS 服务。

有关本工作流的分步示例，请参阅发布地图服务。

ArcGIS API for JavaScript

有关详细信息，请访问 ArcGIS for Developers 网站上的 ArcGIS API for JavaScript 概述。

ArcGIS Runtime SDKs

可以使用 ArcGIS Runtime SDK 在各种流行的平台和设备上构建和部署原生应用程序。向您的原生应用程序添加强大的空间功能，并使应用程序用户即使在离线时仍可执行所有 GIS 任务。有关详细信息，请参阅 ArcGIS Runtime SDKs参考指南。

REST API

ArcGIS 平台提供多种选项用于构建自定义应用程序，通过 web 服务提供自定义应用程序的许多功能。如果 ArcGIS API for JavaScript 或各种 SDK 解决方案不符合您的开发要求，则可使用 ArcGIS REST API。有关详细信息，请参阅 ArcGIS REST API 概述。

ArcGIS Enterprise 或 ArcGIS Online

如果您可以访问 ArcGIS Enterprise 或 ArcGIS Online 并拥有其凭据，则可以使用 REST 端点 (URL) 向Map Viewer添加 GIS 服务，并将其共享为 web 地图和 web 应用程序。可以通过 ArcGIS Server 服务目录获取 GIS 服务 URL。

有关详细信息，请参阅添加项目主题或 ArcGIS Online 帮助。

注：

Web 地图是随 ArcGIS 平台提供的大部分应用程序的框架。如果您使用门户来创建 web 地图，还可以使用其他多个应用程序。有关详细信息，请参阅发布和使用 Web GIS 服务或在 web 应用程序中共享地图中列出的步骤。
