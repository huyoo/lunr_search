ArcGIS URL 组成

在本主题中

  1. ArcGIS Server 管理 URL
  2. ArcGIS Server 最终用户 URL
  3. ArcGIS Online URL
  4. Portal for ArcGIS URL

统一资源定位器 (URL) 是用来标识不同网站的地址。ArcGIS URL 的某些部分表示变量值，您可为这些变量值提供特定于您站点的信息。ArcGIS URL 的其他部分则为常量；在特定类型的网站中，这些常量始终相同。例如，ArcGIS Server Manager 的 URL 始终包含文字“manager”。

本主题帮助您确定必须对 ArcGIS URL 中的哪些部分进行更改，以便反映组织的正确信息。

ArcGIS Server 管理 URL

可将 ArcGIS Server Manager 用于 ArcGIS Server 站点的大部分管理任务；但有些管理任务要通过 ArcGIS Server Administrator Directory 完成。

ArcGIS Server Manager

ArcGIS Server Manager 是 ArcGIS Server 站点的主要管理工具。

默认条件下，禁止通过 ArcGIS Web Adaptor 访问 Manager；因此在大多数情况下，需要通过以下任一格式的 URL 连接到 Manager：

http://gisserver.domain.com:6080/arcgis/manager

或者

https://gisserver.domain.com:6443/arcgis/manager

ArcGIS Server Administrator Directory

连接到 ArcGIS Server Administrator Directory 以编写 ArcGIS Server 的管理脚本，或者创建几何或搜索服务。

在大多数情况下，使用包含端口号的 URL 连接到 ArcGIS Server Administrator Directory。默认情况下，通过 ArcGIS Web Adaptor 阻止管理员访问；因此，Administrator Directory 的 URL 采用以下任一格式：

http://gisserver.domain.com:6080/arcgis/admin

ArcGIS Server 管理员和发布者连接

ArcGIS Server 管理员可从 ArcMap 或 ArcGIS Pro 连接到其 ArcGIS Server 站点，以执行服务器管理任务的子集，例如注册数据库和管理目录及日志文件。

发布者也可以从 ArcMap 连接到其 ArcGIS Server 站点。虽然发布者连接不具有管理员连接的所有功能，但与用户连接相比，需要更高级权限，以便能够发布服务和注册数据库。

您提供的用于在 ArcMap 或 ArcGIS Pro 中将管理员或发布者连接到 ArcGIS Server 的 URL 取决于是否允许通过 ArcGIS Web Adaptor 进行管理访问。在 10.6 版本之前，默认禁用此选项，但是可以启用此选项；在 10.6 及后续版本中，默认启用此选项，但是可以禁用此选项。

如果禁用通过 ArcGIS Web Adaptor 对 ArcGIS Server 进行管理访问，则用于连接到服务器的 URL 将包含端口号。此 URL 采用以下任一格式：

http://gisserver.domain.com:6080/arcgis

或者

https://gisserver.domain.com:6443/arcgis

ArcGIS Server 最终用户 URL

非管理员（如开发人员或最终用户）也可连接到 ArcGIS Server Services Directory 以编写应用程序或浏览服务器。他们可以连接到 ArcGIS Server Services Directory，或者通过在 URL 中包括服务位置、名称和类型以连接到特定服务。

ArcGIS Server Services Directory

ArcGIS Server Services Directory 通过服务器在视图中显示可用的 REST 资源。

在大多数情况下，使用包含端口号的 URL 连接到 ArcGIS REST Services Directory。此 URL 采用以下任一格式：

http://gisserver.domain.com:6080/arcgis/rest

或者

https://gisserver.domain.com:6443/arcgis/rest

ArcGIS Server 用户连接

使用 ArcMap 或 ArcGIS Pro 中的服务（而并非发布或管理服务）的人员使用 ArcGIS Server 用户连接来访问服务。例如，进行用户连接以添加地图服务作为基础图层或通过 ArcMap 或 ArcGIS Pro 中的要素服务进行编辑。

通过端口进行 ArcGIS Server 用户连接时，URL 采用以下格式之一：

http://gisserver.domain.com:6080/arcgis/services

ArcGIS Online URL

访问 ArcGIS Online 主页以登录到 ArcGIS Online。

如果要登录到使用 ArcGIS.com 身份验证的公共帐户或组织帐户，则 URL 为常量值：

https://www.arcgis.com/home

如果要登录到使用组织中实现的企业级安全保护措施的组织账户，如 LDAP 或 Windows 活动目录，则 URL 为如下格式：

https://organization.maps.arcgis.com/home

Portal for ArcGIS URL

用于连接到 Portal for ArcGIS 的 URL 必须始终包含完全限定的服务器名称。门户的最终用户通过包含 ArcGIS Web Adaptor 名称和主机名称的 URL 进行连接。只有管理员应该使用包含门户服务器名称的 URL。以下部分介绍了不同的门户 URL。

Portal for ArcGIS

为 Portal for ArcGIS 配置 ArcGIS Web Adaptor 时提供的 URL 包含 Portal for ArcGIS 的通信端口。这几乎是唯一一次使用此 URL。

http://portal.domain.com:7080

或者

https://portal.domain.com:7443
