将工作组地理数据库注册到 ArcGIS Server

在本主题中

  1. 安装 SQL Server 客户端
  2. 将 ArcGIS Server 帐户添加到数据库服务器。
  3. 授予 ArcGIS Server 帐户对数据的访问权限
  4. 创建数据库连接文件
  5. 注册工作组地理数据库

作为 ArcGIS Server 管理员或组织中的发布者，您可以注册 ArcGIS Server Web 服务的数据源，用于存储地图和图像缓存，也可用于 GeoAnalytics 和栅格分析输出存储。

要将工作组地理数据库注册到 ArcGIS Server，请执行以下操作：

  • 在 SQL Server Express 实例的所有远程客户端计算机上安装 SQL Server 客户端。
  • 将 ArcGIS Server 帐户添加到数据库服务器。
  • 至少授予对要发布到 ArcGIS Server 帐户的数据的只读权限。
  • 创建与工作组地理数据库之间的数据库连接（.sde 文件）。
  • 导入 .sde 文件以将地理数据库注册到 ArcGIS Server 站点。您可以在 ArcMap 或 ArcGIS Server Manager 中完成此操作。

以下各部分详细介绍了如何准备 ArcGIS 客户端以连接到工作组地理数据库并将其注册到 ArcGIS Server 站点。

安装 SQL Server 客户端

使用 ArcGIS Enterprise 工作组时，ArcGIS Server 和 SQL Server Express 实例必须安装在同一计算机上；因此，您无需在 ArcGIS Server 计算机上安装 SQL Server 客户端，因为进行连接所需的文件已随 SQL Server Express 实例安装。

要从远程 ArcMap 计算机连接至 SQL Server Express 实例，必须安装 SQL Server 客户端。

如果其版本受 ArcGIS 支持，您可以从 Microsoft 下载中心获取并安装 SQL Server 客户端。或者，您可以从 My Esri 下载并安装 SQL Server 客户端。在所有已安装 ArcMap 的计算机上安装此客户端。

如果要在 64 位的操作系统上安装此客户端，可运行 64 位可执行文件；将同时安装 32 位和 64 位文件。无法在 64 位操作系统中运行 32 位 SQL Server 客户端。

将 ArcGIS Server 帐户添加到数据库服务器。

工作组地理数据库独占使用经操作系统 (OS) 身份验证的连接。为地理数据库连接指定 OS 身份验证时，ArcGIS Server 使用 ArcGIS Server 帐户连接到数据。

将 ArcGIS Server 帐户添加到数据库服务器。

  1. 使用数据库服务器管理员的登录帐户登录到客户端，然后启动 ArcMap 或 ArcCatalog。
  2. 双击目录树中的数据库服务器以连接到该服务器。
  3. 右键单击“数据库服务器”，然后单击权限以打开数据库服务器权限对话框。
  4. 将 ArcGIS Server 帐户添加到数据库服务器。
    1. 单击添加用户。
    2. 指定 ArcGIS Server 帐户的本地位置或域位置。
    3. 为 ArcGIS Server 帐户输入名称，然后单击检查名称验证登录名是否有效。
    4. 单击确定关闭选择用户或组对话框。

      此登录帐户即被添加到数据库服务器用户列表中。

    5. 单击确定应用更改并关闭权限对话框。

默认情况下，ArcGIS Server 帐户将被添加为数据库服务器上所有地理数据库中的用户；但并未授予此用户任何特定的地理数据库权限。

接下来，必须授予 ArcGIS Server 帐户对想要注册到 ArcGIS Server 的数据的访问权限。

授予 ArcGIS Server 帐户对数据的访问权限

您必须授予 ArcGIS Server 帐户对地理数据库中数据的访问权限。如果您不打算从地理数据库发布可编辑的要素服务，则只需授予 ArcGIS Server 用户对地理数据库的只读权限。如果您打算发布可编辑的要素服务，还需授予 ArcGIS Server 帐户对数据的读/写权限。如果您打算将地理数据库注册为托管数据库，则必须另外授予对地理数据库上的 ArcGIS Server 帐户的写入权限。

  1. 在目录树中的数据库服务器节点下，双击想要注册到服务器的工作组地理数据库以连接到该地理数据库。
  2. 打开地理数据库权限对话框。

    右键单击该地理数据库，指向管理，然后单击权限。

  3. 从数据库服务器用户列表中选择 ArcGIS Server 用户并为其分配地理数据库的访问权限。
    • 对于已注册数据库，单击只读以授予 ArcGIS Server 帐户对地理数据库中所有数据的只读权限。
    • 对于托管数据库，单击读/写以允许 ArcGIS Server 帐户在地理数据库中创建数据集。
  4. 单击确定保存更改并关闭权限对话框。
    5. 如果您打算从注册的数据库发布可编辑的要素服务，还需授予 ArcGIS Server 帐户对要编辑的特定数据集的读/写权限。只有数据所有者才能更改对其数据集的权限；因此，必要时可请求数据所有者执行以下步骤：
  5. 右键单击其中一个要编辑的数据集，指向管理，然后单击权限。
  6. 从数据库服务器用户列表中选择 ArcGIS Server 帐户，单击读/写，然后单击确定关闭对话框。
  7. 为将通过要素服务进行编辑的各个数据集重复执行上述两步。

创建数据库连接文件

要将工作组地理数据库注册到 ArcGIS Server 站点，必须参考数据库连接（.sde 文件），而非数据库服务器连接（.gds 文件）。

最简单的方法是右键单击目录树中数据库服务器下的地理数据库，然后单击保存连接。连接将被创建到目录树中的数据库连接下。

创建此文件后，可在注册地理数据库时将其导入。

注册工作组地理数据库

可以将工作组地理数据库注册为数据库或托管数据库。这可通过 ArcGIS Server Manager 或 ArcMap 或 ArcCatalog 中的目录树来完成。有关注册选项以及如何注册地理数据库的说明，请参阅以下主题：

  • 关于将您的数据注册到服务器
  • 使用 Manager 将数据注册到 ArcGIS Server
  • 使用 ArcGIS Server 将数据注册到 ArcMap
