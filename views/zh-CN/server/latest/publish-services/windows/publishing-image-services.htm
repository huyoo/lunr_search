哪些数据可发布为影像服务？

可将以下数据发布为影像服务：

  • 栅格数据集
  • 栅格图层
  • 镶嵌数据集
  • 镶嵌图层

取决于软件安装的发布方式

影像服务的发布方式有以下几种，具体取决于您的软件：

  • 只有 ArcGIS GIS Server。

    您只能发布栅格数据集和引用栅格数据集的图层文件。可以使用 ArcCatalog 或 ArcMap 中的目录 窗口发布服务。

  • 具有 ArcGIS GIS Server 和 ArcGIS Image Server。

    您可以发布栅格数据集、镶嵌数据集，或者引用栅格数据集或镶嵌数据集的图层文件。可以使用 ArcCatalog 或 ArcMap 中的目录 窗口发布服务。

    许可：

    当使用函数链中的镶嵌函数发布包含镶嵌数据集或栅格图层的地图文档时，也需要使用 ArcGIS Image Server。如果没有 ArcGIS Image Server，那么基于这些文档创建的服务将不会包含镶嵌数据集或图层。

发布选项

当选择将栅格数据共享为影像服务时，将具有下列选项：

  • 发布服务 - 如果做好了发布的准备，您可以执行连接到 ArcGIS Server 并发布影像服务的整个过程。

  • 保存服务定义文件 - 如果要设置影像服务的所有属性并将这些属性传送到另一台计算机上进行发布，那么您可以创建并保存一个服务定义文件。这在云环境、安全环境或服务器非立即可用的情况下有时是必要的。

    了解有关服务定义的信息

  • 覆盖现有服务 - 如果想覆盖现有服务，可选择此选项。

    了解有关编辑服务的信息

发布影像服务

按照此工作流将影像服务发布至 ArcGIS Server。

连接到服务器

先决条件：

如果您刚刚安装了 ArcGIS Server，那么在 ArcMap 中连接到 ArcGIS Server 并发布服务之前，需要完成一些准备工作： 配置一个单机部署。

警告：

发布影像服务时，会将服务定义和所有数据都转移到服务器上。此操作可能非常耗时，因为大多数栅格数据和镶嵌数据集都非常大。除非您发布的数据量较小，否则建议执行以下任一操作：

  • 确保数据位于一个已注册到服务器中的共享驱动器上。例如，如果要发布一个镶嵌数据集，请利用这一共享位置上的数据在共享位置中创建镶嵌数据集。
  • 确保在服务器上复制数据。例如，镶嵌数据集中所使用的数据在服务器上注册位置中所处的文件夹结构位置与在本地计算机上的文件夹结构位置相同。
  1. 如果尚未连接到服务器，请在目录窗口中展开 GIS 服务器，然后单击添加 ArcGIS Server。
    1. 单击发布 GIS 服务，然后单击下一步。
    2. 对于服务器 URL，键入 ArcGIS Server 管理员 URL，例如 http://gisserver.domain.com:6080/arcgis/admin。
    3. 在服务器类型下拉列表中选择 ArcGIS Server。
    4. 如果服务器管理员已为 ArcGIS Server 启用安全功能，请输入用户名和密码。
    5. 对于其余选项，可接受默认设置，然后单击完成。
  2. 注册数据位置（共享位置或冗余位置）。
    1. 右键单击与服务器的连接，然后单击服务器属性。
    2. 要注册存储栅格数据的文件夹位置，请单击添加已注册的文件夹按钮 ，然后输入所需的信息。

      要注册企业级地理数据库，请参阅将数据注册到 ArcGIS Server。

  3. 单击确定关闭对话框。

发布影像服务

  1. 启动 ArcMap。
  2. 在目录 窗口中，右键单击数据集或图层，然后单击共享为影像服务。
  3. 选择发布为服务，然后单击下一步。
  4. 单击选择连接下拉列表并选择服务器。
  5. 输入服务的新名称（可选）。

    名称长度不能超过 120 个字符，并且只能包含字母数字字符和下划线。

  6. 单击下一步。
  7. 默认情况下，服务会发布到 ArcGIS Server 的根目录 [root]。此外，服务也可组织到根目录下的文件夹中。选择要将服务发布到其中的文件夹，或创建一个包含服务的文件夹。
  8. 单击继续。

    服务编辑器随即显示。您将使用服务编辑器 选择用户可对影像服务执行的操作，还可对服务器显示影像服务的方式进行精细的控制。

    有关特定参数和功能的信息，请参阅影像服务及其功能。

  9. 在服务编辑器 的左侧窗格中，单击功能。

    默认情况下，会启用影像。

  10. 在左侧窗格的功能下，单击影像，然后检查信息。

    URL - 这是客户端用来访问影像服务的 URL。SOAP URL 格式如下：http://gisserver.domain.com:6080/arcgis/services/folder/service/ImageServer。REST URL 格式如下：http://gisserver.domain.com:6080/arcgis/rest/services/folder/service/ImageServer。

    允许的操作 - 使用影像服务时，可以使用多种选项。每个选项都可以限制客户端能够对服务执行的操作。有关详细信息，请参阅影像服务及其功能。

  11. 在服务编辑器的左侧窗格中，单击项目描述并输入影像服务的摘要、标签和描述。
  12. 在服务编辑器中，单击分析 。

    这会对数据集进行检查，看其是否能够发布至服务器。结果将显示在准备 窗口中。在发布影像服务之前，您必须使用错误按钮 修正错误。有关解决这些问题的详细信息，请参阅分析 GIS 资源。

    提示：

    要获得更多视图区域，请单击服务编辑器顶部的折叠按钮 。

  13. 还可以在服务编辑器中，单击预览 。

    这样您便可以了解在 web 上查看数据时数据的外观。有关详细信息，请参阅预览地图。

  14. 修复完所有错误后，单击发布 。
