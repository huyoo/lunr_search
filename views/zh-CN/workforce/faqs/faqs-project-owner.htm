什么是 Workforce 工程？

Workforce 工程是 Workforce web 应用程序和 Workforce 移动应用程序所使用的项目类型，用于在外业和后勤办公室中提供公共视图。当在 web 应用程序打开工程时，工程允许调度员创建并分配工作任务。当使用移动应用程序打开 Workforce 工程时，移动工作人员可通过它看到并完成他们的任务，同时根据需要为他们的工作流启动其他应用程序。工程包括参考地图，分配用户调度员或移动工作人员角色，追踪工作人员以及定义移动工作人员可通过 Workforce 启动的应用程序（例如 Collector for ArcGIS 等）。

在工程创建期间，将为您创建工程及其使用的地图和图层。这些地图包括一个用于移动工作人员的地图（由移动应用程序使用）和一个用于调度员的地图（由 web 应用程序使用）。两个地图用来向移动工作人员和调度员呈现不同但相似的系统视图。这些地图是基于模板、通过使用预定义符号系统和配置（例如弹出窗口）的要素图层以及使用地形底图创建的。您可以修改这些地图，根据用户将对工程执行的工作添加其他业务或资产要素图层。有关示例，请参阅“创建您的第一个工程”中的自定义地图。

所创建的四个图层基于预定义模板并用来存储特定于 Workforce 工程的信息。这些图层包含任务、移动工作人员（包括他们最后的已知位置）、调度员以及有关移动工作人员去向的追踪信息（如果启用了位置追踪）。web 应用程序和移动应用程序都需要要素图层才能包含必填字段。

工程创建完成后，会在 ArcGIS 组织中创建一个文件夹和一个群组。工程及其所有相关项目（地图和图层）均存储在已创建的文件夹中，并均已共享至群组。当您将调度员和移动工作人员添加到工程中时，会自动将他们添加到群组中，并授予他们对项目、地图和图层的访问权限。作为工程所有者，您是唯一可以配置或删除该工程的人员。作为一个工程所有者，您也会被作为调度员添加到工程中，以便您可以测试此工程。

由于 Workforce 提供了所有这些组件，因此您可以在创建 Workforce 工程后，通过最少的配置立即使用此工程。

警告：

虽然在浏览器中登录到 ArcGIS 后，工程、地图、图层和群组都在您的门户中且可进行编辑，但您还是需要注意在 Workforce 应用程序之外所做的更改。如果您决定在 Workforce 之外操作这些项目，您应该避免以下操作：

  • 取消与工程群组共享任意项目
  • 删除工程的群组
  • 删除或修改图层中的预定义字段。
  • 删除或修改预定义图层。

虽然可以更改项目的所有权，但请谨慎执行该操作，确保遵照疑难解答：如何将工程所有权移至其他成员？中的指导原则。

我可以将哪些应用程序与 Workforce 相集成，以及这些集成能够提供什么？

Workforce 支持与 Collector for ArcGIS、Explorer for ArcGIS、Navigator for ArcGIS 以及 Survey123 for ArcGIS 进行集成。应用程序集成通过提供方向以及完成工作所需的地图和调查，最大限度地提高移动工作人员的效率。

Collector 是一个以地图为中心的外业数据采集应用程序，用于采集新信息并检查组织中管理的现有数据。添加 Collector 集成，以便 Workforce 打开正确的地图并找到快速数据收集任务。您还可以将全局 ID、任务 ID 和位置传递到 Collector 中的特定字段。

Explorer 是一个地图查看应用程序，用于访问您的数据和资产信息。添加 Explorer 集成，以便 Workforce 可以集中于您的任务，打开正确的地图。

Navigator 可通过 Esri 提供的地图或您自己的地图让您的移动工作人员到达他们的任务位置。添加 Navigator 集成，以便 Workforce 可以打开 Navigator 来提供从工作人员当前位置到任务位置的方向。默认情况下，会启用与 Navigator 的集成。

Survey123 是一个以表单为中心的数据收集应用程序，用于通过预定义问题（使用条件逻辑来提供易于填写的答案）执行调查。添加 Survey123 集成，以便 Workforce 打开正确的调查。您还可以将全局 ID、任务 ID 和位置传递到 Survey123 中的特定问题。

需要下列版本：

  • Collector for ArcGIS 17.0.3 或更高版本
  • Explorer for ArcGIS 17.1.0 或更高版本
  • Navigator for ArcGIS 2.0 或更高版本
  • Survey123 for ArcGIS 1.6 或更高版本

当与 Explorer 或 Collector 集成时，您需要提供地图。当与 Survey123 集成时，您需要提供调查。有关配置应用程序的详细信息，请参阅提示：集成其他 Esri 应用程序。

警告：

移动工作人员需要分别登录到每个应用程序，因为凭据不会在应用程序之间传递。

会将什么类型的通知发送给移动工作人员？

通知用于在任务列表刷新（每 60 秒刷新一次）时提示移动工作人员注意某些任务。

当应用程序在前台运行时，会为新分配的任务添加通栏。如果任务重要，通栏以红色持续显示在屏幕上，直到您与其或任务交互。此外，重要任务会在其他屏幕（例如工程屏幕）中显示对话框。如果任务不重要，那么通栏会以蓝色临时显示。非重要任务通知只会在任务列表中显示。

如果应用程序在后台运行，则使用可预测性较差的 Apple 通知。通知时间取决于您使用应用程序的频度。因此，使用频度较高的应用程序的通知比新的或较少使用的应用程序更快。

如果应用程序关闭，则不会发送通知。团队目前正在研究这种行为。

我可以将 Workforce 与我的工作指令或资产管理系统相集成吗？

与工作指令或资产管理系统相集成是一个常见请求。通常情况下这种集成可自动化任务创建过程。您可以通过自动化和脚本设置创建用于 Workforce 中的任务。

如何创建循环或重复任务？

创建循环和重复任务是一个常见请求。您可以使用自动化和脚本在 Workforce 中创建这些类型的任务。
